## Fix Details

### SQL Quoting (Q-01 through Q-07)

All SQL table references now go through `_qtable(table_name)` from `ingest/metadata.py`, which calls `_validate_table_name()` internally and returns `"data"."<table>"` with proper double-quoting. Column names in `construct_point_geometry`, `construct_wkt_geometry`, and `ensure_geom_column` now use `_sql_quote_ident(col)`, which doubles any embedded double-quotes per PostgreSQL identifier quoting rules. The local `_validate_table_name()` in `features/service.py` was removed entirely — it only validated without quoting, giving a false sense of safety.

### Exception Annotation (E-01 through E-06)

38 `except Exception` blocks across 6 files annotated with `# broad: <justification>`. Justifications fall into three categories:
- `broad: background task — must not crash worker` (tasks_common, tasks_vrt, ai/router)
- `broad: re-raised as HTTP exception after logging` (ingest/router, metadata)
- `broad: catches all embedding/vector store errors for graceful degradation` (embeddings/service)

Ruff auto-reformatted the inline comments to multi-line `except (\n    Exception\n):  # broad:` style; verified `ruff format --check` and `ruff check` both pass.

### Stale Artifacts (A-01 through A-03)

Three root-level snapshot files removed via `git rm`. These were one-time diagnostic dumps that had drifted out of sync with the codebase and provided no ongoing value.

### Timer Cleanup (T-01 through T-06)

All six components converted from fire-and-forget `setTimeout` to the project-standard ref-based cleanup pattern:
```typescript
const timerRef = useRef<ReturnType<typeof setTimeout>>(undefined);
useEffect(() => () => clearTimeout(timerRef.current), []);
// In handler:
clearTimeout(timerRef.current);
timerRef.current = setTimeout(() => setState(resetValue), delayMs);
```
This prevents "Can't perform a React state update on an unmounted component" if the user closes/navigates away during the timeout window.

### Cache Hygiene (C-01, C-02, C-03)

- `use-map-thumbnail.ts`: `staleTime: Infinity` → `staleTime: 60_000` (1 minute). Thumbnails are regenerated server-side on re-upload; stale-forever caching means the UI never reflects an updated thumbnail until hard refresh.
- `use-settings.ts:useConfigMode`: Left as `staleTime: Infinity`. Config mode (`env` vs `file`) is determined at server startup and cannot change at runtime without a restart. Infinity is correct here.
- `platform/cache/redis.py`: Not present in this repo variant — skipped.
