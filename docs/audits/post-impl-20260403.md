## Executive Summary

GeoLens remains a well-maintained codebase — zero ruff violations, zero TypeScript errors, clean console (all `console.debug` gated behind `import.meta.env.DEV`), and no TODO/FIXME comments. The test suite has grown significantly since the last audit (904 frontend, 675+ backend passing locally). The single crash-risk finding is **10+ routes missing React `errorElement`** — render errors on public-facing pages like Search, Collections, Maps, Login, and all admin pages will show a white screen. The highest-impact performance finding is the **per-tile DB lookup and ST_Transform** on the hottest path in the system. The top structural debt remains the **ingest pipeline duplication** (300-line `ingest_file` + 195-line `ingest_service` sharing ~80 lines of copy-paste) and the **ViewerMap/map-sync.ts layer sync duplication**. Pydantic schema input validation (`max_length`, numeric bounds) remains the most impactful type safety gap.

## 2. Performance

### Findings

1. **[PERF] ST_Transform per tile request** — `backend/app/tiles/service.py:57-76`. Every tile transforms geometry from 4326→3857 on the fly. Pre-computing `geom_3857` during ingestion would eliminate per-row CPU cost on the hottest path. **(P1)**

2. **[PERF] Dataset ORM lookup on every tile request** — `backend/app/tiles/router.py:376-381`. Full SQLAlchemy ORM query with `joinedload(Dataset.record)` before cache check. An in-memory LRU cache of `(table_name → visibility, column_info)` would avoid hitting DB per tile. **(P1)**

3. **[PERF] N+1 in `add_datasets_to_collection`** — `backend/app/collections/service.py:132-140`. Each dataset_id triggers a separate SELECT. Should batch with `WHERE ... IN (...)`. **(P1)**

4. **[PERF] N+1 in `generate_distributions`** — `backend/app/records/service.py:353-378`. Up to 6 sequential SELECTs per dataset ingestion. Should batch-fetch existing distributions. **(P2)**

5. **[PERF] `useSharedMap` has no staleTime** — `frontend/src/hooks/use-maps.ts:107-114`. Read-only shared map data refetches every 30s. Should be `staleTime: 5 * 60_000`. **(P2)**

6. **[PERF] `useMap` detail hook inherits 30s staleTime** — `frontend/src/hooks/use-maps.ts:35-41`. Map builder data changes only via explicit saves; 60s+ would reduce unnecessary refetches. **(P2)**

7. **[PERF] Full geometry in search results** — `backend/app/search/service.py:1129-1141`. List endpoint serializes full `spatial_extent` polygon for every result. Frontend only needs bbox. **(P2)**

8. **[PERF] `get_map_with_layers` returns column_info and sample_values** — `backend/app/maps/service.py:131-148`. Not needed for viewer, only builder. Lighter response for public viewer would reduce payload. **(P3)**

9. **[PERF] Non-memoized builder handlers** — `frontend/src/hooks/use-builder-layers.ts`. ~20 handler functions recreated on every render; passed to list items. The most impactful to memoize: `handleToggleVisibility`, `handlePaintChange`, `handleOpacityChange`. **(P3)**

10. **[PERF] Missing staleTime on 5 additional hooks** — `use-dataset.ts`, `use-maps.ts` (multiple hooks). Minor unnecessary refetches. **(P2)**

### Recommendations

1. Add in-memory LRU cache for tile endpoint dataset lookups
2. Pre-compute `geom_3857` column during ingestion to eliminate per-tile ST_Transform
3. Batch-check collection membership instead of N+1 per-dataset SELECT

9. **[TYPE]** `SharedLayerResponse` missing `layer_type`, `dataset_record_type` in frontend

**Missing response_model (P2):**
11. **[TYPE]** `GET /{dataset_id}/maps/` — `backend/app/datasets/router_data.py:153`. Returns `MapListResponse` but no `response_model` declared.
12. **[TYPE]** `GET /.../related/{relationship_id}/` — `backend/app/datasets/router_metadata.py:385`. Returns dict with no schema.

**Type assertions (P3):**
13. **[TYPE]** `ChatPanel.tsx:122,192` — `action.geojson as unknown as GeoJSON.FeatureCollection`. Fix by typing `ChatAction.geojson` as `FeatureCollection`.
14. **[TYPE]** `LayerFilterEditor.tsx:156` — `expr[i] as unknown[]`. Legitimate workaround for MapLibre recursive union type.

**Untyped parameters (P3):**
15. **[TYPE]** `maps/service.py:22` — `map_obj` has no type annotation
16. **[TYPE]** `datasets/service.py:67` — `request` has no type annotation

**Error response consistency (P2):**
17. **[TYPE]** 40+ occurrences of `detail=str(exc)` passing raw Python exception messages to client. Some leak internal details.
18. **[TYPE]** No `ErrorResponse` Pydantic model exists. Ad-hoc error `detail` strings with no machine-parseable codes.

### Recommendations

1. Add `max_length` to all user-input string fields in Pydantic schemas (DatasetCreate, DatasetMeta, LoginRequest, MapCreate, CollectionCreate)
2. Add `ge`/`le` bounds to `MapUpdate.zoom`, `.bearing`, `.pitch` and `MapLayerInput.sort_order`
3. Add missing fields to frontend types to match backend schemas

## 5. Error Handling & Resilience

### Findings

1. **[RESILIENCE] 10+ routes missing `errorElement`** — `frontend/src/App.tsx:47-57`. Routes: `/login`, `/register`, `/oauth/callback`, `/m/:token`, `/`, `/collections`, `/collections/:id`, `/maps`, `/settings`, all admin sub-routes. Render errors show white screen. **Severity: crash. (P0)**

2. **[RESILIENCE] `pool_pre_ping` defaults to False** — `backend/app/config.py:91`. Stale connections after DB restart cause 500s. **Severity: degraded. (P1)**

3. **[RESILIENCE] ViewerMap missing `map.on('error')` handler** — `frontend/src/components/viewer/ViewerMap.tsx`. Public viewer (most exposed surface) doesn't handle tile load errors, unlike BuilderMap and DatasetMap which do. **Severity: degraded. (P1)**

4. **[RESILIENCE] LLM clients have no retry** — `backend/app/ai/llm_loop.py:204,302`. Transient 429/503 fails immediately. Both SDKs support `max_retries=2`. **Severity: degraded. (P1)**

5. **[RESILIENCE] Embedding client has no timeout** — `backend/app/embeddings/helpers.py:87-93`. `OpenAI()` constructed without timeout; `asyncio.to_thread` will block indefinitely if API hangs. **Severity: degraded. (P1)**

6. **[RESILIENCE] Titiler proxy creates new HTTP client per request** — `backend/app/tiles/router.py:244`. No connection reuse, no retry. **Severity: degraded. (P2)**

7. **[RESILIENCE] No catch-all `Exception` handler** — `backend/app/main.py`. Only HTTPException, RequestValidationError, and RateLimitExceeded have custom handlers. Unhandled exceptions return Starlette's default plain text 500. **Severity: degraded. (P2)**

8. **[RESILIENCE] LoginPage auth config error silently hides registration link** — `frontend/src/pages/LoginPage.tsx:40`. **Severity: cosmetic. (P3)**

### Recommendations

1. Add `errorElement={<RouteErrorBoundary />}` to the `<AppLayout>` route to catch errors from all child routes
2. Set `pool_pre_ping=True` as the default in config
3. Add `map.on('error')` to ViewerMap and configure `max_retries=2` on LLM/embedding clients
