Perform a clean rebuild and capture every signal. Steps in order:

1. Verify `.env` exists in the repo root. If missing, copy `.env.example` to `.env` (or note it in SUMMARY Issues — do NOT proceed without a real .env).

2. Bring the stack down with volumes wiped to guarantee a clean rebuild:
   `docker compose down -v`
   Capture the output. If any container fails to stop cleanly, record it under Issues.

3. Rebuild and start the default (non-profile) services detached:
   `docker compose up -d --build`
   This rebuilds db, migrate, api, worker, frontend; pulls titiler. Run with a 600s timeout. Capture full stdout+stderr.

4. Wait for healthchecks. Poll `docker compose ps --format json` in a loop (every 5s, max 180s) until every service shows `health: healthy` OR you've waited 3 minutes. Print the final `docker compose ps` table.

5. Inspect logs for each service to surface errors/warnings the healthcheck didn't catch:
   - `docker compose logs --no-color --tail 200 db`
   - `docker compose logs --no-color --tail 200 migrate`
   - `docker compose logs --no-color --tail 200 api`
   - `docker compose logs --no-color --tail 200 worker`
   - `docker compose logs --no-color --tail 100 titiler`
   - `docker compose logs --no-color --tail 100 frontend`
   Grep each for `ERROR|WARNING|Traceback|FATAL|failed|Exception|fail|error:` (case-insensitive). Record any non-trivial hits with the originating service.

6. Smoke-probe the API and frontend:
   - `curl -sf http://localhost:8080/api/health` (or fall back to `http://localhost:8001/health` if API_PORT was overridden) — expect 200
   - `curl -sf http://localhost:8080/` — expect 200 (Vite dev server)
   - `curl -sf http://localhost:8080/api/collections/ -H "Accept: application/json"` — expect 200 (catalog may be empty at this point — that is fine, just confirm the endpoint responds)

7. Note in the working memory: which services took longest to become healthy, any healthcheck retries observed, and the migrate exit status.

  <name>Task 2: Import canonical seed data and probe catalog state</name>
  <files>(no code changes — observation only; capture command output for Task 3)</files>
  <action>
Run the canonical seed scripts against the running stack. Steps:

1. Ensure `httpx` is installed in the local shell environment (the seed scripts run on the host, not in a container):
   - Try `python3 -c "import httpx"` first.
   - If missing, install: `pip install httpx` (or `uv pip install httpx` if uv is available).
   - If install fails, record under Issues and SKIP to step 5 with a `gap: httpx unavailable on host` finding.

2. Mint a JWT for the seed scripts (most seed scripts authenticate via API key; seed-e2e.py uses admin login). Per MEMORY.md and README:
   `TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login/ -d 'username=admin&password=admin' | python3 -c "import json,sys; print(json.load(sys.stdin).get('access_token',''))")`
   Verify TOKEN is non-empty. If empty, capture the raw login response and record under Errors.

3. Run `python3 scripts/seed-natural-earth.py --help` first to discover supported flags. Then run the full seed (most common invocation pattern is `python3 scripts/seed-natural-earth.py --base-url http://localhost:8080 --api-key <token-or-api-key>`; consult `--help` output for exact flag names — could be `--token`, `--jwt`, or `--api-key`). Use a 1200s timeout. Capture full stdout+stderr to memory or `/tmp/260523-at1-seed-ne.log`.

4. After Natural Earth completes, probe the catalog:
   `curl -s "http://localhost:8080/api/collections/?limit=200" -H "Authorization: Bearer $TOKEN" | python3 -c "import json,sys; d=json.load(sys.stdin); cols=d.get('collections') or d.get('features') or []; print(f'collections_count={len(cols)}'); [print(' -', c.get('id') or c.get('properties',{}).get('id') or c.get('title')) for c in cols[:30]]"`
   Record the count and the first 30 collection IDs.

5. Check whether the e2e fixtures are already present:
   `curl -s "http://localhost:8080/api/collections/ne_10m_admin_0_countries" -H "Authorization: Bearer $TOKEN" -o /dev/null -w "%{http_code}\n"`
   `curl -s "http://localhost:8080/api/collections/ne_10m_reefs" -H "Authorization: Bearer $TOKEN" -o /dev/null -w "%{http_code}\n"`
   If BOTH return 200, skip the seed-e2e.py invocation (Natural Earth already covers them) and note that under Imported.
   If EITHER returns 404 (or anything non-2xx), run `python3 scripts/seed-e2e.py --help` then `python3 scripts/seed-e2e.py --base-url http://localhost:8080 [auth-flag-per-help]` with a 600s timeout. Capture stdout+stderr.

6. Final catalog probe — record the final collection count and a sample of titles:
   `curl -s "http://localhost:8080/api/collections/?limit=500" -H "Authorization: Bearer $TOKEN" | python3 -c "import json,sys; d=json.load(sys.stdin); cols=d.get('collections') or d.get('features') or []; print(f'final_collections_count={len(cols)}')"`

7. Run a `docker compose logs --no-color --since 10m api worker` and grep for ERROR/Traceback/Exception to surface any background ingest failures (worker processes the ingest jobs async). Record any hits.

Do NOT write SUMMARY.md yet — Task 3 consolidates.
