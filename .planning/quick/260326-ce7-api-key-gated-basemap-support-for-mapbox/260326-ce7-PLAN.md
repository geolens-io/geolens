**schemas.py:**
1. Add `api_key: str | None = None` to `BasemapEntry` model (after `attribution`).
2. Update `validate_tile_url`: The current validator rejects URLs without `{z}/{x}/{y}` unless they end with `.json` or contain `/styles/`. URLs like `https://api.maptiler.com/maps/streets-v2/style.json?key={api_key}` have `.json` in the path but NOT at the end of the full string (query params follow). Fix: strip query string before checking `.json` ending. Use `v.split('?')[0].rstrip('/').endswith('.json')` instead of `stripped.endswith('.json')`. The `/styles/` check already works for Mapbox URLs. Also accept URLs containing `{api_key}` in a query param as a signal that this is a provider style URL — but only if the base path looks like a style URL (has `/styles/` or `.json` in path).

**router.py — `get_basemaps()`:**
1. After fetching stored basemaps, process each entry:
   - If `{api_key}` is in `entry["url"]` and `entry.get("api_key")` is falsy → **exclude** from response (basemap not ready).
   - If `{api_key}` is in `entry["url"]` and `entry.get("api_key")` is set → **replace** `{api_key}` in the URL with the actual key value, then set `api_key` to `None` in the response (do not leak the key to non-admin callers).
   - If URL has no `{api_key}` → return as-is.
2. Build the response list from the filtered/resolved entries.

Note: The admin endpoint `get_all_settings()` returns raw values from PersistentConfig — it already returns the unmodified basemap data including `api_key` fields, which is correct for the admin UI to edit.
