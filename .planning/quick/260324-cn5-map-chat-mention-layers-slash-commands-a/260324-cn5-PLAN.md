    **chat-enrichment.ts** -- Pure utility module, no React dependencies.

    Export `enrichMessage(raw: string, layers: MapLayerResponse[]): string`:
    1. Parse @mentions using regex. Layer names may contain spaces -- use bracket syntax `@[My Layer]` for names with spaces, plain `@LayerName` for single-word names. Match against known layer display_name/dataset_name (case-insensitive).
    2. For each resolved mention, append a context block: `[Context: @DisplayName = layer_id:{id}, {geometry_type}, {feature_count} features, columns: {top 5 cols as name(type)}]`
    3. Detect leading slash commands: if message starts with `/style`, `/filter`, `/label`, `/query`, or `/add`, strip the command prefix and prepend `[Intent: {command}] ` to the message.
    4. Both enrichments can co-occur (e.g., `/filter @Parcels where value > 100k`).
    5. Unresolvable @mentions are left as-is (no error, LLM handles gracefully).

    **chat-suggestions.ts** -- Export `getSmartSuggestions(layers: MapLayerResponse[]): string[]`:
    1. Geometry-aware suggestions per CONTEXT.md decisions:
       - Point layers without style_config: "Create a heatmap of @{name}", "Cluster @{name} points"
       - Point layers with numeric columns: "Size @{name} by {numeric_col}"
       - Polygon layers with numeric columns without style_config: "Color @{name} by {numeric_col}"
       - Polygon layers: "Show area labels on @{name}"
       - LineString layers with numeric columns: "Vary @{name} width by {numeric_col}"
       - Raster layers: "Adjust @{name} opacity"
    2. Column-type-aware suggestions:
       - Numeric columns: "Show distribution of {col} in @{name}"
       - Text columns: "Color @{name} by {text_col} categories"
       - Timestamp columns: "Filter @{name} by date range"
    3. Always include "Add another dataset" as last suggestion.
    4. Return max 4 suggestions. Prioritize: layers without style > layers without labels > column-specific.
    5. Detect column types using substring matching: numeric/integer/float/double for numeric; text/varchar/character for text; timestamp/date for temporal.

    **MentionDropdown.tsx** -- Lightweight dropdown component.
    Props: `{ items: { id: string; label: string; description?: string }[]; selectedIndex: number; onSelect: (index: number) => void; type: '@' | '/' }`
    1. Render as absolutely-positioned div above the input (bottom-full), full width of parent.
    2. Show icon prefix: `@` items show a layers icon, `/` items show a terminal/command icon.
    3. Each item shows label (bold for match portion if feasible, otherwise plain) and optional description.
    4. Highlighted item uses `bg-accent` class. Items are clickable (onSelect).
    5. Max height with overflow-y-auto for long lists (unlikely with ~15 layers but safe).
    6. Use role="listbox" with role="option" on items for accessibility. Active item gets aria-selected.
    7. Style with existing Tailwind classes: rounded-lg, border, bg-popover, shadow-md, text-sm.

  <action>
    Create a new ChatInput component that wraps a textarea with autocomplete trigger detection.

    **Props interface:**
    ```typescript
    interface ChatInputProps {
      value: string;
      onChange: (value: string) => void;
      onSubmit: () => void;
      layers: MapLayerResponse[];
      disabled?: boolean;
      placeholder?: string;
    }
    ```

    **Internal state:**
    - `triggerState: { type: '@' | '/'; startIndex: number; query: string } | null`
    - `selectedIndex: number` (dropdown highlight position)
    - `textareaRef: React.RefObject<HTMLTextAreaElement>`

    **Trigger detection** (on every value change + cursor move):
    - Walk backwards from cursor (selectionStart) to find `@` or `/` not preceded by a word character.
    - `@` triggers layer list filtered by query (fuzzy = case-insensitive includes).
    - `/` triggers command list filtered by query. Commands: `[{ id: 'style', label: '/style', description: 'Change layer appearance' }, { id: 'filter', label: '/filter', description: 'Filter features' }, { id: 'label', label: '/label', description: 'Add or change labels' }, { id: 'query', label: '/query', description: 'Ask data questions' }, { id: 'add', label: '/add', description: 'Add a dataset' }]`.

    **Layer items for dropdown:**
    - Map each layer to `{ id: layer.id, label: layer.display_name ?? layer.dataset_name, description: layer.dataset_geometry_type ?? undefined }`.

    **Keyboard handling (onKeyDown):**
    - When dropdown is open (triggerState !== null AND filtered items > 0):
      - ArrowDown: preventDefault, increment selectedIndex (wrap to 0)
      - ArrowUp: preventDefault, decrement selectedIndex (wrap to end)
      - Tab or Enter: preventDefault, call selectItem()
      - Escape: preventDefault, clear triggerState
      - All other keys: pass through normally
    - When dropdown is closed:
      - Enter (without Shift): preventDefault, call onSubmit()

    **selectItem():**
    - For `@` mentions: replace from triggerState.startIndex through current cursor with the layer name. If name contains spaces, insert `@[LayerName] ` (bracket syntax). If single word, insert `@LayerName `. Add trailing space.
    - For `/` commands: replace from triggerState.startIndex through current cursor with `/command ` (with trailing space).
    - After insertion, set cursor position to end of inserted text using textareaRef.current.setSelectionRange() in a requestAnimationFrame or setTimeout(0) to run after React re-render.
    - Clear triggerState after selection.

    **Textarea element:**
    - Use `<textarea>` with `rows={1}`, auto-resize on content change (scrollHeight-based, max 4 lines ~96px).
    - Apply same styling as the existing Input: `text-sm h-8` base, growing as needed. Use className from shadcn input pattern: `flex w-full rounded-md border border-input bg-background px-3 py-1.5 text-sm ring-offset-background placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50 resize-none`.

    **Dropdown rendering:**
    - Wrap textarea in a `relative` div.
    - When triggerState is set and filtered items exist, render `<MentionDropdown>` above the textarea.
    - Pass filtered items, selectedIndex, onSelect handler, and trigger type.

    **Reset selectedIndex** to 0 when query changes.

    **Important edge cases per RESEARCH.md:**
    - Clear triggerState when user clicks away or moves cursor before the trigger character.
    - On selection via MentionDropdown click (onSelect), call selectItem with that index.
  </action>
