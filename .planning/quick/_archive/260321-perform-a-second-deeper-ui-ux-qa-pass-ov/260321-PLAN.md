## Objective

Run a deeper, second-pass QA over the 4 record-detail variants (vector/raster/VRT dataset detail plus collection detail), focusing on responsive behavior, error/fallback paths, and a11y/keyboard. Output:
1) an updated findings list (with baseline re-validation), and
2) a milestone handoff doc with prioritized workstreams, acceptance criteria, and implementation guidance.

## Scope

- Targets: reuse `/Users/ishiland/Code/geolens/.planning/quick/260320-use-the-playwright-mcp-server-to-qa-all-/qa-targets.json`
- Viewports: Desktop (1280x800) and Mobile (375x812)
- Coverage emphasis (deeper than first pass):
  - VRT preview failure path: on-screen error state + retry/backoff behavior (no infinite noisy loops)
  - Mobile header/title/actions: clamping, overflow, and tap targets
  - A11y/keyboard: tab-order sanity, focus visibility, axe scan excluding MapLibre
  - Collection shell: confirm and document intentional differences vs datasets (or mark as inconsistency gap)
- Non-goals: implementing fixes (this quick task is QA + handoff)

     - Console messages written to:
       `${QA_OUTPUT_DIR}/logs/console-<record_type>.log`
     - Network request failures written to:
       `${QA_OUTPUT_DIR}/logs/network-<record_type>.log`

3. Assertions should be high-signal (avoid brittle pixel/layout snapshots):
   - Exactly one visible H1 on the page.
   - Dataset pages: `data-testid="dataset-top-summary"` exists and is visible (this is a key UX shell element).
   - Collection pages: confirm the primary heading exists and that the page shows the dataset list section.
   - Mobile header sanity: ensure the H1 is not collapsed to 0/1 characters (guard against the raster one-letter regression seen in the first pass).

4. VRT-preview error detection (the P0 from the first pass):
   - Track failed network requests (status >= 500 or `requestfailed`) while the hero/preview settles.
   - If the page produces repeated failed tile/preview fetches beyond a small threshold (e.g. > 10 failures within 10s), fail the test with a clear message that points to the saved network/console logs.
   - This does not “fix” the underlying issue, but it ensures we can reliably reproduce and quantify it during QA.

5. A11y scan per page:
   - Use `@axe-core/playwright` similarly to `e2e/accessibility.spec.ts`.
   - Exclude `.maplibregl-map` (WebGL canvas cannot be inspected by axe).
   - If violations exist, fail with a formatted output and (optionally) attach the formatted text to the Playwright report AND persist it under `${QA_OUTPUT_DIR}/logs/axe-<record_type>-<viewport>.log`.
