## What Was Built

### Task 1: Coordinate pill + legend offset
- `MapCoordReadout.tsx`: position changed from `bottom-1.5 right-1.5` to `top-2 right-2`. Clears the MapLibre attribution control at bottom-right in both BuilderMap and ViewerMap contexts.
- `WidgetHost.tsx`: `bottom-left` anchor reduced from `bottom-20` (80px) to `bottom-10` (40px). Legend now sits tighter to the corner while still clearing the ScaleControl (~24px tall).

### Task 2: Blank basemap
- `basemap-utils.ts`: Added `BLANK_BASEMAP_ID = 'blank'` constant, an inline SVG thumbnail (dashed rect with diagonal slash), and a `toMaplibreStyle` branch that returns a `StyleSpecification` with a transparent `background` layer and glyphs URL.
- `BasemapPicker.tsx`: Synthetic blank entry prepended to the options array before the grid render. `current` lookup searches `options` (including blank) so collapsed button shows correct thumbnail when blank is selected.
- i18n: `basemap.blank` key added in en (`None`), es (`Ninguno`), fr (`Aucun`), de (`Keine`).
- Tests: 3 new tests for blank `toMaplibreStyle` (transparent background, empty sources, glyphs) and `basemapThumbnail` return value.

### Task 3: Measurement widget conflict
- `BuilderMap.tsx`: Imported `useWidgetStore`. Added `measureActiveRef` (a `useRef<boolean>`) updated via `useWidgetStore.subscribe()` in a one-time `useEffect`. Added `if (measureActiveRef.current) return` guards at the top of `handleClick` and inside the `handleMouseMove` rAF callback (after the `isStyleLoaded` check). When measurement is active: clicks bypass feature popup logic, cursor stays at `crosshair` set by MeasurementWidget.
