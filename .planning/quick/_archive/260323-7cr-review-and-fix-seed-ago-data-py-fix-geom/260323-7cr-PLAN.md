# Quick Task Plan: Fix Ingest Geometry Column Bug

## Root Cause

When `run_ogr2ogr` imports a file and the PostGIS table already exists from a prior failed ingest:
1. ogr2ogr defaults to **append mode** (no `-overwrite` flag)
2. `-lco GEOMETRY_NAME=geom` is silently ignored in append mode
3. The pre-existing table retains its `wkb_geometry` column
4. `clip_to_mercator_bounds` and `add_4326_column` hardcode `geom` → crash

## Task 1: Fix backend ogr2ogr to handle pre-existing tables

**files:** `backend/app/ingest/ogr.py`
**action:**
- Add `-overwrite` flag to `run_ogr2ogr()` command — this ensures fresh table creation even if a stale table exists from a failed import
- This is safe because `generate_table_name()` already checks the catalog for collisions; any pre-existing table with the same name is orphaned from a failed ingest

**verify:** grep for `-overwrite` in ogr.py
**done:** ogr2ogr always creates fresh tables with correct geometry column name

## Task 2: Add geometry column rename safety net

**files:** `backend/app/ingest/metadata.py`
**action:**
- Add `ensure_geom_column()` function that:
  1. Queries `geometry_columns` view for the actual geometry column name
  2. If it's not `geom`, renames it via `ALTER TABLE ... RENAME COLUMN`
  3. Updates the spatial index name if needed
- This is a defense-in-depth measure for any edge case where ogr2ogr creates a non-`geom` column

**verify:** function exists and is called from tasks.py
**done:** geometry column is always normalized to `geom` before post-processing

## Task 3: Call ensure_geom_column in ingest pipeline

**files:** `backend/app/ingest/tasks.py`
**action:**
- Import and call `ensure_geom_column()` after `run_ogr2ogr()` and before `clip_to_mercator_bounds()`
- Only call when `has_geometry` is True

**verify:** grep for `ensure_geom_column` in tasks.py
**done:** safety net is wired into the ingest pipeline

## Task 4: Fix seed script null handling and add --clean flag

**files:** `scripts/seed-ago-data.py`
**action:**
- Fix `get_service_layers()`: handle `null` values for `layers`/`tables` keys (ArcGIS returns `null` not missing key)
- Add `--clean` flag to delete failed datasets before re-importing (calls DELETE on existing datasets that match)
- Add `--concurrency` flag (default 3) to control parallelism
- Improve error logging: log the actual ogr2ogr/API error details

**verify:** `python scripts/seed-ago-data.py --help` shows new flags
**done:** script handles edge cases gracefully
