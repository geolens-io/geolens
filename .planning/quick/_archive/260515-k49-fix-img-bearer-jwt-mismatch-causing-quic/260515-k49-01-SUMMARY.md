# Phase 260515-k49 Plan 01: Fix img Bearer-JWT mismatch causing quicklook 404s

**One-liner:** Replace anonymous `<img src>` quicklook requests with authenticated `apiFetchBlob + URL.createObjectURL` blob URLs via a new `useQuicklook` hook, eliminating 3 console 404s for private datasets on Search and Builder.

## What Was Built

### Task 1: useQuicklook hook + quicklook-cache comment + tests

Created `frontend/src/components/maps/hooks/use-quicklook.ts`:
- `useQuicklook(datasetId, size=256): { url, status }` hook
- Routes all fetches through `apiFetchBlob` (Bearer JWT from `useAuthStore` attached automatically)
- Short-circuits to `idle` when `datasetId` is null, `missing` when `isQuicklookKnownMissing` returns true
- TanStack Query with `retry: false`, `staleTime: 5min`, `gcTime: 10min`
- `useEffect` cleanup revokes blob URL on unmount AND on `data` change (handles dataset-id change)
- 404 ApiErrors negative-cached via `markQuicklookMissing`; non-404 errors return `status: 'error'` (not cached)

Updated `frontend/src/lib/quicklook-cache.ts`:
- JSDoc updated to reflect new fetch()-404 semantics (API surface unchanged)

Created `frontend/src/components/maps/hooks/__tests__/use-quicklook.test.ts`:
- 8 tests: null idle, known-missing sync, 200 blob URL, 404 negative-cache, 500 no-cache, unmount revoke, id-change revoke, bearer path proof

All 8 new tests pass. Existing 6 quicklook-cache tests pass unchanged.

### Task 2: Convert SearchResultCard + DatasetSearchPanel

`frontend/src/components/search/SearchResultCard.tsx`:
- Replaced `isQuicklookKnownMissing` / `markQuicklookMissing` import with `useQuicklook`
- Replaced `quicklookId` / `quicklookUrl` computation with `enableQuicklook` + `useQuicklook()`
- `quicklookUrl && quicklookId ?` condition replaced with `quicklookBlobUrl ?`
- Dropped `onError` handler entirely (hook owns 404 handling)

`frontend/src/components/builder/DatasetSearchPanel.tsx`:
- Same import swap, same hook call pattern in `DatasetPreview` component
- Basemap thumbnail `<img>` tags (lines ~357, ~373) left untouched — they reference `basemapThumbnail(entry.id)` which are public static assets, not the authenticated quicklook endpoint

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] vi.mock factory ApiError hoisting ReferenceError**
- **Found during:** Task 1 test authoring
- **Issue:** `vi.mock` factories are hoisted to the top of the file before class declarations, so defining `ApiError` outside the factory and referencing it inside caused `ReferenceError: Cannot access 'ApiError' before initialization`
- **Fix:** Defined `ApiError` class inline inside the `vi.mock('@/api/client', () => { ... })` factory, then imported it from the mocked module in the test body
- **Files modified:** `use-quicklook.test.ts`
- **Commit:** 4a067204

**2. [Rule 2 - Structure] useQuicklook split into inner useQuicklookQuery function**
- **Found during:** Task 1 implementation
- **Issue:** Two early returns before hook calls would violate the React rules of hooks. The plan's inline conditional pattern requires a structural split to avoid invalid conditional hook invocations.
- **Fix:** Outer `useQuicklook` handles the two early-return short-circuits (null id, known-missing); inner `useQuicklookQuery` contains all hook calls. Both are in the same file; public API is unchanged.
- **Files modified:** `use-quicklook.ts`
- **Commit:** 4a067204
