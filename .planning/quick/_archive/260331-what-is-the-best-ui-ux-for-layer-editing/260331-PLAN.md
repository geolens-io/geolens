# Quick Task 260331 Plan 01

## Goal

Assess the current map creator layer-editing experience, identify UX and architecture gaps, and recommend the best future-facing UI model for Mapbox style-spec extensibility without changing product code.

## Tasks

### Task 1

- `files`: `frontend/src/pages/MapBuilderPage.tsx`, `frontend/src/components/builder/LayerPanel.tsx`, `frontend/src/components/builder/LayerItem.tsx`, `frontend/src/hooks/use-builder-layers.ts`
- `action`: Inspect the current layer stack and editor interaction model, including selection, expansion, tabbing, ordering, and sidebar layout behavior.
- `verify`: Findings cite exact implementation points and explain the resulting UX constraints.
- `done`: Completed via code review and line-referenced notes.

### Task 2

- `files`: `frontend/src/components/builder/LayerStyleEditor.tsx`, `frontend/src/components/builder/DataDrivenStyleEditor.tsx`, `frontend/src/components/builder/LayerFilterEditor.tsx`, `frontend/src/components/builder/LabelEditor.tsx`, `frontend/src/lib/layer-capabilities.ts`, `frontend/src/components/builder/layer-adapters/types.ts`, `frontend/src/types/api.ts`
- `action`: Review the current style/filter/label model against Mapbox style-spec concepts and identify extensibility blockers.
- `verify`: Recommendations map current abstractions to style-spec concepts like layer type, filter, paint, layout, minzoom/maxzoom, and slot.
- `done`: Completed via code review plus official style-spec source checks.

### Task 3

- `files`: `.planning/quick/260331-what-is-the-best-ui-ux-for-layer-editing/260331-RESEARCH.md`, `.planning/quick/260331-what-is-the-best-ui-ux-for-layer-editing/260331-SUMMARY.md`, `.planning/quick/260331-what-is-the-best-ui-ux-for-layer-editing/260331-VERIFICATION.md`
- `action`: Review the sample map in Playwright, synthesize findings, and produce a concise report with easy wins and a recommended target UX.
- `verify`: Report includes live-observed issues, concrete suggestions, and a verification status with no code changes made.
