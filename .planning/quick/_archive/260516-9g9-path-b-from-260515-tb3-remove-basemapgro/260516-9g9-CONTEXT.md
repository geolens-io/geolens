### Part 2.5 — Runtime application (Path R — REVISED 2026-05-16 after research)

**REVISED scope:** Path B is now **Path B+R** (user-chosen 2026-05-16 after RESEARCH.md CAVEAT-1 surfaced that the master-opacity slider has ZERO runtime effect today). `basemap-utils.ts::applyBasemapConfigToStyle` reads every BasemapConfig field EXCEPT opacity. Extend `applyBasemapConfigToStyle` (`frontend/src/lib/basemap-utils.ts` lines ~342-354) AND `applyBasemapConfigToMap` (`frontend/src/components/builder/map-sync.ts:216-260`) to apply `opacity` to the basemap layers' MapLibre paint properties. Specifically: for each basemap sublayer (Roads / Buildings / Boundaries / Labels / Land-Water), set `raster-opacity` (for raster basemaps) or the equivalent vector paint property multiplied with the layer's existing intrinsic opacity if any. Planner research must locate the exact `setPaintProperty` call sites + layer-id conventions used by `applyBasemapConfigToStyle`.

### Part 3 — Side-finding cleanup (recommended)

`use-layer-map-sync.ts:52` calls `setHasUnsavedChanges(true)` unconditionally in `applyLayerUpdate`, even when the supplied `layerId` doesn't match any layer. The investigation flagged this as a defensive cleanup opportunity. Include it in this task scope: gate `setHasUnsavedChanges(true)` on whether `updated` is actually set (i.e., a layer was found and updated).

**Goal of this task:**
1. BasemapGroupRow renders no row slider; LayerEditorPanel basemap-editor-scene Visibility-section "Master opacity" slider is the canonical control.
2. Master-opacity edits **visually apply to the basemap in real-time** (Path R — new requirement post-CAVEAT-1).
3. Master-opacity edits persist across save / reload.
4. `applyLayerUpdate` no longer dirties state on non-matching layerIds.

### Out of scope

- LayerEditorPanel default-content opacity slider (untouched in 260515-rdn/sqf, also untouched here — no per-row layer involvement).
- Frontend default-value seeding for non-basemap layers (not affected).
- Migrating existing maps' stored basemap_config to include the new opacity field (additive Pydantic default handles this — old records load with `opacity=1.0` default).
