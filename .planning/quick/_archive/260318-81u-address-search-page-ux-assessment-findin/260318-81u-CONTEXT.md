<decisions>
## Implementation Decisions

### Filter Bar Architecture
- **Secondary row pattern**: Global filters stay in the primary row. Type-specific filters appear in a labeled secondary row below when a record type is selected.
- No type selected → only global row visible
- One type selected → show secondary row labeled e.g. "Raster filters"
- Multiple types selected → hide type-specific filters or show intersection set
- Global filters: Type, Location, Time, Keywords, Collection, Status
- Type-specific: Vector (geometry type, feature count, CRS), Raster (band count, resolution, CRS), VRT (VRT type, source count, band count, CRS)
- Small label like "Raster filters" + collapse/expand affordance if row gets long

### VRT Labeling Strategy
- Context-dependent labeling:
  - **Primary UI labels** (buttons, tabs, search filters, create flows): "Virtual Raster"
  - **First mention / detail pages / docs**: "Virtual Raster (VRT)"
  - **Dense badges or compact tables only**: "VRT"
- Create menu: "Virtual Raster"
- Search type filter: "Virtual Raster"
- Result card badge: "Virtual Raster" (or "VR" only if space is very tight)
- Dataset detail header: "Virtual Raster (VRT)"

### Result Card Type Badges
- Both icon + colored badge, but **subtle/muted** (taxonomy badges, not alerts)
- Soft background tint, darker neutral text/icon, modest border or no border, compact size
- Color mapping: Vector=blue, Raster=green, Virtual Raster=purple, Collection=amber
- Type badge visually stronger than metadata chips, weaker than status badges
- Icons: Vector=layers, Raster=grid, Virtual Raster=stacked/grid, Collection=folder/stack

### Language & Label Fixes
- "Search datasets..." → more inclusive language ("Search geospatial data, collections, and derived products" or similar)
- "X datasets found" → "X records found" or "X results"
- "1 bands" → "1 band" (pluralization fix)
- "Collection (2)" → "Collections (2)" (pluralize tab labels)
- "VRT" → "Virtual Raster" in primary UI contexts
- Fix count inconsistency between tabs total and results count
- "Geometry" filter label → rename or make type-aware

### Claude's Discretion
- Exact icon choices (lucide icons preferred per project conventions)
- Exact color hex values for badge tints (should work in both light and dark mode)
- Whether to add subtype metadata to cards in this pass (e.g. "Polygon", "3-band") or defer
- Exact wording of search placeholder text

</decisions>
