Remove Organization and CRS selects from the primary row — they move to the secondary row. Organization is not in the user's global filter list and CRS is explicitly type-specific per CONTEXT.md.

**Secondary row (conditional, below primary):**
- Only visible when `recordType` is set to exactly one non-empty type (not 'all', not '')
- Wrapped in a `div` with: `flex items-center gap-3 px-1 pt-2 border-t border-border/40` (subtle separator)

- Filter content by selected type:
  - **vector_dataset**: Geometry Type select (existing, moved from primary), Organization select (if orgs available), CRS select (if srids available)
  - **raster_dataset**: Organization select (if orgs available), CRS select (if srids available). Note: band_count and resolution filters have NO backend support, so do NOT add filter controls for them — only show what the API can actually filter on.
  - **vrt_dataset**: Organization select (if orgs available), CRS select (if srids available). Same constraint — no backend support for vrt_type/source_count filters.
  - **collection**: No secondary filters (hide the row entirely for collections)

**Implementation approach:**
1. Extract the existing Geometry Type, Organization, and CRS filter blocks into named helper functions (e.g. `renderGeometryFilter()`, `renderOrganizationFilter()`, `renderCrsFilter()`) if not already — they are already inline JSX blocks

3. Place the secondary row div immediately after the primary row div, still inside the outer fragment
4. For the type label in the secondary row, use: `recordType === 'vector_dataset' ? t('filters.vector') : recordType === 'raster_dataset' ? t('filters.raster') : t('filters.vrt')`
