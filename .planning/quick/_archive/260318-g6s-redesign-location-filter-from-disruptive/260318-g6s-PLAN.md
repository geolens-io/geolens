  <name>Task 1: Create SpatialFilterPanel component</name>
  <files>frontend/src/components/search/SpatialFilterPanel.tsx</files>
  <action>
Create a new `SpatialFilterPanel` component that renders as a fixed right-side panel (NOT using the Sheet/Radix Dialog component, to avoid the modal overlay that blocks results per user decision).

**Panel container:**
- Fixed position: `fixed inset-y-0 right-0 z-40 w-[400px] border-l bg-background shadow-lg`
- Slide animation: `transition-transform duration-300 ease-in-out`, `translate-x-0` when open, `translate-x-full` when closed
- Accept props: `open: boolean`, `onClose: () => void`, `onApply: (bbox: string) => void`, `initialBbox?: string`

**Panel contents (top to bottom):**
1. Header: "Search area" title + close X button (top-right)
2. Mode toggle: `Rectangle | Polygon` using ToggleGroup, default to "rectangle"
3. MapGL instance (~300px tall, rounded corners) with Terra Draw initialized with BOTH `TerraDrawRectangleMode` and `TerraDrawPolygonMode`
4. Contextual instruction text below map:
   - Rectangle mode: "Click and drag to draw a bounding box"
   - Polygon mode: "Click to add points, double-click to finish"
5. "Clear area" button (ghost variant, only visible when a shape is drawn)
6. "Apply" button (default variant, disabled when no shape drawn)

**Terra Draw behavior:**
- Initialize with both modes: `new TerraDrawRectangleMode({...})` and `new TerraDrawPolygonMode({...})` using `MAP_COLORS.default` for styles
- `td.setMode('rectangle')` by default; switch mode on toggle change
- On mode switch: clear any existing drawn feature via `td.removeFeatures()`, then `td.setMode(newMode)`
- On `td.on('finish')`: store the drawn GeoJSON Feature in a React ref (`drawnFeatureRef`). Do NOT remove the feature from Terra Draw (keep it visible on the map). Do NOT auto-apply. Extract bbox from coordinates and store in local state (`pendingBbox`).
- Only allow ONE drawn feature at a time: on new finish event, remove the previous feature before storing the new one
- "Clear area" button: removes feature from Terra Draw canvas and clears `pendingBbox`
- "Apply" button: calls `onApply(pendingBbox)` and `onClose()`

**Geometry preservation across open/close:**
- When panel opens with `initialBbox` and no drawn feature exists: parse the bbox string back to a rectangle polygon and use `td.addFeatures()` to restore it on the map. Store as the current drawn feature.
- When panel closes without Apply, the drawn feature is preserved in the ref for next open.
- On reopen: if `drawnFeatureRef` has a feature, restore it to Terra Draw with `td.addFeatures()`.

**Map setup:**
- Use same basemap pattern as BboxMapPicker: `useTheme()` + `useBasemaps()` + `getThemeBasemap()` + `toMaplibreStyle()`
- Use `onLoad` callback for `map.setTransformRequest()` (per project memory: transformRequest prop is silently ignored in react-maplibre v8)
- `initialViewState={{ longitude: 0, latitude: 20, zoom: 1 }}`
- `attributionControl={false}`

**Imports needed:** `@vis.gl/react-maplibre` Map, `terra-draw` TerraDraw/TerraDrawRectangleMode/TerraDrawPolygonMode, `terra-draw-maplibre-gl-adapter`, `@/lib/map-colors`, `@/components/theme-provider`, `@/hooks/use-settings`, `@/lib/basemap-utils`, `@/components/ui/toggle-group`, lucide-react X icon, `maplibre-gl/dist/maplibre-gl.css`
