# Quick Task 260508-nl9: Seeder + Playwright MCP Smoke Checks

## Context

The previous quick task 260508-lkz rebuilt the demo themes/fixtures (5 maps across 2 themes) and passed verification on code-only must-haves (14/14). The deferred work was the live seeder run + browser smoke check, both of which require an interactive Docker host. This task runs that deferred work end-to-end and produces a pass/fail report.

## Tasks

### Task 1: Bring up fresh stack (destructive)

**Files / state touched:**
- `docker compose` services: db, migrate, api, worker, titiler, frontend, valkey, minio
- Volume `geolens_pgdata` (wiped)

**Action:**
```bash
docker compose -f docker-compose.yml -f docker-compose.demo.yml down -v --remove-orphans
docker compose -f docker-compose.yml -f docker-compose.demo.yml up -d --build api worker frontend titiler db
# Wait for api healthy
for i in {1..60}; do
  status=$(docker compose ps --format json api 2>/dev/null | python3 -c "import json,sys; d=json.loads(sys.stdin.read() or '{}'); print(d.get('Health',''))")
  [ "$status" = "healthy" ] && break
  sleep 2
done
```

**Verify:**
- `docker compose ps` shows api/db/worker/frontend healthy
- `curl -s http://localhost:8000/healthz` returns 200

**Done when:** All four services healthy, frontend reachable on :8080.

### Task 2: Run seeder

**Files / state touched:** Postgres demo schema, seeder image (rebuilt to include new run-seeder.sh + fetch_external.py).

**Action:**
```bash
docker compose -f docker-compose.yml -f docker-compose.demo.yml build seeder
docker compose -f docker-compose.yml -f docker-compose.demo.yml run --rm seeder 2>&1 | tee .planning/quick/260508-nl9-run-seeder-and-playwright-mcp-smoke-chec/seeder.log
```

**Verify:**
- Seeder exit code 0
- seeder.log contains "Theme 1: When the Land Speaks" and "Theme 2: When the Earth Moves"
- seeder.log contains "applied fixture: 1-grand-canyon" through "2-wildfires" (5 fixtures)

**Done when:** seeder.log shows 5 maps applied, exit code 0.

### Task 3: Verify maps via API

**Action:**
```bash
TOKEN=$(curl -s -X POST http://localhost:8000/api/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"admin"}' | python3 -c 'import json,sys;print(json.load(sys.stdin)["access_token"])')
curl -s -H "Authorization: Bearer $TOKEN" http://localhost:8000/api/maps/ | python3 -m json.tool | head -60
```

**Verify:**
- 5 maps in response with names matching DEMO_MAP_NAMES from `e2e/demo-smoke-shared.ts`

**Done when:** API confirms 5 demo maps exist.

### Task 4: Anonymous browser smoke (Playwright MCP)

**Action via mcp__playwright__browser_*:**
1. `browser_navigate` to `http://localhost:8080` (or `/demo` if there's a gallery)
2. For each of 5 maps, click in, wait for canvas, snapshot
3. Capture console errors

**Verify:**
- Each map's `<canvas class="maplibregl-canvas">` is present in snapshot
- 3D maps show non-flat pitch (visible 3D bars / extruded buildings / DEM relief)
- No red console errors

**Done when:** 5 snapshots captured, errors documented.

### Task 5: Admin login + authed map list

**Action via mcp__playwright__browser_*:**
1. Navigate to login (likely `/login`)
2. Fill admin/admin, submit
3. Navigate to map list (likely `/maps`)
4. Snapshot

**Verify:** All 5 demo map names visible in the list.

**Done when:** Authed snapshot captured with all 5 maps.

- This task is **operational/validation only** — no code changes expected.
- If a map fails to render or seeder fails, the failure is documented in SUMMARY.md and (if obvious) a fix follow-up may be filed; we do not fix in this task unless the fix is a one-liner adjacent to a fixture.
