## Document structure:

### 1. Executive Summary
2-3 paragraph overview: the GTM docs are a reasonable framework but have a significant execution gap. Quantify: X of Y community features exist, 0 of Z enterprise features exist.

Table comparing CKAN, GeoNode, MapStore, Terria, GeoServer on:
- License, Revenue Model, Pricing Range, Open-Core Boundary, Target Buyer

### 5. Pricing Benchmark Analysis
Compare GTM proposed pricing ($8K-$200K) against Terria ($3.8K-$13.2K) and CKAN managed ($6K-$20K). Assess whether GeoLens pricing is defensible at each tier. Reference RESEARCH.md findings.

### 6. Year 1 Revenue Projection Reality Check
Evaluate the $50K-$200K Year 1 claim. Factors: zero installed base, no website, no sales pipeline, solo developer. Compare to Terria and CKAN ecosystem revenue patterns. Provide realistic alternative projection.

### 7. Gap Analysis: What Must Be Built Before Paid Tiers
List the minimum viable enterprise features needed to justify each tier:
- Team ($8K-$15K): What's the minimum? (SSO + support is likely enough)
- Business ($25K-$60K): What's the minimum? (RBAC + audit export + governance basics)
- Enterprise ($75K+): What's the minimum? (white-label + air-gapped + federation)

### 8. Repo Architecture Assessment
Evaluate repo-split.md recommendations vs current monorepo state. Is the extension seam pattern already in place? What's needed?

### 9. Recommendations (Prioritized)

- Publish FEATURES.md on a landing page
- Set up a public demo instance
- Create a pricing page (even if "contact us" for paid tiers)
- Write installation quickstart
- Define the free/paid boundary publicly before launch
- Add "Powered by GeoLens" branding that enterprise can remove
- Create a GitHub Discussions or Discord community channel

### 11. Risks and Concerns
Key risks: overbuilding enterprise before community traction, pricing too high for entry, no design partners, consulting dependency trap.
