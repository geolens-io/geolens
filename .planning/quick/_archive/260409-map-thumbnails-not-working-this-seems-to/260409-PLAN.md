<objective>
Fix the map thumbnail regression where builder-generated thumbnails are captured before map layers finish syncing/loading, producing previews that show only the basemap instead of the map content.

Purpose: restore trustworthy map previews on the maps page without widening scope into unrelated thumbnail display code.
Output: thumbnail capture waits for sources + idle, regression test coverage, and browser validation against a duplicated map with no initial thumbnail.
</objective>

<context>
- Playwright reproduction on `http://localhost:8080/maps` showed existing thumbnail requests succeeding, so display/fetch is healthy.
- Duplicating `Global Bathymetry` produced a new map with `thumbnail_url: null`; opening the duplicate triggered `PUT /api/maps/{id}/thumbnail/` before any raster/vector tile requests completed.
- The resulting card preview on `/maps` showed only the basemap, confirming generation timing is wrong.
- `frontend/src/hooks/use-builder-save.ts` currently auto-captures from `handleMapRef`, which can run before `syncLayersToMap(...)` has added GeoLens sources and before tiles render.
</context>
