<objective>
Restructure SearchResultCard from a 2-column grid (content + large preview sidebar) to a 4-band vertical layout (header / facts / tags / footer) with an inline 80x80 preview in the header. This creates stronger visual hierarchy, recovers ~60% of preview space for content, and segments metadata into meaningful rows.

Purpose: Cards currently devote too much area to preview at the expense of metadata hierarchy. The 4-band layout makes scanning faster and metadata more discoverable.
Output: Restructured SearchResultCard.tsx + updated test file.
