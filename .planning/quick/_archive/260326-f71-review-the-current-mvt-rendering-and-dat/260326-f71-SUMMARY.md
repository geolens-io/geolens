## Changes Made

### Fix 1: Bounds CTE precomputation (service.py)
Added `ST_Transform(ST_TileEnvelope(...), 4326) AS geom_4326` to the bounds CTE. Replaced both `ST_Transform(bounds.geom, 4326)` references in the WHERE clause with `bounds.geom_4326`. Guarantees single evaluation regardless of query planner behavior.

### Fix 2: Per-tile logging to DEBUG (router.py)
Changed `logger.info("tile_access", ...)` to `logger.debug(...)`. A single map view generates 20-50 tiles; this eliminates 20-50 INFO entries per view.

### Fix 3: Cache-Control on cache hits (router.py)
Cache-hit responses now return `Cache-Control: {scope}, max-age={ttl}` instead of `no-cache`. The existing `TileCacheProvider` (binary Redis cache at `backend/app/cache/tile_cache.py`) already handles invalidation via `invalidate_table()` on feature edits, so browser caching is safe.

### Fix 4: Empty tile sentinel caching (router.py)
Empty tiles (PostGIS returns NULL) are now cached as `b""` sentinel in Redis. On cache hit, empty sentinel is detected and 204 is returned without querying PostGIS. For sparse datasets where ~60-80% of tiles are empty, this significantly reduces database load.

### Tests
Updated `test_tile_query_uses_correct_params` with bounds.geom_4326 assertions. Added `test_tile_query_single_transform_in_where` verifying no redundant ST_Transform.

### Findings Document
Full review document covering: architecture overview with TileCacheProvider in the pipeline, 8 strengths, 4 implemented fixes, 2 medium-effort recommendations (dataset lookup caching, zoom-dependent simplification).

## Remaining Opportunities

1. **Dataset lookup caching** — short-TTL in-memory cache to avoid 20+ identical ORM queries per map pan
2. **Zoom-dependent geometry simplification** — extend `ST_SimplifyPreserveTopology` to z < 10-12
