# Phase 260328-b42 Plan 01: Squash Migrations Summary

Single-file Alembic migration replaces all 12 incremental migrations using a clean pg_dump of the fully-migrated catalog schema.

## What Was Done

Squashed 12 incremental Alembic migrations (0002-0012) into a single `initial_schema.sql` + `0001_initial_schema.py` for clean public release. The new `initial_schema.sql` is a `pg_dump --schema-only` output from the fully-migrated database, cleaned to remove:
- `SELECT pg_catalog.set_config(...)` (asyncpg incompatibility)
- `SET idle_in_transaction_session_timeout` (not supported in all PG versions)
- `SET transaction_timeout`
- `alembic_version` table CREATE and constraint (Alembic manages this itself)
- `CREATE SCHEMA catalog` changed to `CREATE SCHEMA IF NOT EXISTS catalog`

The `downgrade()` function was replaced with `raise NotImplementedError("Initial migration cannot be reversed")`.
