<objective>
Enable non-spatial CSV ingestion, create a dedicated detail layout for tabular datasets, and implement FK-based related record display.

Purpose: Currently, uploading a CSV without lat/lon columns fails because the ingestion pipeline unconditionally runs spatial operations (clip_to_mercator_bounds, add_4326_column, quicklook generation). Non-spatial datasets also get classified as vector_dataset and rendered with a meaningless map hero. Additionally, there is no way to model or display FK relationships between datasets (e.g., parcels.owner_id -> owners.id). This plan fixes the entire non-spatial path: ingestion, classification, frontend presentation, and FK relationship support.

Output: Working end-to-end CSV table ingestion with data grid as primary view, plus FK-based related records panel on spatial dataset detail pages.
</objective>
