**#1 — `get_user_roles()` uncached (auth/dependencies.py + visibility.py):**
Add a new dependency `get_cached_user_roles` in `backend/app/auth/dependencies.py`:

```python
async def get_cached_user_roles(
    request: Request,
    db: AsyncSession = Depends(get_db),
    user: User | None = Depends(get_optional_user),
) -> set[str]:
    """Return user roles, cached for the lifetime of this request."""
    if user is None:
        return set()
    cached = getattr(request.state, "_user_roles", None)
    if cached is not None:
        return cached
    roles = await get_user_roles(db, user)
    request.state._user_roles = roles
    return roles
```

Update `require_role` and `require_permission` closures to use `get_cached_user_roles` instead of calling `get_user_roles(db, current_user)` directly. The closures already have access to the dependency injection chain — add `request: Request` parameter and use the cached path. Note: service-layer calls that receive `(db, user)` without `request` keep using `get_user_roles` directly — the caching handles the hot path (router-level repeated dependency resolution).
