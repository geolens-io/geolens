  <name>Task 1: Add nginx proxy_intercept_errors for Titiler 404 responses</name>
  <files>frontend/nginx.conf</files>
  <action>
In frontend/nginx.conf, add an internal named location that returns 204 with no body, and configure
the raster-tiles location block to intercept Titiler 404 errors and route them to this location.

1. Add a named location `@empty_tile` BEFORE the `location /api/` block (after the raster-tiles block closes):
   ```
   location @empty_tile {
       return 204;
   }
   ```

2. Inside the existing `location ~ ^/raster-tiles/...` block, add these two directives AFTER the
   `proxy_set_header Host $host;` line (before the proxy_cache directives):
   ```
   proxy_intercept_errors on;
   error_page 404 = @empty_tile;
   ```

This converts Titiler 404 (no-data tile) responses into 204 (no content) at the proxy layer,
preventing MapLibre from ever seeing an HTTP error. The auth_request subrequest is unaffected
because error_page only applies to the proxy_pass response. Only intercept 404 -- do NOT intercept
500s, as those indicate real Titiler failures that should still surface.
  </action>

<task type="auto">
  <name>Task 2: Add error event listener to BuilderMap for tile fetch errors</name>
  <files>frontend/src/components/builder/BuilderMap.tsx</files>
  <action>
In BuilderMap.tsx, add a `map.on('error', ...)` listener inside the `handleLoad` callback, after the
`map.setTransformRequest(...)` call and before `onMapRef?.(map)`.

The error handler should:
1. Check if the error is a tile-related error by testing if the error message contains a managed
   source ID (pattern: `source-`). Import `getSourceId` is already imported from `./map-sync`.
2. If the error matches a managed raster tile source, silently absorb it (return early, no logging).
   This catches residual errors that bypass nginx (dev mode via Vite proxy, or 500s from Titiler).
3. For non-tile errors, let them propagate normally (do nothing, MapLibre handles default behavior).

Implementation pattern (mirror DatasetMap lines 619-631 but simpler -- no threshold/onTileError needed):
```typescript
// Suppress expected raster tile errors (no-data tiles outside extent)
map.on('error', (e: { error: { message?: string; status?: number } }) => {
  const msg = e.error?.message ?? '';
  // Only suppress errors from our managed tile sources
  if (msg.includes('source-') || e.error?.status === 404) {
    return; // Expected no-data tile, suppress
  }
  // Non-tile errors: let MapLibre default handling proceed
  console.warn('[BuilderMap] Map error:', e.error);
});
```

Do NOT add any user-visible error overlay or threshold logic. The builder does not have an error
overlay UX pattern. This handler is purely for console noise reduction.
  </action>
