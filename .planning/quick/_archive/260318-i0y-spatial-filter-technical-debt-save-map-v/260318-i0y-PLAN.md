Address two spatial filter technical debt items: (1) save the panel mini-map viewport across open/close so it doesn't reset to lon=0/lat=20/zoom=1 each time, and (2) add arbitrary geometry support so polygon draws send actual GeoJSON to the backend instead of extracting a lossy bounding box.

Purpose: Polygon spatial filtering currently loses precision by converting to bbox. Users who carefully draw a polygon boundary should get results filtered by that exact shape.
Output: Backend accepts optional `geometry` GeoJSON param, frontend sends polygon GeoJSON for polygon draws and bbox for rectangle draws, mini-map remembers its viewport.
