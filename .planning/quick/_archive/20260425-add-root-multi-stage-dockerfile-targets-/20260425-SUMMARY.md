# Quick Task 20260425 Summary

Task: `Add root multi-stage Dockerfile targets preserving separate api worker frontend runtime images`

## Outcome

Added a repository-root multi-stage `Dockerfile` with named `api`, `worker`, and `frontend` runtime targets. The targets preserve the existing split runtime model: API and worker share a backend base but keep separate entrypoints, commands, healthchecks, labels, and image names; the frontend remains an unprivileged nginx runtime that serves the Vite build and keeps the existing nginx config.

Added a root `.dockerignore` that keeps the root build context small and excludes secrets/local artifacts while still allowing backend, frontend, `scripts/`, and `docker/seeder/` inputs needed by the root Dockerfile and demo seeder.

Updated Compose so `api`, `worker`, and `migrate` build from the root Dockerfile targets, while the base development frontend still uses `frontend/Dockerfile.dev`. The demo overlay now builds the production frontend from the root `frontend` target and maps to nginx's container port `8080`.

Updated publish and published-image verification workflows so `geolens-api`, `geolens-worker`, and `geolens-frontend` remain separate GHCR artifacts from the same root Dockerfile. The README prebuilt image snippet now includes the worker image.

Implementation commit: `cded7cd5`
