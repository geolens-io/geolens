### LOW

- **F.9 git pathspec `'frontend/src/**'` won't glob (lines 892–893).** Git's default pathspec semantics treat `**` as literal characters unless prefixed with `:(glob)` or shell globstar is enabled. `git log -1 --format=%ct -- 'frontend/src/**'` will silently return empty for most repos. The downstream `[ -n "$LATEST_UI_COMMIT" ]` guard prevents an outright crash, so the check no-ops rather than fails — but it means F.9 stale-screenshot detection silently won't compare anything. Fix: `frontend/src/` (trailing slash, no glob) or `':(glob)frontend/src/**'`.

- **F.4 awk range pattern is off-by-one in line count (line 799).** `awk '/^## \[Unreleased\]/,/^## \[[0-9]/'` includes both the `## [Unreleased]` line and the next numbered version's `## [X.Y.Z]` line in its output — so `wc -l` is inflated by 2. The 200-line threshold is loose enough that this doesn't matter in practice, but the comment claims this measures the Unreleased section size and it actually measures Unreleased + 1 line of the next version header. Pedantic.

- **Awesome-list bash uses `$list-owner` placeholder that doesn't expand cleanly (lines 338–339).** `gh repo view "$list-owner/$list"` will expand `$list` to the loop variable and concatenate with `-owner`, producing strings like `awesome-fastapi-owner/awesome-fastapi` which won't resolve on GitHub. The intent is clearly "fill in the real owner" (the comment on the next line acknowledges this), but a copy-pasting auditor would see a confusing 404. Either replace with a comment-only template or use a real owner mapping.

- **F.10 grep for "RBAC" is overbroad (line 925).** Geolens has open-source RBAC at the role level (admin/user) — flagging every mention of `RBAC` as an "untagged enterprise feature" will produce false positives in any docs that mention permissions at all. Tighten to enterprise-specific terms: `\b(SAML|SSO|audit log|multi-org|multi-tenant|enterprise edition)\b`. (Same caveat applies to "audit log" — geolens has audit log functionality in OSS depending on the audit module's edition gating.)

- **F.5 demo-link liveness scans only first 100/200 lines (lines 819, 822).** Many READMEs put demo links in the hero (covered) but also in a footer/links section beyond line 200. If geolens README ever exceeds 200 lines (it likely already does), footer demo links will be missed. Either drop the `head -100` limit or scan the full README.

- **Subagent F output description says "Pass through to Phase 2.5 for synthesis" (line 946) but the Task tool boundary makes that ambiguous.** Phase 2 explicitly says "spawn all 6 subagents" via the Task tool. Subagent outputs return to the parent — not to each other. The wording suggests a hand-off between subagents that doesn't exist in the actual orchestration model. Minor — readers will figure it out from context.
