# Quick Task 260326-a07: Bundle default basemaps for public release with licensing review - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Task Boundary

For public release, which basemaps should come bundled with GeoLens? Consider licensing or any other issues.

</domain>

<decisions>
## Implementation Decisions

### Commercial Use Model
- GeoLens is open-source under MIT/Apache license
- Bundled basemaps must allow redistribution and commercial use by downstream users
- Any provider whose terms restrict commercial use or redistribution is disqualified

### Basemap Providers
- Keep the current 4 defaults: CARTO Positron, CARTO Dark Matter, OpenStreetMap raster, OpenFreeMap Bright
- Review each provider's licensing terms for compatibility with MIT/Apache distribution
- Replace only if a provider's terms are problematic for open-source commercial use

### Mapbox API Key Setting
- Consider adding API-key-gated basemap options (Mapbox, MapTiler, etc.)
- Need admin settings for API keys (similar to existing settings pattern)
- API-key basemaps would only be available when keys are configured
- Research should cover Mapbox and MapTiler licensing/terms for bundling

### Attribution UX
- Use MapLibre's built-in AttributionControl (reads attribution from style/source metadata)
- Industry standard approach, no custom UI needed
- Ensure each basemap entry has proper attribution strings in the source config

</decisions>

<specifics>
## Specific Ideas

- Current basemaps defined in `backend/app/persistent_config.py` lines 430-462
- Attribution already stored per basemap entry in the `attribution` field
- Frontend uses `toMaplibreStyle()` in `frontend/src/lib/basemap-utils.ts` which passes attribution to raster sources
- GL style JSON basemaps (CARTO, OpenFreeMap) carry their own attribution in the style spec

</specifics>
