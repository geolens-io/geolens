# Quick Task 260326-a07: Bundle Default Basemaps for Public Release Summary

Replaced all CARTO basemap defaults and hardcoded references with OpenFreeMap equivalents for MIT-compatible public distribution, with legacy ID mapping for existing saved maps.

## Task Results

| Task | Name | Commit | Status |
|------|------|--------|--------|
| 1 | Replace CARTO defaults with OpenFreeMap in backend and frontend | 6f56baf6 | Done |
| 2 | Update basemap-utils tests and fix remaining CARTO references | a1c3a2d5 | Done |

## What Changed

### Backend (`persistent_config.py`)
- `carto-positron` replaced with `openfreemap-positron` (URL: `tiles.openfreemap.org/styles/positron`)
- `carto-dark-matter` replaced with `openfreemap-dark` (URL: `tiles.openfreemap.org/styles/dark`)
- OpenStreetMap and Stamen Terrain defaults unchanged

### Frontend (`basemap-utils.ts`)
- `LIGHT_PRESET_ID` changed from `carto-positron` to `openfreemap-positron`
- `DARK_PRESET_ID` changed from `carto-dark-matter` to `openfreemap-dark`
- `LEGACY_KEY_MAP` expanded to resolve all 5 old keys: `positron`, `dark-matter`, `voyager`, `carto-positron`, `carto-dark-matter`
- `toMaplibreStyle` updated to treat `/styles/` URLs as GL style endpoints (OpenFreeMap URLs don't end in `.json`)

### Map Components
- BboxMapPicker, DatasetMap, BuilderMap, ViewerMap: hardcoded CARTO fallback URLs replaced
- BasemapPicker: thumbnail keys updated from `carto-*` to `openfreemap-*`
- MapBuilderPage: default basemap state updated

### Tests (`basemap-utils.test.ts`)
- 20 tests covering: resolveBasemapId (7), getThemeBasemap (3), findBasemapById (5), toMaplibreStyle (3), preset IDs (2)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] toMaplibreStyle did not handle /styles/ URLs**
- **Found during:** Task 2
- **Issue:** OpenFreeMap URLs (`/styles/positron`) don't end in `.json`, so they were incorrectly wrapped as raster tile sources instead of being passed through as GL style URLs
- **Fix:** Added `/styles/` URL detection to `toMaplibreStyle`
- **Files modified:** `frontend/src/lib/basemap-utils.ts`
- **Commit:** a1c3a2d5

**2. [Rule 1 - Bug] Hardcoded CARTO fallback URLs in map components**
- **Found during:** Task 2
- **Issue:** BboxMapPicker, DatasetMap, BuilderMap, ViewerMap all had hardcoded `basemaps.cartocdn.com` fallback URLs that would bypass the settings system
- **Fix:** Replaced all with OpenFreeMap equivalents
- **Files modified:** 4 map components
- **Commit:** a1c3a2d5

**3. [Rule 1 - Bug] BasemapPicker thumbnail keys and MapBuilderPage default**
- **Found during:** Task 2
- **Issue:** Thumbnail lookup used old `carto-*` keys (no match for new IDs), MapBuilderPage defaulted to `carto-positron`
- **Fix:** Updated keys to `openfreemap-*`
- **Files modified:** `BasemapPicker.tsx`, `MapBuilderPage.tsx`
- **Commit:** a1c3a2d5
