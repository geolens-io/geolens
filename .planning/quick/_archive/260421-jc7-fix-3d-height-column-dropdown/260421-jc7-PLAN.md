  <name>Task 1: Replace GeoFabrik download with NYC Open Data building footprints</name>
  <files>docker/seeder/Dockerfile</files>
  <action>
Replace lines 197-209 in docker/seeder/Dockerfile (the GeoFabrik Manhattan buildings block) with a new block that:

1. Downloads Manhattan buildings from NYC Open Data Socrata API as GeoJSON:
   ```
   curl -fsSL -o /tmp/manhattan_buildings_raw.geojson \
     "https://data.cityofnewyork.us/resource/5zhs-2jue.geojson?\$where=within_box(the_geom,40.80,-74.05,40.68,-73.90)&\$limit=50000"
   ```
   NOTE: Escape `$where` and `$limit` as `\$where` and `\$limit` in the Dockerfile RUN directive.

2. Use ogr2ogr to select relevant columns and convert HEIGHT_ROOF (feet) to meters as `height`:
   ```
   ogr2ogr -f GeoJSON \
     -lco COORDINATE_PRECISION=6 \
     $DATA_DIR/manhattan_buildings.geojson \
     /tmp/manhattan_buildings_raw.geojson \
     -sql "SELECT name, height_roof, ROUND(height_roof * 0.3048, 1) AS height FROM manhattan_buildings_raw" \
     -dialect SQLite
   ```
   - `-lco COORDINATE_PRECISION=6` reduces file size by limiting coordinate decimals
   - `-dialect SQLite` is required for arithmetic expressions
   - The table name `manhattan_buildings_raw` matches the input filename stem (GDAL convention)
   - Do NOT use COALESCE/NULLIF for null heights -- let MapLibre coalesce expression handle zeros at render time (preserves data fidelity)

3. Clean up temp file: `rm -f /tmp/manhattan_buildings_raw.geojson`

Update the comment block above the RUN to describe the NYC Open Data source instead of GeoFabrik. Mention dataset ID 5zhs-2jue, that HEIGHT_ROOF is in US feet, and the conversion factor.
  </action>
