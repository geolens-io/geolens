    **Step 2: Create error boundary components.**

    All boundaries are class components (React error boundaries require `componentDidCatch`/`getDerivedStateFromError`). Use `withTranslation` HOC from react-i18next for i18n in class components, OR use a functional wrapper that passes `t()` as a prop. Prefer the functional wrapper pattern for simplicity:

    1. **`AppErrorBoundary.tsx`** — Class component error boundary.
       - `getDerivedStateFromError` sets `hasError: true`, stores error.
       - Fallback UI: full-screen centered card using the existing `ErrorState` component pattern (AlertCircle icon, destructive border/bg). Shows `t('errorBoundary.appTitle')`, `t('errorBoundary.appMessage')`, and a "Reload page" button that calls `window.location.reload()`.
       - Since this wraps ABOVE the i18n provider, use a hardcoded English fallback with `useTranslation` inside a functional fallback child component. The boundary itself is a class; its `render()` when errored renders `<AppErrorFallback error={this.state.error} />` which is a functional component that can call `useTranslation('common')` with a try/catch fallback to hardcoded strings.
       - Logs error to `console.error` in `componentDidCatch`.

    2. **`MapErrorBoundary.tsx`** — Class component error boundary specifically for map rendering.
       - Props: `children`, `hasUnsavedChanges?: boolean` (for builder context).
       - Fallback: map-placeholder div with gray background, centered error icon (MapPin or AlertCircle), title `t('errorBoundary.mapTitle')`, message `t('errorBoundary.mapMessage')`. If `hasUnsavedChanges` is true, also show `t('errorBoundary.builderUnsavedWarning')`.
       - "Reload map" button that resets error state (`this.setState({ hasError: false })`) so React re-mounts the map.
       - Use a functional `MapErrorFallback` inner component for hooks access.

    3. **`RouteErrorBoundary.tsx`** — This is NOT a class component. It's a functional component designed for react-router's `errorElement` prop. Uses `useRouteError()` from react-router.
       - Renders inside AppLayout so has access to all providers.
       - Shows `ErrorState` with title from `t('errorBoundary.routeTitle')`, error message, and two action buttons: "Go back" (`navigate(-1)`) and "Go to home" (`navigate('/')`) using `useNavigate`.

    4. **`LazyLoadErrorBoundary.tsx`** — Class component error boundary.
       - Detects chunk load errors: check if `error.name === 'ChunkLoadError'` or `error.message?.includes('Loading chunk')` or `error.message?.includes('dynamically imported module')`.
       - On chunk load error: auto-retry once by calling `this.setState({ hasError: false })` after a 1s timeout (store retry count in state, max 1 auto-retry).
       - If auto-retry fails or non-chunk error: show fallback with `t('errorBoundary.lazyTitle')`, `t('errorBoundary.lazyMessage')`, and a "Try again" button that resets state. Show "Retrying..." during auto-retry.
       - Use functional `LazyLoadFallback` inner component for hooks.

    5. **`index.ts`** — barrel export all 4 components.
