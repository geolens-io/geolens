# Quick Task 260328-g46: Widget Ecosystem for Map Creation - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Task Boundary

Explore how we can implement a widget ecosystem for map creation — a modular framework for adding interactive widgets (query/results, measurement, attribute table, data viz, etc.) to maps.

</domain>

<decisions>
## Implementation Decisions

### Widget Scope & Types
- General-purpose modular widget framework, not limited to a fixed set
- Initial target widgets: layer query + results display, measurement tool, attribute table, data visualization (charts/stats)
- Framework should be extensible enough that adding new widget types is straightforward

### Composition Model
- Fixed slot positions (top-left, bottom-right, sidebar, etc.)
- Predefined layout zones — no drag-and-drop or floating panel logic needed
- Simple, predictable placement without z-index/overlap management

### Extensibility Boundary
- Dev-only: widgets are React components registered in code
- Type-safe widget API with clean registration pattern
- Ships with the app — no dynamic loading, no plugin marketplace, no admin config UI

### Claude's Discretion
- None — all areas discussed

</decisions>

<specifics>
## Specific Ideas

- Query a layer and see results as a widget
- Measurement widget for distances/areas
- Attribute table widget for browsing feature data
- Data viz widgets (charts, stats panels)
- Modular enough that new widgets follow a consistent pattern

</specifics>
