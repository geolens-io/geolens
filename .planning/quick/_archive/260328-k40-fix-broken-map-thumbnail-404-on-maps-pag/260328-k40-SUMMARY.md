# Phase 260328-k40 Plan 01: Fix Broken Map Thumbnail 404 Summary

**One-liner:** Added `onError` + `imgError` state fallback to MapCard and MapCardGrid so 404 thumbnails gracefully show the MapIcon placeholder instead of a broken image icon.

## What Was Built

Both `MapCard` (list view) and `MapCardGrid` (grid view) on the `/maps` page now handle 404 thumbnail responses gracefully:

- Added `useState(false)` (`imgError`) to track load failure
- Changed thumbnail conditional from `map.thumbnail ?` to `map.thumbnail && !imgError ?`
- Added `onError={() => setImgError(true)}` to both `<img>` elements
- When the browser fires the error event (e.g., 404 from storage), the component re-renders showing the `<MapIcon>` placeholder instead of a broken image icon

Test coverage added in `MapCard.test.tsx`:
- MapCard renders img when thumbnail is present
- MapCard renders placeholder when thumbnail is null
- MapCard renders placeholder after img onError fires
- MapCardGrid renders img when thumbnail is present
- MapCardGrid renders placeholder after img onError fires

All 5 tests pass.
