**1. Create `backend/app/modules/catalog/sources/adapters/ogcapi.py`** following the wfs.py adapter pattern exactly:

`probe_ogcapi(url, client, token=None) -> dict | None`:
- Send `GET {url}` with `Accept: application/json` header (+ Bearer token if provided). Wrap in try/except for httpx errors, return None on failure.
- Parse JSON response. Check for `conformsTo` array at top level.
- If no `conformsTo`, look in `links` for `rel: "conformance"`. If found, fetch that href (use `urljoin(url, href)` for relative URLs) and check its `conformsTo`. Also check for `rel: "data"` link as secondary signal.
- Validate OGC API Features: check if any URI in `conformsTo` contains `"ogcapi-features"`. If neither conformsTo match nor `data` rel link, return None.
- Fetch `{url.rstrip("/")}/collections` with same headers. Parse `collections` array.
- Build layers list: `{"name": c["id"], "title": c.get("title", c["id"]), "crs": None}` for each collection.
- Return `{"service_type": "OGC API Features", "layers": layers}`.
- Use structlog logger, not stdlib logging (match probe.py convention).

`enrich_ogcapi_layers(url, layers, client, token=None) -> list[dict]`:
- Clone the `enrich_wfs_layers` pattern exactly: `asyncio.Semaphore(5)`, `_enrich_one` inner async, `asyncio.gather`.
- Run `ogrinfo -json -so OAPIF:{url} {layer_name}` per layer.
- Set `GDAL_HTTP_HEADERS` env var when token provided (same as WFS).
- Parse geometry_type from `geometryFields[0].type`, feature_count from `featureCount`.
- On failure, return `{**layer, "geometry_type": None, "feature_count": None}`.
- Use 30-second timeout per layer (same as WFS).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User URL -> probe_ogcapi | Untrusted URL supplied by user; fetched server-side |
| External OGC API -> landing page/collections parse | Untrusted JSON from external service |
| External OGC API -> GDAL subprocess | OAPIF URL passed to ogrinfo/ogr2ogr |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-d1g-01 | Spoofing | probe_ogcapi | mitigate | SSRF protection already enforced by existing probe router middleware (url_is_safe check in router.py); OGC probe inherits same path |
| T-d1g-02 | Tampering | OGC API JSON response | accept | Malformed JSON causes probe to return None (detection fails gracefully); no SQL injection vector since collections are passed as subprocess args, not SQL |
| T-d1g-03 | Information Disclosure | GDAL_HTTP_HEADERS | mitigate | Bearer token only set in subprocess env (not logged); same pattern as existing WFS token handling |
| T-d1g-04 | Denial of Service | enrich_ogcapi_layers | mitigate | Semaphore(5) concurrency limit + 30s per-layer timeout; same guards as WFS enrichment |
</threat_model>
