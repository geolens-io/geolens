# Preview Improvements - Research

**Researched:** 2026-03-18
**Domain:** Frontend preview rendering, backend quicklook generation
**Confidence:** HIGH

## Summary

The preview system has two distinct paths: raster/VRT datasets fetch a server-generated quicklook PNG, while vector datasets and collections render a client-side SVG bounding box. The "black box" problem is a real rendering issue in the quicklook generator -- single-band rasters with flat/low-value data produce black thumbnails due to the percentile stretch algorithm. The fallback system is minimal: on quicklook fetch failure, it silently falls back to BBoxPreview with no label explaining why.

**Primary recommendation:** Add a labeled fallback state for failed/unavailable quicklooks, improve quicklook contrast for edge cases, and add type-specific icons to the fallback view.

## Backend Quicklook System

### Endpoint

**File:** `backend/app/datasets/router.py` (lines 611-665)

```
GET /api/datasets/{dataset_id}/quicklook?size=256|512
```

- Only works for `raster_dataset` and `vrt_dataset` record types
- Returns 400 for vector datasets
- Returns 404 if raster asset not found or quicklook URI is null
- Serves stored PNG from managed storage with `Cache-Control: public, max-age=3600`
- Reads pre-generated quicklook from `RasterAsset.quicklook_256_uri` or `quicklook_512_uri`

### Quicklook Generation

**File:** `backend/app/raster/quicklook.py`

`generate_quicklook(cog_path, size)` -- called during ingest (not on-demand):

**Multi-band (>=3 bands):**
- Reads bands 1-3 as RGB
- Crops to valid-data extent (removes nodata borders)
- For non-uint8 data: per-band 2nd/98th percentile stretch
- Result: RGB thumbnail

**Single-band:**
- Reads single band as float64
- Masks nodata pixels
- Crops to valid-data extent
- 2nd/98th percentile stretch to 0-255 grayscale
- **BLACK BOX CASES:**
  - `valid.size == 0` or `np.all(mask)` -> pure black image (line 129-131)
  - `p98 == p2` (flat/constant values) -> pure black image (line 137-139)
  - Very low contrast data -> near-black stretched image

**Canvas:** Aspect-preserving thumbnail pasted on black square canvas with letterboxing.

### Storage Pattern

**File:** `backend/app/ingest/tasks.py`

Quicklooks are generated at ingest time (step 8) and stored at:
- `rasters/{dataset_id}/{sha256}/quicklook_256.png`
- `rasters/{dataset_id}/{sha256}/quicklook_512.png`

For VRT datasets, quicklook generation is non-fatal (wrapped in try/except).

## Problems Identified

### 1. Black Box Problem (HIGH confidence)

Root cause in `generate_quicklook()`:
- Single-band rasters with constant or very narrow value range get `p98 == p2`, producing `np.zeros` (black)
- Single-band rasters with all-nodata produce black
- Letterboxing on black canvas means even valid thumbnails can appear mostly black for narrow/tall rasters

**Fix options:**
- Add minimum contrast: if stretched range is very narrow, use a different colormap (e.g., viridis)
- Convert single-band grayscale to a colormap (terrain, viridis) for visual differentiation
- Detect "effectively black" quicklooks (mean pixel value < threshold) and mark them as unavailable
- Frontend: detect dark images via canvas pixel sampling and show labeled fallback

### 2. No Labeled Fallback State

Current behavior when quicklook fails:
- `imgError = true` -> renders `<BBoxPreview>` silently
- No indication to user that this IS a fallback, not the actual preview
- No type-specific messaging ("Raster preview unavailable", "Generating...")

**Fix:** When `hasQuicklook && imgError`, show a labeled state:
```tsx
<div className="flex flex-col items-center justify-center h-full text-muted-foreground">
  <ImageOff className="h-6 w-6 mb-1" />
  <span className="text-xs">Preview unavailable</span>
</div>
```

### 3. No Loading State for Quicklook Fetch

Current: while quicklook fetches, `quicklookSrc` is null and `imgError` is false, so BBoxPreview shows. This creates a flash: BBoxPreview -> quicklook image.

**Fix:** Add loading state:
```tsx
const [loading, setLoading] = useState(false);
// Show skeleton/spinner while loading
```

### 4. No Preview Status in Backend

There is no `quicklook_status` field on `RasterAsset`. The only way to know if a quicklook exists is checking if `quicklook_256_uri` is non-null. There is no "generating" state tracked.

### 5. Vector Datasets Have No Preview

Vector datasets always show BBoxPreview. No geometry thumbnail is generated. This is acceptable but could be labeled better.

## Recommended Changes

### Frontend (DatasetCard.tsx)

| Change | Description |
|--------|-------------|
| Loading state | Track `isLoading` state, show subtle spinner/skeleton in preview panel during fetch |
| Error fallback | When `hasQuicklook && imgError`: show `ImageOff` icon + "Preview unavailable" label instead of BBoxPreview |
| Vector label | For vector datasets, BBoxPreview is fine but could add small "Extent" label |
| Dark image detection | Optional: after loading quicklook, check if image is predominantly dark via canvas sampling |

### Backend (quicklook.py)

| Change | Description |
|--------|-------------|
| Colormap for single-band | Use matplotlib/pillow colormap (viridis) instead of raw grayscale for single-band rasters |
| Flat-value handling | For `p98 == p2`, use mid-gray (128) instead of black (0) |
| White canvas | Use white/transparent canvas instead of black for letterboxing |

### Current Fallback Logic (DatasetCard.tsx L186-194)

### Proposed Three-State Preview
```tsx
{hasQuicklook && !imgError && quicklookSrc ? (
  <img src={quicklookSrc} alt={...} className="w-full h-full object-cover rounded-md" />
) : hasQuicklook && imgError ? (
  <div className="flex flex-col items-center justify-center h-[120px] w-full bg-muted/30 rounded-md text-muted-foreground">
    <ImageOff className="h-5 w-5 mb-1 opacity-50" />
    <span className="text-xs">{t('card.previewUnavailable')}</span>
  </div>
) : hasQuicklook && !quicklookSrc ? (
  <div className="flex items-center justify-center h-[120px] w-full bg-muted/30 rounded-md">
    <Loader2 className="h-4 w-4 animate-spin text-muted-foreground" />
  </div>
) : (
  <BBoxPreview bbox={bbox} />
)}
```

### Improved Single-Band Quicklook (quicklook.py)
```python
# Instead of grayscale, apply a colormap for visual clarity
from PIL import ImageOps
# After percentile stretch to 0-255:
img_gray = Image.fromarray(stretched, mode="L")
# Apply a simple colormap: convert L -> RGB via palette
img = ImageOps.colorize(img_gray, black="black", white="white", mid="#4477aa")
```
