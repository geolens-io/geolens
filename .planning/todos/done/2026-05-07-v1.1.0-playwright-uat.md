## Infra fixes applied during stack bring-up

The v13.12/v13.13 hardening sweep introduced four latent bugs that
blocked `docker compose up -d --build` from succeeding on a fresh
volume. All four fixed inline so the UAT could proceed; rolled into the
v1.1.0 commit:

1. **migrate service** missing tmpfs mount for `/app/staging` —
   shared `api-entrypoint.sh` probes the dir writable before alembic
   runs, but `read_only: true` (Phase 272) blocked it. Added
   `/app/staging:size=8m,uid=1001,gid=1001` to migrate's tmpfs list.
2. **migrate command** missing `--no-dev` — `uv run alembic` (no
   `--no-dev`) tries to install dev deps to the small home tmpfs and
   ENOSPCs. Mirrored the entrypoint default: `uv run --no-dev alembic`.
3. **api command** YAML folded scalar (`>`) preserved newlines on
   more-indented continuation lines — sh saw a multi-line script and
   ran `uvicorn` (defaulting to 127.0.0.1) on line 1 then errored on
   line 2 (`--host: not found`). Frontend container couldn't reach api.
   Aligned all continuation lines to the same indent so YAML folds them
   into a single space, plus added `--no-dev` symmetric to migrate.
4. **frontend Dockerfile.dev** `npm ci` runs as root, leaving
   `/app/node_modules` root-owned; the anonymous volume copies that
   ownership and vite (running as `node` uid 1000) can't write its dep
   optimizer cache. Added `chown -R node:node /app/node_modules` to
   the same `RUN` step.
5. **frontend memory cap** 128M was the floor in compose; vite + esbuild
   bundling OOM-kills inside that budget. Bumped to 512M.

## 2. CODE-05 — 4-flow store relocation

**Context:** Phase 276-05 moved `drawing-store`, `map-widget-store`, and
`search-store` from `components/<feature>/` to `src/stores/`. Auth store
gained `version: 1` + `migrate`. Source-level smoke test (9 cases)
proves all 4 stores resolve from `@/stores/` and actions still mutate
state. Visual UAT confirms no regression in the user flows.

```
1. Login + auth-store persist:
   - Sign in as admin/admin
   - Open DevTools → Application → Local Storage → http://localhost:8080
   - Confirm the `geolens-auth` blob contains `"version":1`
   - Refresh the page; auth survives.
2. Map builder + map-widget-store:
   - Open /maps/<id>
   - Toggle a widget on/off
   - Add a layer
   - Click Save — no console errors.
3. Dataset map + drawing-store:
   - Open a vector dataset detail page
   - Activate the drawing toolbar
   - Draw a polygon (3-4 vertices, double-click to close)
   - Polygon renders; toolbar transitions to "drawn" state.
4. Search + search-store:
   - At /search, type a query
   - Apply a facet filter
   - Save the search
   - Reload — saved search persists.
```

Pass criterion: all 4 flows complete without console errors or visual
regressions vs. pre-v13.13 baseline.
