<objective>
Drop the four internal `await session.commit()` calls from phase-2 helpers in `backend/app/processing/ingest/metadata.py` (P2-02 from `INGEST-AUDIT-2026-05-21.md`). Let `_finalize_ingest` in `backend/app/processing/ingest/tasks_common.py:821` own the phase-2 commit boundary as the audit prescribes. Add a regression test asserting a phase-2 failure correctly rolls back the column-add change.

Purpose: Eliminate the forward-only commit hazard documented as P2-02. Today, if `add_4326_column` commits successfully but a downstream phase-2 step (e.g. `extract_metadata`, `create_dataset`) fails, the `session.rollback()` cannot undo the column-add — the rename/column-add is already durable on disk. This is currently low-impact in production because the operations are idempotent, but it subverts the documented "phase-2 commits once at `_finalize_ingest`" invariant and traps future maintainers.

Output:
- 4 internal `await session.commit()` calls removed from `metadata.py` (`ensure_geom_column:814`, `clip_to_mercator_bounds:1029`, `add_4326_column:1076`, `grant_reader_access:1091`)
