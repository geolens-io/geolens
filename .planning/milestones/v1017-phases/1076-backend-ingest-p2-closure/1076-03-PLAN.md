<objective>
Restrict the worker exports temp-dir sweep at `backend/app/platform/jobs/worker.py:174-185` to entries older than 1 hour via `stat.st_mtime`; log skipped items so operators can correlate ingest interruptions with worker restarts (ING-04 / P2-04 from `INGEST-AUDIT-2026-05-21.md`).

Purpose: Avoid truncating in-flight large exports on worker restart. Today, every worker startup wipes the entire `<staging>/exports/` directory unconditionally. A rolling deploy or worker OOM-kill during an active 10-minute COG export currently corrupts the download mid-stream. With the mtime guard, exports younger than 1 hour survive worker restarts.

    ```python
        # Sweep orphaned export temp dirs from previous crashes.
        # ING-04 (P2-04): only delete entries older than EXPORTS_SWEEP_AGE_SECONDS
        # (1 hour). In-flight exports started shortly before a rolling worker
        # restart survive the sweep — a 10-minute COG export is no longer
        # truncated mid-download because the new worker process happened to
        # boot during the export's stream phase.
        import shutil

        exports_dir = Path(settings.upload_staging_dir) / "exports"
        if exports_dir.exists():
            entries = list(exports_dir.iterdir())
            if entries:
                now_ts = datetime.now(timezone.utc).timestamp()
                deleted_count = 0
                skipped_count = 0
                for item in entries:
                    try:
                        item_mtime = item.stat().st_mtime
                    except FileNotFoundError:
                        # Raced with another worker / external cleanup — treat as already-gone.
                        continue
                    age_seconds = now_ts - item_mtime
                    if age_seconds < EXPORTS_SWEEP_AGE_SECONDS:
                        log.info(
                            "sweep_skipped_recent_export",
                            path=str(item),
                            age_seconds=round(age_seconds, 1),
                            threshold_seconds=EXPORTS_SWEEP_AGE_SECONDS,
                        )
                        skipped_count += 1
                        continue
                    if item.is_dir():
                        shutil.rmtree(item, ignore_errors=True)
                    else:
                        item.unlink(missing_ok=True)
                    deleted_count += 1
                if deleted_count or skipped_count:
                    log.info(
                        "exports_sweep_complete",
                        deleted=deleted_count,
                        skipped=skipped_count,
                    )
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Worker process → local filesystem (`<staging>/exports/`) | Worker has full read/write to the staging volume; sweep is a destructive housekeeping op |
| Worker restart → in-flight export streams | THIS PLAN'S BOUNDARY — protect in-flight downloads from worker restart |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1076-09 | Denial of Service | In-flight export truncated by worker restart | mitigate | mtime guard preserves entries younger than 1 hour, defaulting to the same window as the worker stale-job recovery (`JOB_TIMEOUT_SECONDS = 1h` per `recover_stale_jobs` docstring at worker.py:74) |
| T-1076-10 | Tampering | Adversary touches export-temp file to extend its lifetime | accept | Sweep runs in worker process with the same fs privileges as the export-writer; an adversary who can mutate `<staging>/exports/*` already has write access to user uploads — this is not a new attack surface |
| T-1076-11 | Information Disclosure | Recent exports linger longer on disk | accept | Exports are tied to download jobs that already encode ACLs in the download token (P0-01); a longer disk lifetime does not expose data to unauthorized callers; clean-up happens on the next sweep after 1h |
| T-1076-12 | Repudiation | Sweep deletions are now selectively logged | mitigate | `exports_sweep_complete` log event captures deleted + skipped counts; per-skipped-entry `sweep_skipped_recent_export` log lines enable forensic correlation when an in-flight export does end up corrupted |
| T-1076-SC | Tampering | npm/pip/cargo installs | accept | Pure code refactor + test addition; no new deps. |
</threat_model>
