    ```python
    def _is_lock_timeout_error(exc: BaseException) -> bool:
        """Detect PostgreSQL lock_timeout (SQLSTATE 55P03) across asyncpg + SQLAlchemy wrapping.

        asyncpg raises `LockNotAvailableError`; SQLAlchemy wraps it in
        `DBAPIError` with `.orig` pointing at the original asyncpg exception.
        Check both shapes so behavior is identical regardless of where the
        exception bubbles up from.
        """
        # Direct asyncpg exception
        try:
            from asyncpg.exceptions import LockNotAvailableError

            if isinstance(exc, LockNotAvailableError):
                return True
        except ImportError:
            pass

        # SQLAlchemy-wrapped: check the underlying .orig for SQLSTATE 55P03
        orig = getattr(exc, "orig", None)
        if orig is not None:
            sqlstate = getattr(orig, "sqlstate", None)
            if sqlstate == "55P03":
                return True

        return False
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Reupload worker → PostgreSQL DDL | Swap renames two tables and drops one — has elevated privileges within the data schema |
| Lock contention → user-visible failure | THIS PLAN'S BOUNDARY — translate transient contention into a single retry that ops can observe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1076-13 | Denial of Service | Reupload swap fails late, after staging-load work is sunk | mitigate | Single retry with 3x longer lock_timeout window after a 200ms autovacuum-clearing sleep; bounded latency overhead (max 15.2s additional in worst case) |
| T-1076-14 | Tampering | Adversary triggers autovacuum to deny reupload service | accept | Autovacuum is internal PG bookkeeping; an adversary cannot trigger it on demand at user-visible cadence; the retry mitigates the natural collision, not an attack surface |
| T-1076-15 | Information Disclosure | Structured log emits dataset_id + table_name | accept | Log fields are operational metadata only — no PII, no row data; structlog already masks secrets via its processor chain |
| T-1076-16 | Repudiation | Contention events untracked for ops correlation | mitigate | `reupload_swap_lock_contention` WARNING + `reupload_swap_retry_succeeded` INFO give ops the data points needed to correlate with autovacuum schedules and tune `autovacuum_vacuum_cost_delay` if contention is recurring |
| T-1076-17 | Elevation of Privilege | Retry path bypasses any RBAC check | accept | Retry runs inside the same session/transaction that originally passed the upstream RBAC checks; no new permission gates are crossed |
| T-1076-SC | Tampering | npm/pip/cargo installs | accept | No new dependencies — uses `asyncpg.exceptions.LockNotAvailableError` (already a transitive dep via SQLAlchemy + asyncpg) and `asyncio.sleep` (stdlib). |
