# Plan 1078-01: CI Alembic Clean-DB Upgrade Workflow

## Threat Model

| Threat | Mitigation |
|--------|------------|
| Migration breaks against clean DB but tests-on-existing-DB pass; goes undetected until prod rollout | This job catches it pre-merge |
| Workflow YAML error blocks all CI | Verify YAML before commit; the new job is additive — existing jobs untouched |
| Job runs on every PR even when no migration-relevant files changed | Path filter ensures the job only fires on relevant changes (push to main is the catch-all) |
| Docker build cache miss on first run takes ~2-3 min | Accepted — see CONTEXT.md "Deferred Ideas"; revisit if CI becomes a bottleneck |

## Tasks

<task id="1">
<action>
Edit `.github/workflows/ci.yml`:

1. Add a new output to the `changes` job for the `alembic` filter:
   ```yaml
   outputs:
     # ... existing outputs ...
     alembic: ${{ steps.filter.outputs.alembic }}
   ```

2. Add the `alembic:` filter block under `filters:`:
   ```yaml
   alembic:
     - 'backend/alembic/**'
     - 'backend/scripts/test_alembic_upgrade_clean_db.sh'
     - 'backend/app/models/**'
     - 'db/**'
   ```

3. Add a new job after `backend-tests` (or wherever makes sense in the file's flow — backend jobs are grouped together; insert at the end of the backend section):

   ```yaml
   # ---------- alembic clean-DB upgrade smoke ----------
   alembic-clean-db:
     name: Alembic Clean-DB Upgrade
     needs: changes
     if: needs.changes.outputs.alembic == 'true' || github.event_name == 'push'
     runs-on: ubuntu-latest
     timeout-minutes: 15
     steps:
       - uses: actions/checkout@v6

       - uses: astral-sh/setup-uv@v8.1.0
         with:
           version: "0.10.2"
           enable-cache: true
           cache-dependency-glob: "backend/uv.lock"

       - uses: actions/setup-python@v6
         with:
           python-version: "3.13"

       - name: Install backend dependencies
         working-directory: backend
         run: uv sync --locked

       - name: Run alembic clean-DB upgrade smoke
         run: ./backend/scripts/test_alembic_upgrade_clean_db.sh
         env:
           # The script reads these for the throwaway container; explicit
           # is better than relying on bash defaults.
           ALEMBIC_TEST_DB_PORT: 54399
           ALEMBIC_TEST_TIMEOUT: 90
   ```
</action>
