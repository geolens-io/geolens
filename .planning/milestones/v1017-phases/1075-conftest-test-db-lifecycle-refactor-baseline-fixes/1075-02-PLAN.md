<root_cause_hypothesis>
The 3 failures in this file are LIKELY caused by one of:

1. **Module-collection cascade from `InvalidCatalogNameError`** (PRIMARY HYPOTHESIS). The conftest `_test_db_lifecycle` fixture is `autouse=True, scope="session"` — every test file inherits it. If the session fixture fails mid-setup (e.g., because the alembic migration referenced a column that didn't exist yet), every test in this file enters an ERROR state at collection, not at execution. **If Plan 01's conftest refactor lands first (Wave 1), these tests likely auto-resolve.** Run them first, see what happens.

2. **Mock signature drift.** The tests mock `reupload_commit`, `add_vrt_source`, `remove_vrt_source`, `regenerate_vrt_endpoint` with specific positional argument shapes. If a downstream v1015 / v1016 phase added a new required parameter (e.g., a `request: Request` arg for rate-limiting, or a new keyword-only `idempotency_key`), the test invocations would crash with `TypeError`. Visible in pytest output as `TypeError: reupload_commit() got an unexpected keyword argument 'X'` or `missing 1 required positional argument: 'X'`.

3. **Async-loop bug.** Tests use `asyncio.run(_check())` instead of `@pytest.mark.asyncio`. This works fine BUT is fragile if any imported module instantiates a long-lived asyncio object at import time. If v1015/v1016 introduced module-level `aiohttp.ClientSession()` or `httpx.AsyncClient()`, the `asyncio.run()` calls would crash with "RuntimeError: Event loop is closed".

4. **Schema field drift.** Tests construct `ReuploadCommitRequest(token=None)` and `VrtAddSourceRequest(source_dataset_id=source_id)`. If either Pydantic model added a required field in a later phase, the test would fail with `ValidationError`.

**Triage protocol:**
1. Run `cd backend && uv run pytest tests/test_defer_orphan_guard.py -x -v --tb=long 2>&1 | tee /tmp/dog.log`. The first failure gives the failure shape.
2. Identify which of the 9 tests are the 3 failures (the file has 9 tests; 6 must already be passing).
3. For each failure, determine root cause from the traceback:
   - If `InvalidCatalogNameError` → Plan 01 should have fixed it; re-run after Plan 01 lands
   - If `TypeError` on the production-code call → mock signature drift (fix the mock)
   - If `ValidationError` from Pydantic → schema field drift (update the test fixture)
   - If `AssertionError` on rollback/job-state mutation → production code drift (investigate; the test docstring documents the contract)
   - If `RuntimeError: Event loop` → asyncio refactor needed (convert tests to `@pytest.mark.asyncio`)
4. Apply the smallest fix that makes the test green WITHOUT changing what the test asserts (the contract is sacrosanct — preserve the "what is verified", change the "how it's mocked").
5. If ANY test cannot be fixed root-cause within 30 minutes of investigation, fall back to `pytest.mark.skip(reason="<github-issue-url>")` AND file the issue via `gh issue create --title "..." --body "..."`.
</root_cause_hypothesis>
