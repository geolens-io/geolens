<root_cause_hypothesis>
The CONTEXT.md notes: "5 failures share a fixture or model issue" — likely the case because:

1. **Shared helper drift (PRIMARY HYPOTHESIS).** `_map()`, `_layer()`, `_dem_layer()` are SHARED across multiple test functions. If ONE field added to the `Map` ORM model became required (e.g., a new NOT NULL column in v1015 alembic 0021 or v1016 alembic 0022), the helper's instantiation crashes the test BEFORE any assertion runs. Result: every test using that helper fails with the same exception. 5 sibling failures = 5 tests using the broken helper.

2. **Style-JSON schema drift in production.** `build_maplibre_style()` and `parse_maplibre_style_import()` define the wire format. If a recent phase added a new top-level key to the output (e.g., `terrain`, `transition`, `pitchAlignment`) or renamed an existing key (e.g., `sources` → `source-collection`), tests asserting on specific JSON shapes fail.

3. **MapLayerResponse Pydantic schema drift.** The `_layer()` helper constructs a `MapLayerResponse(...)` with ~25 fields. If any field became required, optional, or renamed between v1015 baseline and v1016 close, every test using `_layer()` errors at fixture-construction time with `ValidationError`.

4. **Tile-signing time-sensitivity.** `verify_tile_signature` may have time-bounded tokens. If recent phases changed the token validity window or the signing-key environment variable, tests asserting "URL contains valid signature" fail on every run.

5. **Cluster / DEM / 3D-extrusion field drift.** v1004/v1005 cluster intent metadata, v1008 basemap-position jsonb-additive (per MEMORY.md), v1011 paint._height_column — any of these added new style fields. If `parse_maplibre_style_import` rejects unknown fields strictly, OR `build_maplibre_style` no longer emits a field the test expects, the assertion fails.

3. Read `backend/app/modules/catalog/maps/models.py::Map` and `backend/app/modules/catalog/maps/schemas.py::MapLayerResponse` to identify any new required fields since v1015.
4. Apply the SHARED fix to the helper if hypothesis 1/3 is confirmed (e.g., add a default value for the new required field). Single helper change → 5 failures auto-resolve.
5. If hypothesis 2 or 5 is the root cause, the fix is to the test assertions (not the helpers). Apply per-test.
6. Skip-with-issue only if root cause is fundamental production-code drift that exceeds milestone scope.
