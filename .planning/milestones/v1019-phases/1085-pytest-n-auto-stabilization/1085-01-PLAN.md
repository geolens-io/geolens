**Postgres connection ceiling** (`db/postgresql.conf:11`):
  max_connections = 30          # PERF-05 (Phase 274): matches v13.13 worst-case
                                # API(16) + Worker(13) + Admin(1). See README
                                # "Connection Pool Budget".

**Test-DB asyncpg pool sizing** (`backend/tests/conftest.py:354-360`):
  test_engine = create_async_engine(
      settings.test_database_url,
      pool_size=5,
      max_overflow=2,
      pool_timeout=30,
      echo=False,
  )

  # Per-worker pool ceiling = pool_size + max_overflow = 5 + 2 = 7 connections.
  # On `-n auto` (16 workers on a 16-core M-series macOS dev host), the
  # theoretical peak = 16 × 7 = 112 connections, ~3.7× the max_connections=30
  # ceiling. Postgres rejects new conns past max_connections with
  # `FATAL: sorry, too many clients already`, which asyncpg surfaces as
  # `ConnectionRefusedError` / `CannotConnectNowError` and (when it happens
  # to a session mid-DDL) cascades into a recovery cycle that flushes shared
  # buffers, surfacing as `InvalidCatalogNameError` against worker test DBs
  # that DROP-then-CREATE around the same window.

**Per-worker test DB name shape** (`backend/tests/conftest.py:90-109`):
  _worker_test_database_name(base_name) returns "{base_name}_{worker_id}_{8-hex-uuid}"
  where worker_id ∈ {"master", "gw0", "gw1", ..., "gw15"}.

**xdist worker count capture** (`backend/tests/conftest.py:25-40`):
  pytest_configure() captures `config.workerinput["workerid"]` into
  PYTEST_XDIST_WORKER env var. Worker count == max(int(workerid[2:])) + 1
  across all live workers, OR can be read from `pytest --collect-only -n auto`
  output's `gw0..gwN` lines.

**Test DB connection target** (`.env.test` + `backend/app/core/config.py`):
  Tests connect to Postgres on `127.0.0.1:${DB_PORT:-5434}` against the live

  `docker-compose.yml:88-92` (defaults to 5434 when `.env` sets `DB_PORT=5434`,
  falls back to 5432 if `.env` is absent).

**pg_stat_activity sampling pattern** (from existing teardown at conftest.py:316-320):
  SELECT pid, datname, application_name, state, query_start, state_change
  FROM pg_stat_activity
  WHERE pid <> pg_backend_pid()
  -- Filter by test-DB name pattern for per-worker breakdown:
  AND datname LIKE 'geolens_test_gw%'
  ORDER BY datname, pid;
