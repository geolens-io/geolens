<objective>
Eliminate the `/api/api/` double-prefix bug on quicklook proxy URLs at source. The bug: `frontend/src/components/maps/hooks/use-quicklook.ts:58` passes a hardcoded path starting with `/api/datasets/...` to `apiFetchBlob`, which itself does `safeFetch(\`${API_BASE}${path}\`)` (where `API_BASE = '/api'` from `src/lib/constants.ts:3`). The two prefixes concatenate to `/api/api/datasets/...`.

Every other `apiFetch`/`apiFetchBlob` caller in the codebase passes paths WITHOUT the `/api` prefix (e.g., `src/api/maps.ts:50` does `apiFetch('/maps/...')`, `src/api/datasets.ts:388` does `apiFetch('/datasets/...')`). `useQuicklook` is the sole outlier and must conform.

Purpose: TD-12 close. The doubled prefix is cosmetic today (all responses 200 OK per REQUIREMENTS.md) but blocks future per-prefix auth/rate-limit middleware and pollutes smoke logs / network-error dashboards.

Output: One production line changed in `use-quicklook.ts`, one test assertion updated in `use-quicklook.test.ts`, Playwright MCP smoke session asserting zero `/api/api/` URLs in the network log.

Scope guardrails (REQUIREMENTS.md TD-12 explicit):
- Fix at source in `frontend/src/api/` (or its consumer hooks) — NOT via nginx config patch
- Production hook line 58 is the single source of truth — do not modify `apiFetchBlob`, `apiFetch`, `API_BASE`, or any backend route
- Other `apiFetchBlob` callers (`useMapThumbnail` already uses backend-supplied URLs that omit `/api`; the test fixtures in `MapCard.test.tsx` and `use-map-thumbnail.test.ts` contain `/api/maps/1/thumbnail/` strings but the mock layer prevents real concatenation — leave those alone; they are mock fixtures, not bug surfaces)
</objective>

API_BASE definition (`src/lib/constants.ts:3`):

```
export const API_BASE = getEnvConfig().API_BASE_URL || '/api';
```

`apiFetchBlob` concatenation (`src/api/client.ts:174-189`, key lines shown):

```
export async function apiFetchBlob(
  path: string,
  options: RequestInit = {},
): Promise<Blob> {
  const response = await authenticatedFetch(path, options, ...);
  // ...
}
// authenticatedFetch internally does:
//   const response = await safeFetch(`${API_BASE}${path}`, { ... });
// (src/api/client.ts:102)
```

So whatever path is passed in, `API_BASE` (`/api`) is prepended by `safeFetch`.

The bug site (`src/components/maps/hooks/use-quicklook.ts:55-58`):

```
const { data, isPending, error } = useQuery({
  queryKey: ['quicklook', datasetId, size],
  queryFn: async () => {
    const blob = await apiFetchBlob(`/api/datasets/${datasetId}/quicklook?size=${size}`);
    return URL.createObjectURL(blob);
  },
  // ...
});
```

The string template `/api/datasets/...` includes `/api/` literally → `apiFetchBlob` prepends `/api` → final URL is `/api/api/datasets/<id>/quicklook?size=256`.

Conformant callers for reference (single-prefix pattern):

```
// src/api/maps.ts:50
return apiFetch<MapListResponse>(`/maps/${qs ? `?${qs}` : ''}`);
// src/api/maps.ts:54
const resp = await apiFetch<MapResponse>(`/maps/${id}`);
// src/api/datasets.ts:388
return apiFetch<...>(`/datasets/${datasetId}/relationships/`);
// src/api/datasets.ts:412
return apiFetch<...>(`/datasets/${datasetId}/...`);
```

Sibling hook `useMapThumbnail` (`src/components/maps/hooks/use-map-thumbnail.ts:36`) is NOT a bug — it passes `thumbnailPath` (a URL received from the backend), and the backend returns paths shaped `/maps/<id>/thumbnail/` (confirmed at `backend/app/modules/catalog/maps/router.py:325`, `service_crud.py:212`, `service_public.py:559` — all use `f"/maps/{map_obj.id}/thumbnail/"` with NO `/api/` prefix). So `useMapThumbnail`'s call to `apiFetchBlob(thumbnailPath)` produces a clean `/api/maps/<id>/thumbnail/` URL.
