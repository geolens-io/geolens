Fix: add a non-null narrowing step in each test (an `expect(result).not.toBeNull()` followed by a `const r = result!` re-bind, or an `if (!result) throw new Error('test setup invariant failed')`).

StyleConfig contract (driving 7 errors in `map-sync.data-driven-cols.test.ts`) at `src/types/api.ts:836`:

```
export interface StyleConfig {
  [key: string]: unknown;
  mode: 'categorical' | 'graduated';

  ramp: string;  // required, not optional
  classCount?: number;
}
```

The test creates `{ mode: 'categorical', column: 'X', categories: [] }` literals (missing `ramp`). The test exercises column extraction, not ramp validation, so adding the missing required field does not alter intent.

Fix: narrow at the source — type `arr` as `[string, string][]` (or `[string, string, unknown][]`), or destructure without tuple annotation and let inference flow.
