<objective>
Add `line-cap` and `line-join` Selects to LineEditor — both are MapLibre **layout** properties (NOT paint). Extend `line-adapter.ts` with a new exported `LINE_OWNED_LAYOUT_PROPERTIES` tuple AND wire `lineAdapter.syncPaint` to reconcile owned layout properties via the existing `syncOwnedLayoutProperties` helper from `shared.ts`. Editor writes route through the existing `onLayoutChange` callback (BaseStyleEditorProps), which dispatches through the reconciler hook `handleLayoutChange` at `use-layer-map-sync.ts:291` (a v1010/v1011 documented exception per WALK-04 audit — NOT a new direct callsite).

Purpose: Close EDITOR-LINE-01 + EDITOR-LINE-02 (audit rows WALK-L-01, WALK-L-02). Today `lineAdapter.addLayers` hard-codes `'line-cap': 'round'` and `'line-join': 'round'` (line-adapter.ts:189-190); the editor exposes no UI. After this plan, both are user-configurable, persisted in `layer.layout`, and reconciled to MapLibre on subsequent `syncPaint` runs.

Output:
- `line-adapter.ts` exports `LINE_OWNED_LAYOUT_PROPERTIES = ['line-cap', 'line-join'] as const` (LAYOUT, not PAINT — the audit row WALK-L-02 explicitly calls this out).

- `LineEditor.tsx` adds a "Line ends" section heading + 2 `<Select>` controls (Cap, Join) following UI-SPEC § Surface 2.
- 1 new English i18n key set under `style.lineEnds` / `style.lineCap*` / `style.lineJoin*`.
- 6 new tests in `LineEditor.test.tsx` + 1 new test row in `line-adapter.test.ts`.
