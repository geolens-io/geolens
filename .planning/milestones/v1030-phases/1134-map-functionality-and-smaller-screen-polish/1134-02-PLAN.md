<objective>
Close the audit-flagged delete-layer orphan-source surface (WALK-F-02 fill outline/extrusion, WALK-X-02 cluster source race) so every render mode's delete path removes every adapter-owned MapLibre layer.

Purpose: The current `removePerLayerCompanions` at `builder-layer-mutations.ts:14` sweeps a hard-coded suffix list `['', '-outline', '-label', '-extrusion', '-arrow', '-cluster', '-cluster-count']`. That list is correct for today's adapters but is decoupled from the `LayerAdapter.getLayerIds()` contract introduced in v1026 — any new adapter or any future companion id will silently orphan. The fix is to drive the sweep from `getLayerIds` (the source of truth) so the next adapter shipped automatically participates.

Output:
- One source change: `removePerLayerCompanions` consults the layer-adapter registry to derive ids when the layer's render mode is known; falls back to the existing suffix sweep when not.
- One new test file (`builder-layer-mutations.test.ts`) — 8 per-render-mode regression cases.
- One extended test file (`use-builder-layers.delete.test.ts`) pinning the call-site contract.

This plan does NOT modify `use-builder-layers.ts` itself — the call-site at line 310 already passes `[layerId]` to `removePerLayerCompanions` and that contract is preserved via optional new argument.
</objective>
