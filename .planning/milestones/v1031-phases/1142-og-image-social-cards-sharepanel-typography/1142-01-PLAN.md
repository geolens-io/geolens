<objective>
Add the SHARE-08 backend surface (Path A, per locked decision D-03): a crawler-facing
`GET /api/maps/shared/{token}/card` HTML route that emits server-rendered per-map OG/Twitter
meta, plus a dedicated 1200x630 OG-image upload/serve pipeline (`PUT`/`GET /maps/{id}/og-image/`)
backed by a new nullable `catalog.maps.og_image_uri` column.

Purpose: Vite-SPA share links (`/m/{token}`) are invisible to social crawlers (no JS execution),
so shared maps unfurl with a generic site card. A dedicated backend HTML route is the
smallest-blast-radius mechanism (zero nginx change, existing `/m/` flow untouched) that gives
crawlers per-map meta. Path A is locked because the existing thumbnail upload caps payload at
100KB (`ThumbnailUploadRequest`, schemas.py:954) — a 1200x630 JPEG exceeds it, and relaxing that
cap would break the existing tested contract.

Output: migration 0024, `og_image_uri` column, `OgImageUploadRequest` schema, `MapResponse.og_image_url`
field, three routes, and a focused pytest module. Frontend wiring is Plan 02.
</objective>

|----------|-------------|
| anonymous crawler → `GET /shared/{token}/card` | untrusted token + crawler reads server-rendered HTML built from map name/description |
| map owner → `PUT /maps/{id}/og-image/` | authenticated owner uploads attacker-controllable image bytes |
| anonymous client → `GET /maps/{id}/og-image/` | unauthenticated read of stored image bytes |
