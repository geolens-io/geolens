<objective>
Add a built-in fill-pattern authoring control to the fill-render-mode editor (`FillEditor`). The user picks a pattern from a curated built-in set (hatch / cross-hatch / diagonal / dots / grid) or "None" (solid). Selecting a pattern writes the real MapLibre `fill-pattern` paint property; selecting "None" clears it back to a solid `fill-color` render â€” no page reload.

Purpose: Closes **EDITOR-FILL-01** (the last v1031 render-mode control). Patterns make polygon layers visually distinguishable on the map without relying on color alone.
