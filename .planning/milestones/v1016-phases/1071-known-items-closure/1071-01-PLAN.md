# Current state — current lockfile entry for idna
@backend/pyproject.toml

The CVE: `idna.encode()` (and `idna.uts46_remap()` indirectly) can be made to
exhibit super-linear runtime on attacker-controlled input strings containing
long IDNA labels. Fixed in idna 3.15 (October 2025).

Transitive sources for idna in this repo: `httpx` (auth, embedding, AI providers),
`requests` (probe adapters), `email-validator` (via `pydantic[email]`).

The fix pattern matches the precedent set for `urllib3>=2.7.0` and
`cryptography>=46.0.7` at `backend/pyproject.toml:47-53` — add an explicit
floor constraint in `dependencies` and let the resolver propagate.

</context>
