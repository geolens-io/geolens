## Naming the shared helper
The current name `_gdal_safe_env` is a single-leading-underscore (module-private)
helper in vrt.py. Two viable refactor shapes:

**Shape A — Drop the underscore (make it public):**
Rename to `gdal_safe_env()` in vrt.py, import directly from cog.py.

**Shape B — Move to a new shared module:**
Create `backend/app/processing/raster/gdal_env.py` with the helper +
constants, and have both vrt.py and cog.py import from there.

Shape A is the lighter touch and matches the existing convention (other
single-underscore helpers in the raster module are imported across siblings
when needed). Pick Shape A unless cog.py imports raster/vrt.py for ONLY
this purpose (in which case Shape B's cleaner module boundary wins).

      """Returns a (fake_run, captured) pair. captured['env'] holds the most
      recent env= kwarg passed to subprocess.run."""

      def fake_run(cmd, *args, **kwargs):
          captured["cmd"] = cmd
          captured["env"] = kwargs.get("env", os.environ.copy())
          return mock.Mock(returncode=0, stderr="", stdout="")

  class TestCogSubprocessEnvOverlay:
