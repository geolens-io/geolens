<objective>
Close REMED-01 (ingest-audit P2-06): wire `onSuccess` handlers on every reupload/VRT mutation hook so `jobStatusByDataset` cache is invalidated when the underlying ingest state mutates, eliminating stale warning banners on the dataset-detail page.

Purpose: Today `useDatasetJobStatus` is configured with `staleTime: Infinity` (`use-ingest.ts:46`), so after a successful re-upload the dataset-detail warnings banner shows the OLD job's warnings until the tab is hard-refreshed. This plan adds targeted invalidations so TanStack refetches the new job exactly when one would be created.

Output: 3 hook files updated, 3 test files updated with regression coverage.
</objective>
