# Phase 999.4 Plan 01: Shared Vector Staging Pipeline Summary

**One-liner:** Extracted ~150-line duplicated ogr2ogr-through-metadata pipeline from `ingest_file` and `reupload_file` into a single `_ingest_vector_into_staging` helper with `StagingResult` dataclass return.

## What Was Built

### StagingResult dataclass

Added at `tasks.py:54` — bundles the post-ogr2ogr results (metadata, sample_values, three_d, has_geometry, geometry_type) so callers can feed them into their divergent downstream steps.

### _ingest_vector_into_staging helper

Added at `tasks.py:305` — single async function covering the shared pipeline:
1. `run_ogr2ogr` — load file into target_table
2. `rename_reserved_columns` — append warning if renames occurred
3. DBF truncation detection — .zip files only, re-runs ogrinfo_preview if columns not available
4. Geometry override — runs `_detect_and_override_geometry` when `user_wants_geom=True` (ingest-only per D-04)
5. `ensure_geom_column` + `clip_to_mercator_bounds` + `add_4326_column` — when has_geometry=True
6. `grant_reader_access`
7. `extract_metadata` + `detect_3d_metadata` + conditional `promote_z_to_elev` + `get_column_info`
8. `get_sample_values`
9. Returns StagingResult

### Rewired callers

**ingest_file** (`tasks.py:~870`): Removed inline ogr2ogr + rename + DBF + geometry override blocks. Calls `_ingest_vector_into_staging` and passes `staging=staging` to `_finalize_ingest`.

**reupload_file** (`tasks.py:~1382`): Removed inline ogr2ogr + rename + DBF + post-process + metadata extraction blocks. Calls `_ingest_vector_into_staging` and uses `staging.metadata`, `staging.sample_values`, `staging.three_d` in downstream calls.

### _finalize_ingest updated

Accepts optional `staging: StagingResult | None = None` parameter. When `None` (service ingestion path), runs full post-processing inline unchanged. When provided (vector file path), uses pre-computed results — skips redundant DB work.

## Deviations from Plan

### Auto-resolved Implementation Detail

**[Rule 1 - Deviation from D-04 literal] Geometry override runs inside helper, not before it**
- **Found during:** Task 1 analysis
- **Issue:** The plan's D-04 states "ingest can run the override before calling the helper and pass the result in" — but `_detect_and_override_geometry` requires the ogr2ogr-loaded table to exist, so it cannot run before the helper if ogr2ogr is inside the helper
- **Fix:** Helper accepts `user_wants_geom: bool` and `user_metadata: dict` instead of a pre-computed `geometry_override_result`. When `user_wants_geom=True`, helper calls `_detect_and_override_geometry` internally after ogr2ogr. Reupload passes `user_wants_geom=False` (default), honoring the D-04 spirit: geometry override never applies to reupload
- **Files modified:** `backend/app/ingest/tasks.py`
