# Phase 999.4 Plan 02: Integration Tests for Shared Vector Staging Pipeline Summary

**One-liner:** Added 3 real-ogr2ogr integration tests for `_ingest_vector_into_staging` covering spatial GeoJSON (ingest path), staging table naming (reupload path), and non-spatial CSV, with skipif guard for hosts without ogr2ogr.

## What Was Built

### TestStagingPipelineIntegration

`backend/tests/test_staging_pipeline_integration.py` — 3 integration tests in `class TestStagingPipelineIntegration`, all marked `requires_ogr2ogr` (activates `_point_ogr2ogr_at_test_db` fixture from conftest automatically).

**test_ingest_path_spatial_geojson**
- Creates a 3-feature GeoJSON via `_create_test_geojson(tmp_path)`
- Runs `run_ogrinfo` to detect SRID/geometry type
- Calls `_ingest_vector_into_staging` with `has_geometry=True`, `effective_srid=4326`
- Asserts `StagingResult.has_geometry=True`, `feature_count=3`, `column_info` has "name" and "height"
- Verifies DB: `count(*) == 3` in `data."{table_name}"`, `geom_4326` column exists
- Cleans up in `finally` block

**test_reupload_path_staging_table**
- Same GeoJSON setup; uses `{base}_staging` table name pattern (reupload convention)
- Same metadata assertions
- Verifies staging table can be renamed to `{base}_final` (simulating `_apply_reupload_swap`)
- Cleans up both staging and final tables in `finally`

**test_nonspatial_csv_path**
- Creates `test_data.csv` (id/name/value, 3 rows)
- Calls helper with `has_geometry=False`, `ogr_geometry_type=None`
- Asserts `StagingResult.has_geometry=False`, `geometry_type=None`, `feature_count=3`
- Verifies DB: 3 rows loaded, NO `geom_4326` column
- Cleans up in `finally`

## Tests

All 3 tests collect and skip gracefully when `ogr2ogr` binary is absent (dev hosts). Skip guard matches the existing pattern in `test_ingest_column_preservation.py`. Tests run in full in Docker / CI where GDAL is installed.

Regression baseline: `test_staging_pipeline.py` — 10 passed.

## Deviations from Plan

### Minor: No pytest.mark.asyncio in pytestmark

**Found during:** Task 1 implementation
**Issue:** Plan specified `pytestmark = [pytest.mark.requires_ogr2ogr, pytest.mark.asyncio]` but the project uses `anyio_mode = "auto"` in `pyproject.toml`, so `pytest.mark.asyncio` would conflict with the anyio runner and is not used in existing integration tests
**Fix:** Omitted `pytest.mark.asyncio` from pytestmark, matching the pattern in `test_ingest_column_preservation.py`
**Files modified:** `backend/tests/test_staging_pipeline_integration.py`
**Commit:** 89292244

### Minor: Used existing GeoJSON fixture path instead of inline helper

**Found during:** Task 1 implementation
**Issue:** Plan provided a `_create_test_geojson` helper that generates GeoJSON in `tmp_path`. While that helper is implemented in the test file, the existing `fixtures/ingest/basic_attrs.geojson` would also work. Decision: keep the helper as specified in the plan so tests are self-contained and don't depend on fixture file contents staying stable.
**Result:** Both approaches present — `_create_test_geojson` creates deterministic 3-feature NYC landmarks GeoJSON in `tmp_path`.
