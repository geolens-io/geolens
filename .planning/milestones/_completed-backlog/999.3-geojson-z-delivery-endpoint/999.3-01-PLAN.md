```python
async def get_features_geojson_z(
    db: AsyncSession,
    table_name: str,
    *,
    cap: int = 5000,
    cached_feature_count: int | None = None,
) -> tuple[list[dict], bool, int]:
```

Implementation details per D-01, D-03, D-04, D-08:
- Call `_validate_table_name(table_name)` first (SQL injection safety).
- SELECT: `gid, ST_AsGeoJSON(geom_4326, 6)::json AS geometry, to_jsonb(t.*) - 'gid' - 'geom' - 'geom_4326' AS properties` — same pattern as `get_features` line 84-86. ST_AsGeoJSON already preserves Z natively.
- ORDER BY gid, LIMIT cap+1 (fetch one extra to detect truncation without separate COUNT).
- If len(rows) > cap: truncated=True, trim to cap rows.
- For total_count: if not truncated, total_count = len(rows). If truncated and cached_feature_count is provided, use it. Otherwise run `SELECT COUNT(*) FROM data.{table_name}`.
- Return tuple: (rows, truncated, total_count).
- No bbox, no pagination, no property filters (per D-04, D-05).

    "/{dataset_id}/features.geojson",

async def get_features_geojson_z_endpoint(
    dataset_id: uuid.UUID,
