# Phase 999.3 Plan 02: GeoJSON-Z API Propagation and Map Sync Branching Summary

**is_3d + feature_count wired through map layer API responses and syncLayersToMap extended with a GeoJSON-Z source branch, enabling auto-switch between MVT and GeoJSON-Z sources for small 3D datasets**

## Performance

- **Duration:** ~15 min
- **Started:** 2026-04-12T00:00:00Z
- **Completed:** 2026-04-12
- **Tasks:** 2
- **Files modified:** 10

## Accomplishments

- Added `is_3d` to `MapLayerResponse` and both `is_3d`/`feature_count` to `SharedLayerResponse` in backend schemas and frontend TS types
- Extended `get_map_with_layers` and `get_shared_map` queries with `Dataset.is_3d` and `Dataset.feature_count` columns; tuple unpacking updated throughout
- All four vector adapters (circle, fill, line, heatmap) conditionally omit `source-layer` when `sourceType === 'geojson'`
- `syncLayersToMap` extended with optional `geojsonDataMap` parameter and a GeoJSON-Z branch that handles add/update/zoom-range/labels/visibility — existing callers unaffected

## Decisions Made

- `geojsonDataMap` is an optional trailing parameter so all existing `syncLayersToMap` callers remain valid without changes — the GeoJSON-Z branch never activates until Plan 03 wires it up
- GeoJSON label layers pass `sourceLayer: ''` (empty string) because MapLibre GeoJSON sources do not use source-layer routing
- All four adapters updated for consistency even though heatmap + 3D rendering is unusual
