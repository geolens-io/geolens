## What Was Built

### Service function (`backend/app/features/service.py`)

`get_features_geojson_z(db, table_name, *, cap=5000, cached_feature_count=None)`:
- Calls `_validate_table_name()` first (SQL injection safety per T-999.3-01)
- `SELECT gid, ST_AsGeoJSON(geom_4326, 6)::json AS geometry, to_jsonb(t.*) - 'gid' - 'geom' - 'geom_4326' AS properties FROM data.{table_name} t ORDER BY gid LIMIT cap+1`
- LIMIT cap+1 detects truncation without extra COUNT query
- `ST_AsGeoJSON` natively preserves Z when geometry has Z — no special flag needed
- Returns `(rows, truncated, total_count)`: total_count = `len(rows)` when not truncated; `cached_feature_count` when truncated and provided; `COUNT(*)` otherwise

### Router endpoint (`backend/app/features/router.py`)

`GET /{dataset_id}/features.geojson`:
- `get_current_active_user` dependency (JWT + API key) — requires authentication (T-999.3-04)
- `check_dataset_access()` enforces visibility + publication status per RBAC (T-999.3-02)
- 404 on non-existent dataset, 404 on private dataset for non-owner
- 400 on dataset with `geometry_type is None`
- Returns `JSONResponse(content=body, media_type="application/geo+json")`
- Response body: `{"type": "FeatureCollection", "features": [...], "truncated": bool, "total_count": int}`

### Tests (`backend/tests/test_features_geojson_z.py`)

10 tests covering:
- Service: Z coordinates preserved, truncation at cap, cached count behavior
- HTTP: 401 unauthenticated, 404 non-existent dataset, 404 private dataset non-owner, 200 FeatureCollection shape, Z coords in response, 400 no-geometry dataset, truncation at 5000

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] geom_4326 column type incompatible with Z geometry in tests**
- **Found during:** Task 2, first test run
- **Issue:** `geometry(Geometry, 4326)` column type in PostGIS rejects Z geometry insertions with "Geometry has Z dimension but column does not". The plan's test helper used this type, matching the production ingestion pattern, but PostGIS enforces 2D-only for typed geometry columns.
- **Fix:** Changed test table `geom_4326` column to plain `geometry` (no type/srid modifier) in both `_create_test_table_and_dataset_3d` and `_create_large_table_and_dataset`. Plain `geometry` accepts any geometry including Z.
- **Files modified:** backend/tests/test_features_geojson_z.py
- **Commit:** ec7673be
