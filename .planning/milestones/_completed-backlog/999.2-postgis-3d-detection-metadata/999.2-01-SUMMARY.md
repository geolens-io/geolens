# Phase 999.2 Plan 01: PostGIS 3D Detection Foundation Summary

**One-liner:** Alembic migration adds 4 nullable 3D columns to datasets; detect_3d_metadata() queries ST_NDims/ST_Is3D/ST_ZMin/ST_ZMax; DatasetResponse exposes all four fields.

## What Was Built

- **Alembic migration** `2026_04_12_1500` adds `is_3d` (Boolean), `n_dims` (SmallInteger), `z_min` (Float), `z_max` (Float) as nullable columns to `catalog.datasets`. Downgrade drops all four.
- **Dataset ORM model** gains four mapped columns in the auto-extracted metadata section, with `SmallInteger` added to the sqlalchemy import block.
- **`detect_3d_metadata(session, table_name)`** async function in `metadata.py` queries PostGIS: samples one row for `ST_NDims`/`ST_Is3D`, then runs a full-table `MIN(ST_ZMin)/MAX(ST_ZMax)` aggregate only when the geometry is 3D. Uses `_validate_table_name` and `_qtable` for all table name interpolation (mitigates T-999.2-01).
- **DatasetResponse** schema gains `is_3d`, `n_dims`, `z_min`, `z_max` fields (all `Optional`, default `None`) after `geometry_type`.
