# Phase 999.1: 3D Viewer Toggle (Terrain + Extrusions) - Context

**Gathered:** 2026-04-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a 3D mode toggle to `ViewerMap.tsx` that enables MapLibre terrain (via Titiler `terrainrgb` algorithm on DEM rasters) and building extrusions (via `fill-extrusion` layer reading a height column). Viewer-only — Builder 3D support is a separate follow-on. KISS over complexity; enhance later.

</domain>

<decisions>
## Implementation Decisions

### 3D Toggle UX
- **D-01:** Use MapLibre's built-in `TerrainControl` component for the 3D toggle — zero custom UI. Matches what users of other MapLibre apps expect.
- **D-02:** 3D mode always enables pitch/rotate navigation and extrusions even without DEM terrain. Terrain stays flat if no DEM is present. 3D mode = "3D camera controls + extrusions", not "terrain required".

### Height Column Binding
- **D-03:** Style editor dropdown only — lists numeric columns from the dataset. No auto-detection of common column names (height, HEIGHT, BLDG_HT). Explicit user choice, zero false positives.
- **D-04:** Builder configures height column (new dropdown in layer style editor), Viewer renders extrusions. This is the minimal scope — users set it up in Builder, viewers see the result.

### DEM Identification
- **D-05:** Auto-flag single-band float rasters as "likely DEM" during ingest (soft heuristic in `extract_raster_metadata`). Manual `is_dem` toggle in dataset metadata editor for overrides. Catches most DEMs automatically; admin corrects false positives like NDVI rasters.
- **D-06:** Dataset owner + admin can toggle `is_dem`. Matches existing dataset edit permission model.

### Empty/Error States
- **D-07:** Graceful no-op when 3D is toggled with no DEM and no height-configured layers. Just pitch/rotate controls on a flat map. No toast, no warning. Zero edge-case UI.
- **D-08:** Existing `MapErrorBoundary` (added in Phase 224) handles WebGL 3D failures. No 3D-specific error handling needed.

### Locked from Design Doc (260408-aa5)
- **D-09:** Session-only 3D toggle — resets to 2D each session. No schema changes for persistence.
- **D-10:** Hardcoded terrain exaggeration `1.5`. No user-adjustable slider for v1.
- **D-11:** Embed-token smoke test is an acceptance criterion — confirm terrain tiles flow through existing `X-Embed-Token` proxy path.
- **D-12:** Warn in layer style editor when a bound height column is absent after re-upload (use existing schema-diff preview infrastructure).
- **D-13:** Viewer-only scope. Defer `BuilderMap.tsx` full 3D mode to a follow-on phase.

### Claude's Discretion
- Pitch transition animation (instant vs smooth) when toggling 3D
- `minzoom` threshold for `fill-extrusion` layers (design doc suggests 14 for performance)
- Terrain source re-application in `style.load` hook implementation details

</decisions>

## Existing Code Insights

### Reusable Assets
- `MapErrorBoundary` — Already wraps ViewerMap (Phase 224). Handles WebGL crashes.
- `syncLayersToMap` helper — Shared by viewer and builder for layer re-application after style changes.
- `map.setTransformRequest()` in ViewerMap `onLoad` — Already covers terrain tile auth via `X-Embed-Token`.

- Imperative `map.addSource()` + `map.addLayer()` for all vector tile layers (declarative `<Source>` unreliable for vector tiles per CLAUDE.md)
- Declarative `terrain` and `sky` props on `<Map>` DO work in react-maplibre v8
- `style.load` listener at ViewerMap:373-391 re-syncs data layers after basemap change — terrain must hook into this
