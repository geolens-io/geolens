# Phase 999.1 Plan 01: is_dem Backend Infrastructure Summary

**One-liner:** `is_dem` boolean on `raster_assets` auto-set by single-band float heuristic during ingest, PATCH-able by owner/admin, terrain tiles use `algorithm=terrainrgb`, and `SharedLayerResponse` exposes the flag for the viewer.

## What Was Built

### Task 1: is_dem column, DEM heuristic, ingest wiring (commit 68e9695f)

- **Alembic migration** `2026_04_12_1400-add_is_dem_to_raster_assets.py`: adds `is_dem BOOLEAN NOT NULL DEFAULT false` to `catalog.raster_assets`, chains from revision `989ae68d7859`.
- **RasterAsset model**: `is_dem: Mapped[bool]` field added after `is_rotated`.
- **extract_raster_metadata** in `cog.py`: computes `is_dem_candidate = src.count == 1 and _is_float_dtype(src.dtypes[0])`, returns it in the metadata dict. Builds on the already-present `_is_float_dtype` helper.
- **Ingest pipeline** in `tasks.py`: `create_raster_dataset` passes `is_dem=meta.get("is_dem_candidate", False)` to the `RasterAsset` constructor.

### Task 2: API surface exposure (commit aed9c984)

- **tiles/router.py**: `_resolve_raster_access` SQL now selects `ra.is_dem`. `raster_auth_check` conditionally prepends `algorithm=terrainrgb` to `render_params` for DEM datasets — applies to both nginx auth_request path (via `X-GeoLens-Render-Params` header) and the `raster_tile_proxy` fallback path.
- **datasets/schemas.py**: `DatasetMeta` gains `is_dem: bool | None = None`.
- **datasets/service.py**: `update_user_metadata` checks `meta.is_dem is not None` and updates the associated `RasterAsset.is_dem` if a raster asset exists for the dataset. Non-raster datasets silently skip (no raster asset found).
- **maps/schemas.py**: `SharedLayerResponse` gains `is_dem: bool = False`.
- **maps/service.py**: `get_shared_map` adds `outerjoin(RasterAsset, RasterAsset.dataset_id == Dataset.id)` to the layer query, destructures `ds_is_dem` from results, serializes `"is_dem": bool(ds_is_dem)` into each layer dict.

## Decisions Made

1. **is_dem update in service layer, not router**: The plan suggested updating `RasterAsset.is_dem` in the router handler. Instead, the logic was placed in `update_user_metadata` in `datasets/service.py` — consistent with how all other `DatasetMeta` fields are handled. The router delegates to the service; no change to the router was needed.

2. **Heuristic: single-band float**: Auto-flags DEMs at ingest time. Admin/owner can override via PATCH. False positives (NDVI, single-band SAR) need manual correction, which is acceptable per D-05.

3. **terrainrgb placement**: `algorithm=terrainrgb` is prepended before any existing `render_params` (band/rescale params) to ensure titiler receives it first.

- **Fix:** Added `is_dem` handling in `update_user_metadata` in `datasets/service.py` instead, consistent with all other `DatasetMeta` field updates.
