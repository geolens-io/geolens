# Phase 999.1 Plan 04: Height Column Dropdown in LayerStyleEditor Summary

**Height column Select dropdown for polygon layers with numeric columns, storing selection in paint._height_column, with AlertTriangle warning when bound column is absent from column_info**

## Performance

- **Completed:** 2026-04-12

## Accomplishments

- Added `NUMERIC_COLUMN_TYPES` constant set covering all PostgreSQL numeric types: `integer`, `bigint`, `smallint`, `numeric`, `real`, `double precision`, `float4`, `float8`, `int2`, `int4`, `int8`
- Added `AlertTriangle` to lucide-react import and `useMemo` to React import
- Added `isPolygon`, `numericColumns`, and `currentHeightCol` derivations inside the component function
- Added height column `Select` dropdown in the polygon fill section, rendered only when `isPolygon && numericColumns.length > 0`; selecting a column writes `paint._height_column` via `onPaintChange`; selecting "None" (`__none__` sentinel) deletes `_height_column` from paint
- Added `AlertTriangle` warning block that renders when `isPolygon && currentHeightCol` is set but the column name is no longer present in `layer.dataset_column_info` — covers D-12 without requiring schemaDiff infrastructure
- TypeScript compiles clean with no errors

## Task Commits

## Files Created/Modified

- `frontend/src/components/builder/LayerStyleEditor.tsx` — NUMERIC_COLUMN_TYPES, AlertTriangle import, useMemo, height column dropdown, re-upload warning

## Decisions Made

- `__none__` sentinel value for the "None" select item — shadcn's `<Select>` component does not accept empty string as a valid `value` prop; `__none__` is checked on `onValueChange` to trigger `delete newPaint['_height_column']`
- Warning uses `!(layer.dataset_column_info ?? []).some(col => col.name === currentHeightCol)` directly rather than requiring the `schemaDiff` infrastructure — the condition is equivalent for the user's purpose and does not depend on any upload flow context
- Height column controls placed after stroke controls and before the section closing tag — consistent with the UI spec placement
