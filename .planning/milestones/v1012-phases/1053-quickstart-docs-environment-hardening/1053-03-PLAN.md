  1. After the first paragraph (the one ending with `docker compose up -d`), add an Aside block:

  ```mdx
  <Aside type="note" title="Admin credentials — interactive vs unattended">

  `install.sh` decides which mode to use based on whether the script has a TTY
  attached:

  - **Interactive (default)** — the script prompts for an admin username (default
    `admin`) and password (default `admin`) via `/dev/tty`. Defaults are accepted
    if you press Enter on each.
  - **Unattended** — if the script is piped (`curl … | bash`), running in CI, or
    otherwise has no TTY, the prompt is silently skipped and the script reads
    `GEOLENS_ADMIN_USERNAME` and `GEOLENS_ADMIN_PASSWORD` from the environment.
    Set both before running:

    ```bash
    export GEOLENS_ADMIN_USERNAME=admin
    export GEOLENS_ADMIN_PASSWORD=$(openssl rand -base64 16)
    bash scripts/install.sh
    ```

  If neither the env vars NOR the TTY prompt provide values, the application
  refuses to start at first boot (the empty-string guard from Phase 273
  SEC-15).

  </Aside>
  ```

  2. Remove the now-redundant `For unattended installs, ...` paragraph (its content is fully absorbed into the Aside).
