<objective>
The audit's IMPORT-02 (MEDIUM) finding: Playwright's strict pointer-event click on the visible "Choose File" / browse trigger area times out with `subtree intercepts pointer events`. Root cause is a decorative `<span class="absolute -inset-1.5 rounded-[20px] border border-dashed border-primary/40">` at `frontend/src/components/import/FileDropzone.tsx:90`. This span is a visual flourish (dashed ring around the upload glyph) at absolute positioning that overlays the entire glyph block — and its default `pointer-events: auto` makes it a hit-test target.

For real users, mouse clicks usually work because browsers do hit-testing through the pointer-events chain in `<input type="file">`'s native handler, but keyboard activation, screen-reader interaction, and automated testing all fail. The fix is one Tailwind class: `pointer-events-none` on the decorative span.

Per CONTEXT.md's locked decision: this is a single one-class-attribute change. No tests required (CSS-only change, no behavior to assert beyond pointer-event semantics).

Purpose: Close IMPORT-02 — restore reliable click hit-testing on the upload dropzone.
Output: One-line CSS change in `FileDropzone.tsx`.
</objective>
