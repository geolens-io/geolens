<objective>
The audit's ROUTE-01 finding: `/admin/saml` in community edition silently redirects to `/admin/overview` with no toast, banner, or breadcrumb hint — an operator hitting the URL deliberately (bookmark, copy-paste from old docs) has no signal that SAML is enterprise-only.

Per the audit's recommendation (option A) and CONTEXT.md's locked decision, replace the `<Navigate to="/admin" replace />` short-circuit with an inline "Enterprise Feature" placeholder that:

1. Explains briefly what SAML SSO is and that it requires the GeoLens Enterprise overlay.
2. Links to the relevant docs at `https://docs.getgeolens.com/guides/enterprise/saml/` (use the docs base from the existing footer's `t('common:footer.docs')` link if there's a centralized constant; otherwise the literal URL is fine).
3. Keeps the URL at `/admin/saml` (no redirect).
4. Renders inside the existing `PageHeader` + `PageShell` chrome so it visually matches the rest of the admin section.

The `useEdition().isEnterprise === true` path is unchanged.

Purpose: Close ROUTE-01 — operators discovering SAML get a signal instead of a vanish.
Output: Updated `AdminSamlPage.tsx` + new vitest tests + 4-locale i18n keys.
</objective>
