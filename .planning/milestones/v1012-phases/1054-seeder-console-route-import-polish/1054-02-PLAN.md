CONSOLE-01 closes the audit's "12 console errors on anonymous Search page" finding. Investigation against the live audit evidence shows that the v1010.2 SF-06 gating IS in place on the direct admin probe consumers (`useAIStatus` is gated at `AIStatusCard.tsx:22` and `SettingsAITab.tsx:50`, both with `{ enabled: !!token && isAdmin }`), and `useUser`/`usePermissions`/`useSavedSearches` are correctly gated on `!!token`.

The remaining leak is **`useAIAvailability`** at `frontend/src/hooks/use-ai-availability.ts:7`, which gates on `!!token` only (no `isAdmin` check), and is consumed by FOUR non-admin user-facing surfaces:
- `MapCreateDialog.tsx:39` (anyone can create a map)
- `OverviewTab.tsx:167` (dataset detail Overview tab вЂ” public datasets viewable anonymously)
- `MetadataTab.tsx:44` (dataset detail Metadata tab вЂ” same)
- `SourceQualityTab.tsx:82` (dataset detail Source Quality tab вЂ” same)

When a NON-admin authed user (editor/viewer) hits any of those, OR when the audit's anonymous-session test browsed `/datasets/{Wetlands}` with a stale localStorage token from a prior admin session (the audit's "fresh" install had previously run with `admin/admin`), the hook fires `GET /api/admin/ai-status/` and the backend `admin_required` dependency returns 401/403. The audit's verbatim `[ERROR] 401 @ ai-status/ Г—3` count maps cleanly to the dataset-detail tabs that share `useAIAvailability`.

Other audit-listed 401s вЂ” `/api/auth/refresh/` Г—2, `/api/auth/me/` Г—2, `/api/auth/me/permissions/` Г—3, `/api/search/saved/` Г—2 вЂ” are accountable for by the documented gating already in place. Per CONTEXT.md's audit-recommendation note, the auth-store rehydration probes (`/auth/refresh/`, `/auth/me/`) are "arguably expected" вЂ” they fire ONCE on persist hydration when a stale `refreshToken` lingers from a prior session. Closing those would require either (a) clearing all persisted tokens on logout (already done by `logout()` at `auth-store.ts:76`), or (b) verifying the refresh-token isn't expired before calling `tryRefresh` (currently `tryRefresh` short-circuits on `!refreshToken` but doesn't check expiration). Both are out of scope for CONSOLE-01 per the audit's explicit "out of scope" rationale that they are load-bearing flow.

This plan fixes the one demonstrably-fixable leak вЂ” `useAIAvailability` вЂ” and pins it with a regression test. The audit's anonymous-page count goes from 12 в†’ в‰¤6 (the auth-store rehydration probes remain, deliberately).
