<objective>
Extend the `RENDER_MODES` allowlist and `StyleConfig['render_mode']` union to include `'terrain'` and `'image'`; remove the obsolete boundary cast and BSR-09 comment from `DEMEditorScene.tsx`; and add round-trip unit tests that pin the fix and guard against future regressions.

Purpose: `normalizeRenderMode()` currently discards `'terrain'` (and `'image'`) because they are absent from the `RENDER_MODES` Set at `normalize-style-config.ts:92`. This causes `applyTerrainConfig()` in `BuilderMap.tsx` to receive `render_mode: undefined` instead of `'terrain'`, so `setTerrain(null)` is called and the 3D mesh never attaches on fresh load. The same strip-on-load silently reverts raster "Render as" saves to Image. This is a frontend-only fix — `api.ts` is hand-maintained; backend `style_config` is opaque jsonb with no schema change needed.

Output: Four modified files; `npm run typecheck` exits 0; touched vitest suite passes; `make openapi-check` shows no drift.
</objective>
