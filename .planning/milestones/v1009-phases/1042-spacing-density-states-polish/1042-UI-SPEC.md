## Motion Timing Reference

Token definitions to land in `index.css :root` (AUD-08 — must be first fix in phase):

```css
/* Phase 1042 AUD-08: motion tokens */
--motion-fast: 150ms;
--motion-base: 250ms;
```

| Transition | Token | Tailwind class |
|---|---|---|
| Caret rotate (section expand/collapse) | `--motion-fast` | `transition-transform duration-[--motion-fast]` |
| Row hover background | `--motion-fast` | `transition-colors duration-[--motion-fast]` |
| Insertion line appear/disappear | `--motion-fast` | CSS rule `transition: opacity var(--motion-fast)` |
| BulkActionBar mount/exit slide | `--motion-fast` | `transition-all duration-[--motion-fast]` |
| New StackRow entry (freshLayerId) | `--motion-fast` | `animate-in fade-in duration-[--motion-fast]` (tw-animate-css) |
| SidebarRail icon opacity on selection | `--motion-fast` | `transition-colors duration-[--motion-fast]` |
| EmptyStackState search container color | `--motion-fast` | `transition-colors duration-[--motion-fast]` |
| LayerEditorPanel section expand | `--motion-fast` | `transition-transform duration-[--motion-fast]` |
| Drag ghost appear | none (instant) | `DragOverlay dropAnimation={null}` already shipped |
| Color token changes (dark mode swap) | `--motion-base` | `transition-colors duration-[--motion-base]` on layout root |

**What to avoid:** Do NOT use `transition-all` without a `duration-*` qualifier — it uses the browser default (typically 0ms or 300ms depending on property) and produces inconsistent results. Always pair `transition-{property}` with `duration-[--motion-fast]` or `duration-[--motion-base]`.

- **New design tokens.** Hard constraint from CONTEXT.md. The two motion token additions (`--motion-fast`, `--motion-base`) are explicitly authorized by the audit (AUD-08) and the sketch SKILL — they are not new visual tokens, they are timing constants that codify already-implicit values.

All are Phase 1043.
- **Modifying the saved-map shape.** Hard constraint.
- **Touching public viewer / shared / embed surfaces.** Hard constraint.
- **New backend endpoints.** Hard constraint.
- **New shadcn components.** All controls are pre-installed.
- **`transition-all` without a duration.** Always pair with `duration-[--motion-fast]` or `duration-[--motion-base]`.
- **Hardcoded hex or rgb values.** All color must reference `var(--*)` tokens.
- **Adding a third breakpoint class** to BulkActionBar labels. If `sm:inline` still renders hidden at 340px sidebar width, the correct fallback is Tooltip wrappers — not adding `xs:inline` or similar non-standard breakpoints.
