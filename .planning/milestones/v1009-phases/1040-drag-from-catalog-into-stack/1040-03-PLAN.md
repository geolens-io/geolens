<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| catalog drag data ↔ basemap swap | A drag payload with recordType: 'basemap' triggers onBasemapChange with the dataset id; must gracefully handle unknown ids |
| folder-group drop ↔ parent_group_id | Local state mutation, persisted via existing per-layer PATCH on save |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1040-07 | Spoofing | Catalog drag claims recordType: 'basemap' but datasetId is not a basemap | mitigate | The basemap-swap call to setLocalBasemap(datasetId) resolves via resolveBasemapId() which falls back to the default basemap when the id is not in the registry — graceful failure, no crash |
| T-1040-08 | Tampering | Crafted parent_group_id pointing to a non-existent folder group | mitigate | handleAddLayerToExistingGroup is invoked only when targetLayer && isFolderGroupLayer(targetLayer) is true — the layer existence and type check are the gates |
| T-1040-09 | Information Disclosure | Toast reveals dataset name | accept | The dataset is one the user owns or can access via the catalog; revealing the name to its owner is by design |
</threat_model>
