<objective>
Make catalog rows draggable from the Add Dataset modal and wire the cross-context drop handler to call the existing `handleAddDataset` hook with modal-stays-open semantics. This implements POL-01 (drag affordance) and POL-02 (drop adds layer at position) for the loose-row drop case. Group-drop and basemap-swap drop semantics land in Plan 03; keyboard fallback + DragOverlay ghost for catalog drags lands in Plan 04.

The plan depends on Plan 01's lifted DndContext — it adds `useDraggable` to dataset rows AND extends the lifted `handleDragEnd` with a catalog-source branch.

Purpose: Convert the click-only Add Dataset modal into a drag-and-drop accelerator. After this plan, a user can drag a vector or raster dataset row from the modal onto the unified stack and have it added without closing the modal. The toast confirmation pattern is identical to the existing click-add path.

Output:
- `DatasetSearchPanel.tsx` registers each result row (and each basemap-tab row) as `useDraggable` items with namespaced ids (`catalog:{datasetId}` and `catalog-basemap:{basemapId}`) and `data: { source: 'catalog', ... }`.
- `MapBuilderPage.tsx` `handleDragEnd` branches on `data.source === 'catalog'` and routes to `layers.handleAddDataset(datasetId)` without `onSuccessCb` (so the modal stays open).
- `en/builder.json` has two new toast keys.
- Success toast fires with the dataset name; error toast falls back to existing `toasts.layerAddFailed` (no change).
</objective>
