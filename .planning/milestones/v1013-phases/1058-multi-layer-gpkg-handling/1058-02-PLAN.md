<objective>
GPKG-02 (P1 UX). Polish the Reupload preview pane so the chosen layer name is explicit and schema-change warnings are unmistakable for multi-layer file reuploads.

Purpose: Even after GPKG-01 (Plan 1058-01) plumbs `selectedFileLayer` through preview + commit, the preview pane today still says "File: {name}" without surfacing the chosen layer. Users cannot tell which layer the preview is based on. Service URL preview is the design reference — it already shows "Layer: {name}" + a schema-change warning. This plan brings the file path to parity.

Output:
- Preview-step JSX in `ReuploadDialog.tsx` renders an explicit "Layer: {name}" line above the SchemaDiffView when the file path's `selectedFileLayer` is non-null.
- A new schema-change banner block: "Schema differs from previous version: N columns added, M removed." renders ABOVE SchemaDiffView when `columns_added.length + columns_removed.length > 0`. (The existing `hasWarning` at ReuploadDialog.tsx:389 only triggers on `columns_removed.length > 0 || type_changes.length > 0` — that's a YELLOW destructive warning; this new banner is an informational ADVISORY surfacing the diff summary, distinct from `hasWarning`.)
- File-path and Service URL preview panes use the same visual treatment for the Layer line and schema-change banner.

Wave 2: depends on Plan 1058-01 because the file-path preview only knows `selectedFileLayer` after Plan 1058-01 introduces it. Both plans touch `ReuploadDialog.tsx`, so they MUST run serially — Plan 1058-01 ships the state-machine refactor; this plan ships the preview UI polish on top.
</objective>
