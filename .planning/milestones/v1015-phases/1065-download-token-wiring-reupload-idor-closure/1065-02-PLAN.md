<objective>
Close the reupload IDOR by adding resource-level access enforcement to all 6 router_reupload.py handlers,
then delete the pre-commit exclusion that was tracking this debt.

Purpose: All 6 handlers currently enforce only `require_permission("edit_metadata")` (role-level). Any user
with the editor role can reupload, preview, or commit a dataset they do not own — regardless of visibility.
v1014 SEC-S02 established the `check_dataset_access` pattern for metadata mutations; this plan applies it
identically to all reupload paths.

The 6 handlers (confirmed by grepping `require_permission("edit_metadata")` in router_reupload.py):
  1. reupload_dataset          (line 114) — POST /{dataset_id}/reupload

  3. reupload_preview          (line 287) — POST /{dataset_id}/reupload/{job_id}/preview

  5. request_presigned_reupload (line 569) — POST /{dataset_id}/reupload/presigned
  6. complete_presigned_reupload (line 685) — POST /{dataset_id}/reupload/presigned/{job_id}/complete
