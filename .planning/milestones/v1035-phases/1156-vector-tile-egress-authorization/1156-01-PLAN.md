<objective>
Close SEC-01: the vector-tile authorization path gates on dataset `visibility` only and never consults `record_status`, so a `visibility=public` but **unpublished** (draft/ready/internal) dataset leaks its MVT feature data + a valid HMAC tile token to any anonymous caller (live-proven: 200 + 1842 bytes of MVT served to anon).

This plan makes all five vector-tile entry points enforce the same contract the **raster** path already enforces: anonymous access requires `visibility=='public' AND record_status=='published'`. Owner / admin / embed-token paths remain unchanged (no over-gating).

Purpose: Stop an active anonymous data leak (production security blocker, GitHub issue #124).
Output: A single-file change to `backend/app/processing/tiles/router.py` covering the `_DatasetMeta` field prerequisite, the two token endpoints, and the shared `_authorize_vector_tile_request` consumed by both `.pbf` routes.
</objective>
