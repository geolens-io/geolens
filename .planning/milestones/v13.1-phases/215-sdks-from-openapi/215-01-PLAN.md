## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → published artifact | Anything committed to `sdks/` ends up bundled into the eventual PyPI/npm package. License declarations, `files` whitelists, and version pins all flow through this boundary. |
| developer → generator tool | `uvx openapi-python-client@0.28.3` and `npx @hey-api/openapi-ts@0.96.1` execute third-party code at build time; pinned versions limit blast radius. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

Verified live against PyPI/npm 2026-04-27. |
| T-215-01-02 | Information disclosure | Accidental commit of secrets/build artifacts | mitigate | `.gitignore` in both SDK directories blocks `dist/`, `node_modules/`, `*.egg-info/`, `*.tgz`. The `package.json` `files` whitelist limits npm-published content to `dist/`, `LICENSE`, `README.md` — source files and configs never ship to the registry. |
| T-215-01-03 | Tampering | Wrong license declared | mitigate | Both `pyproject.toml` and `package.json` declare `Apache-2.0` explicitly; `LICENSE` files copied verbatim from repo root via `diff -q` verification (acceptance criterion). Closes ROADMAP SC#1+#2 license requirement. |
| T-215-01-04 | Spoofing | Package-name typosquatting | mitigate | Names match RESEARCH §Environment Availability registry-availability check (PyPI `geolens-sdk` available, npm `@geolens/sdk` available, `@geolens` org unclaimed). | T-215-01-05 | Repudiation | License/version mismatch between SDKs | accept | Both SDKs declare `Apache-2.0` and version `1.0.0` at scaffold time; Plan 02's `sync_sdk_versions.py` keeps versions locked to `backend/openapi.json info.version` thereafter.
