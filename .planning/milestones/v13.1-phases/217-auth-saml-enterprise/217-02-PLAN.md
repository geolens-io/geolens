NEW metadata endpoint (D-08, RESEARCH §9 item 4):
```python
@router.get("/{slug}/metadata")
async def saml_metadata(slug: str, request: Request, db: AsyncSession = Depends(get_db)):
    provider = await get_provider_by_slug(db, slug)
    if not provider or not provider.enabled or provider.provider_type != "saml":
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND)
    public_api_url = await get_public_api_url(db, request=request)
    acs_url = f"{public_api_url}/auth/saml/{slug}/acs"
    client = build_saml_client(provider, acs_url)
    from saml2.metadata import create_metadata_string
    metadata_xml = create_metadata_string(configfile=None, config=client.config, sign=False)
    return Response(content=metadata_xml, media_type="application/samlmetadata+xml")
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → ACS endpoint (`POST /auth/saml/{slug}/acs`) | Untrusted SAMLResponse XML enters here. Signature validation is the only thing standing between an attacker and arbitrary user impersonation. |
| Browser → metadata endpoint (`GET /auth/saml/{slug}/metadata`) | Public by design (IdPs poll); zero secrets cross this boundary. SP signing key is not used in V1 (D-15). |
| OAuth provider DB row → SAML config builder | `idp_certificate` is Fernet-encrypted at rest; `decrypt_secret()` runs in process memory. Cert is the trust anchor for assertion-signature verification. |
| Extension registry → FastAPI app | `register_extensions()` populates `registry['_routers']`; the app mounts each router. Misregistration could expose SAML routes in community (defense: `Depends(require_enterprise)` on the router). |
