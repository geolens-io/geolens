<objective>
Close SEC-FU-01 and SEC-FU-08 — observability + test-infra additions on top of Phase 1061's HIGH severity fixes.

**SEC-FU-01 (STAC 5xx-mutation test fixtures):** Phase 1061 Plan 01 closed the STAC visibility bypass and added a happy-path regression suite (`test_stac_visibility.py`). The follow-up adds **5xx-mutation** fixtures: a pytest fixture that can force `apply_visibility_filter` to raise, so we can assert the route returns 5xx WITHOUT leaking private record IDs, titles, or property keys in the error body. This pins the "no information disclosure on backend failure" contract the audit's S01 spec assumed.

**SEC-FU-08 (column DDL change log / owner notification surface):** Phase 1061 SEC-S03 wired `audit_emit(AuditEvent(action="layer.add_column", ...))` (and `rename_column` / `alter_column_type` / `drop_column`) into the 4 column-DDL endpoints, so the `audit_logs` table already carries the events. The follow-up adds an **owner-facing feed endpoint** that filters those events for a single dataset and enforces `check_dataset_access` so only the dataset's owner / granted users / admins can read its DDL history. The CONTEXT.md "pg_audit OR per-table change log" choice resolves to the lowest-friction option: no new extension, no new table, no new migration — just a query + endpoint on top of the existing `audit_logs` table.

Purpose:
- SEC-FU-01: gives `e2e/sec-audit.spec.ts` (and the backend tests it mirrors) a way to drive the 5xx code path without taking down the database. Closes the documented spec gap.
- SEC-FU-08: gives dataset owners an API surface to detect editor-initiated schema changes after-the-fact. Combined with SEC-S03's ownership gate, this closes the "editor mutates schema silently" residual risk the audit flagged.

Output:
