<objective>
Close SEC-FU-03 (frontend ESLint `react/no-danger`) and SEC-FU-04 (GDAL Authorization header base64url charset filter) — two input-validation hardening additions on opposite ends of the stack.

**SEC-FU-03 (react/no-danger ESLint rule):** The audit S14 verified `dangerouslySetInnerHTML` has zero hits in `frontend/src/**`, but the popup-template surface (`frontend/src/lib/popup-template.ts`) explicitly bans the pattern in a comment rather than via lint. Installing `eslint-plugin-react` and enabling `react/no-danger: 'error'` makes the ban machine-checkable. Includes a regression-fixture file (mirroring the SEC-S14 `sec-s14-eslint-regression.skip.ts` pattern from Phase 1062 Plan 06) so the rule activation can be verified by renaming the fixture from `.skip.tsx` to `.tsx` and running lint.

**SEC-FU-04 (GDAL Authorization base64url charset):** `backend/app/processing/ingest/ogr.py` line ~644 composes `GDAL_HTTP_HEADERS=f"Authorization: Bearer {token}"` as an environment variable for the ogr2ogr subprocess. If a `token` ever contains a literal `\r\n` (CRLF), libcurl could parse the injection as an additional HTTP header (header smuggling). Tokens are nominally OAuth bearer tokens (base64url), so filtering to that charset (`A-Z`, `a-z`, `0-9`, `_`, `-`, `=`) is a tight defense-in-depth restriction with no legitimate-traffic impact. The existing Phase 1061 SEC-S04 fix added `GDAL_HTTP_FOLLOWLOCATION=NO` to the same env block — this plan extends the env-composition function to also sanitize the token.

Purpose:
- SEC-FU-03: locks the popup-template ban + the audit's "no dangerouslySetInnerHTML sinks" invariant via lint, preventing future regressions in tandem with SEC-S14 (httpOnly migration plan documented separately).
- SEC-FU-04: closes the CRLF header-smuggling defense-in-depth gap flagged by Subagent A in the audit (line 535). Pure additive — clean base64url tokens pass through unchanged.

Base64url charset definition (RFC 4648 §5):
- Letters: A-Z, a-z
- Digits: 0-9
- URL-safe substitutes: `_` (replaces `+`), `-` (replaces `/`)
- Padding: `=` (often dropped in JWT; allow for compatibility)
- Plus the `.` character — required because JWTs are dot-separated base64url segments (`header.payload.signature`)
- Total allowed set: `^[A-Za-z0-9._\-=]+$`

       def _sanitize_authorization_token(token: str | None) -> str | None:
           """SEC-FU-04 (sec-audit-20260519.md line 535): pin Authorization bearer token to base64url charset
           before composing GDAL_HTTP_HEADERS. CRLF in a token would let an attacker inject additional

           Returns the token unchanged if it matches the charset; raises ValueError otherwise.
           None passes through (caller's no-token path)."""
           if token is None:
               return None
           if not token or len(token) < 8:
