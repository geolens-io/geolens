<objective>
Close SEC-S15 (MEDIUM, CVSS 4.3) and SEC-S16 (MEDIUM, CVSS 4.3). The current access JWT (`backend/app/modules/auth/service.py:42-52`) carries only `sub`/`username`/`exp`/`iat` — no `jti`, no `token_version` — so `POST /auth/logout/` (which only revokes the opaque refresh token) leaves the access JWT silently valid until `exp` (default 15 min). Password validation at `backend/app/modules/auth/schemas.py:27` (and the admin sibling at `backend/app/modules/admin/schemas.py:23`) is `Field(min_length=8)` only — `admin`, `geolens1`, `password` all pass.

This plan adds the missing revocation primitive (`User.token_version` column + JWT claim) and a single shared password-policy module gated at all four password entry points (register, self-change-password, admin create-user, admin convert-saml-to-local). Both fixes live in the auth module so they ship as one cohesive plan with shared test scaffolding.

Purpose:
- SEC-S15 closes the "logout doesn't actually log out the access JWT" gap. Same primitive lets a future SEC-FU phase wire an admin "revoke all sessions" surface without further model changes.
- SEC-S16 closes the "min 8 chars, no class diversity, no denylist" gap. Defensible default of 12 chars + 3-of-4 class diversity matches the audit recommendation; config knobs allow operators to relax in dev/test.

Output:
- Alembic migration `0019_users_token_version.py` adding `catalog.users.token_version INTEGER NOT NULL DEFAULT 1` with a clean `downgrade()`.
- `User.token_version` column declared on the SQLAlchemy model.
- `AuthService.create_access_token` payload extended with `jti=uuid4().hex` and `token_version=user.token_version` (requires loading the User row at issuance — already loaded at login per `router.py:79`, so wire through).
- `AuthService.revoke_all_tokens(user_id)` helper that (a) revokes refresh tokens (existing behavior) and (b) `UPDATE users SET token_version = token_version + 1 WHERE id = :user_id`. Returns the new version.
- `get_current_user` / `get_optional_user` in `dependencies.py` extended to compare `payload.get("token_version", 0)` against the loaded `user.token_version` and 401 if stale.
- `POST /auth/logout/` calls `revoke_all_tokens` instead of `revoke_all_refresh_tokens`. `POST /auth/change-password/` calls it too so password rotation kicks other devices off.
- New `backend/app/modules/auth/password_policy.py` with `validate_password_complexity(password: str, settings: Settings) -> None` (raises `ValueError` on weak input). Implements `min_length` + N-of-4 class diversity (lower, upper, digit, symbol).
- `UserCreate` (register), `ChangePasswordRequest`, `AdminUserCreate`, `SamlToLocalConversion` schemas all gain a `@field_validator("password" | "new_password")` that delegates to the shared validator.
- `Settings` (config.py) gains `password_min_length: int = 12` and `password_require_classes: int = 3` (with `Field(ge=1, le=4)` for the class-count knob).
- Backend pytest in `test_jwt_revocation.py` (4+ tests) and `test_password_policy.py` (8+ tests covering all four entry points + config override).
</objective>
