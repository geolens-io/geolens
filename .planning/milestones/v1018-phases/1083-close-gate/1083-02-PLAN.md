<objective>
Live Playwright MCP smoke check on 5 surfaces of the running GeoLens stack (`localhost:8080`) to confirm no console errors or failed network requests after the v1018 TD-01..07 fixes have landed. This plan is **autonomous: false** — Playwright MCP browser tools are orchestrator-scoped (per v1010.1 + v1011 + v1017 close-gate precedent in MEMORY.md). Purpose: Headless e2e smoke gates (e2e:smoke:builder) and unit tests are necessary but not sufficient — they miss boundary regressions, console errors, and network-level issues that only surface in a live browser session against a real stack. v1010.1 and v1011 both caught P0 regressions via live MCP that headless gates missed. Live MCP smoke is now a justified pre-tag gate for any milestone touching backend state or build pipelines.

Output: `.planning/phases/1083-close-gate/1083-02-SUMMARY.md` with per-surface PASS/FAIL results, console output excerpts for any anomalies, and an overall verdict (5/5 PASS → CLOSE; any FAIL → surface to orchestrator for triage).

If a surface regression is discovered during this MCP smoke, the orchestrator must decide: (a) fix inline and re-tag (delete + recut both tags at the fixed commit), or (b) defer to v1018.1 with rationale. The v1011 pattern is to fix inline when fix is <1 hr and safe; defer when fix risks scope creep.

### 5 surfaces to verify (per 1083-CONTEXT.md decisions; mirrors v1017 close gate per 1079-SUMMARY.md)

1. **Catalog list** — `http://localhost:8080/` (or `/catalog` if root redirects). Confirm dataset cards/list render; no console errors; no failed network requests (`/api/datasets`, `/api/maps`, `/api/auth/...`).

3. **Map builder** — navigate to map builder for any map. Confirm canvas mounts, base layer loads, layer editor panel opens; no console errors; tile requests succeed.
4. **Map viewer** — navigate to a saved map's viewer route (read-only view). Confirm map loads with all layers; no console errors; tile + style requests succeed.
5. **Login / auth surface** — log out (clear `geolens-auth` localStorage OR click logout if affordance exists), navigate to `/login`, log back in with admin credentials. Confirm redirect to catalog post-login; no console errors during the round-trip.

### Pass criteria per surface

- **PASS:** Page renders; browser console shows zero `error`-level messages (warnings are acceptable if pre-existing in v1017); network panel shows zero failed requests (4xx/5xx) attributable to the surface (NOT counting pre-existing third-party tracker failures or known v1017-deferred issues).
- **FAIL:** Page fails to render, OR a new console error appears that wasn't in v1017 baseline, OR a backend API call returns 5xx, OR navigation/login round-trip breaks.

### MCP tools the orchestrator drives

Per v1011's MEMORY.md note, the orchestrator uses Playwright MCP (chrome devtools tools): navigate, wait for load, screenshot if needed, check console messages, check network requests. Do NOT delegate to a subagent — subagents lack the MCP tool surface.

- `accessibility.spec.ts:151` axe failures (per MEMORY.md v1011 pre-existing e2e failures note) — out of scope for live MCP; only matters for headless e2e

1. Capture the console error text and network request URL+status verbatim.
2. Reproduce on the pre-v1018 tree (`git stash` any uncommitted work, `git checkout c968392b` which is the v1017 tag commit, restart stack, retest).
3. If the regression reproduces on c968392b: NOT a v1018 regression → mark PASS-with-note + create `.planning/quick/2026-05-21-{slug}.md` for v1019 triage.
4. If the regression does NOT reproduce on c968392b: v1018 introduced it → surface to orchestrator with the failing surface, expected vs actual, and a candidate root cause. Orchestrator decides: fix inline + re-tag, or defer to v1018.1.

| MCP browser session → console / network state | Console errors and network failures are the canonical signal — if MCP can't read them reliably (e.g., due to a flaky devtools connection), the smoke is invalid |
