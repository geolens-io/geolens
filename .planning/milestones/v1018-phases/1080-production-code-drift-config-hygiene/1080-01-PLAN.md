use the `# broad: <reason>` style, e.g.:

  except Exception as archive_exc:  # broad: archive is best-effort; S3/local I/O can fail for any reason

Justification rationale for the two sites in `_job_phase_session`:
- Both `yield session, ...` blocks hand control to arbitrary caller code inside an async
  context manager. ANY exception from the caller must roll back the in-flight session
  before re-raising, or the session leaks a dirty connection back to the pool. The catch
  is intentionally broad because the caller's failure modes are unknown to the bracket.
