Close TD-07: honour the documented `database_ssl_mode == "disable"` configuration surface. Low operational risk (production never sets `disable` per the v1017 audit note), but the disable branch is part of the public config contract and must be honoured. Pinning behaviour with a 3-case unit test prevents regression and serves as living documentation for the three branches.

Output: A one-line addition to the `disable` branch in `backend/app/core/config.py:database_connect_args` plus updated and added tests in the existing `backend/tests/test_config.py::TestDatabaseConnectArgs` class.
