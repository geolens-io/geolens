<objective>
Close SEC-FU-05, SEC-FU-06, and SEC-FU-07 — three Postgres / PostGIS input-validation hardenings clustered on shared helpers and service modules.

**SEC-FU-05 (STAC intersects max_length):** The STAC `/search` GET handler accepts `intersects: str | None = Query(None, ...)` with no length cap (router.py:1101-1103). A multi-megabyte GeoJSON polygon string would force a JSON parse + ST_GeomFromGeoJSON evaluation before any DB-side index could short-circuit — a cheap-to-write DoS amplifier. The fix is a single `Query(None, max_length=10000, ...)` change. 10000 chars is large enough for a complex polygon (~150 vertices at 2-decimal-place lat/lon) and small enough that JSON parsing stays inexpensive. The POST handler with `StacSearchBody.intersects: dict` is bounded by uvicorn's 1MB request limit and doesn't need the explicit cap.

**SEC-FU-06 (parse_bbox math.isfinite):** `parse_bbox` in `backend/app/modules/catalog/features/service.py:47-68` converts string coordinates via `float(p)` — Python's `float()` accepts `"nan"`, `"inf"`, `"-inf"` returning the IEEE 754 special values. PostGIS / PostgreSQL handle these inconsistently (some functions raise, others silently return malformed geometries). Add a `math.isfinite()` check on each coordinate; raise `ValueError` if any is non-finite. The helper is shared between OGC router (line 309) and search/export routers — single fix benefits all callers.

**SEC-FU-07 (ILIKE escape in service_crud.py):** `backend/app/modules/catalog/maps/service_crud.py:140-147` builds an ILIKE pattern from user input via `pattern = f"%{search}%"`. The `%` and `_` characters carry their LIKE-wildcard meaning when un-escaped — a malicious search of `"%"` matches every row. The escape pattern `.replace("%", r"\%").replace("_", r"\_")` is already in use at `service_public.py:407-409` from prior phases; lift it into `service_crud.py:140-147`. CONTEXT.md also referenced `service_collections.py:29-35` but that file does not exist in the current tree (was likely removed in v13.6 Catalog Maps/Search Service Decomposition or renamed); confirm during execution by grep and skip with a note in SUMMARY if absent.

Purpose: All three closures harden input handling at the helper / service layer. None touch routes for handlers, RBAC, or authentication — they reduce the blast radius of malformed or hostile inputs reaching shared helpers.
