## Component Inventory

### New: Export Split Button

**Location:** AuditLogViewer CardHeader, right side, between CardTitle and DataTableSearch.

**Structure:**
```
[CardTitle]  [ExportSplitButton]  [DataTableSearch]
```

**Split button anatomy:**
| Part | Component | Variant | Size |
|------|-----------|---------|------|
| Primary action | `<Button>` | `default` | `sm` |
| Divider | Border-left on chevron trigger | `border-l border-primary-foreground/20` | - |
| Dropdown trigger | `<Button>` inside `<DropdownMenuTrigger>` | `default` | `sm` |
| Dropdown menu | `<DropdownMenuContent>` | default | default |
| JSON option | `<DropdownMenuItem>` | default | default |

**Visual spec:**
- Primary button: `Download` icon (lucide) at 14px (h-3.5 w-3.5) + "Export CSV" label
- Primary button has `rounded-r-none` to join with chevron
- Chevron trigger: `ChevronDown` icon at 14px (h-3.5 w-3.5), `rounded-l-none border-l-0 px-1.5`
- Both halves share the same `default` variant (primary blue) so the split button reads as one unit
- Dropdown contains single item: "Export as JSON"

**States:**
| State | Behavior |
|-------|----------|
| Default | Split button visible, enabled |
| Downloading | Primary button shows `Loader2` spinner (animate-spin) replacing `Download` icon. Label changes to "Exporting...". Both halves disabled. |
| Success | Button returns to default state. No toast needed -- browser download notification is sufficient. |
| Error | Toast notification via existing toast system: "Export failed. Try again or narrow your date range." |
| Community edition | Entire split button is not rendered (hidden via `useEdition()` check) |
| No filter results | Button remains enabled. Export produces a valid file with header-only CSV or empty JSON array `[]`. |

**Interaction:**
- Primary click: triggers CSV download with current filter state
- Chevron click: opens dropdown with JSON option
- Keyboard: Tab reaches primary button, then chevron. Enter/Space activates each. Arrow keys navigate dropdown items.

### Enterprise Gating

- `useEdition()` hook controls render: `{isEnterprise && <ExportSplitButton ... />}`
- No placeholder, tooltip, or "upgrade" prompt in community edition -- the button simply does not exist
- Matches Phase 207 branding toggle pattern (D-05, D-06)

## Layout Changes

This phase makes zero layout changes. The split button is inserted into the existing `CardHeader` flex row:

**Before:**
```
<div className="flex items-center justify-between">
  <CardTitle>...</CardTitle>
  <DataTableSearch ... />
</div>
```

**After:**
```
<div className="flex items-center justify-between gap-2">
  <CardTitle>...</CardTitle>
  <div className="flex items-center gap-2">
    {isEnterprise && <ExportSplitButton ... />}
    <DataTableSearch ... />
  </div>
</div>
```

The CardHeader right side gains a wrapping flex container with `gap-2` (8px) between the export button and search input.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Split button label | Primary button has visible text "Export CSV". Chevron trigger gets `aria-label="Export format options"` (`admin:audit.export.formatOptions`) |
| Dropdown | Radix DropdownMenu handles aria-haspopup, aria-expanded, focus management |
| Loading state | Disabled attribute on both button halves during download. `aria-busy="true"` on primary button. |
| Keyboard | Tab order: primary button -> chevron trigger. Enter/Space activates. |
