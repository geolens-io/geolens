# Plan 208-02: Frontend Split Button UI, Blob Download, i18n

## Objective

Add an export split button to the AuditLogViewer that triggers CSV (primary) or JSON (dropdown) downloads via the streaming backend endpoints from 208-01. Enterprise-gated via useEdition().

## Context

- @frontend/src/components/admin/AuditLogViewer.tsx -- existing viewer to extend
- @frontend/src/api/admin.ts -- admin API client
- @frontend/src/hooks/use-edition.ts -- useEdition() hook
- @frontend/src/api/client.ts -- apiFetch and auth headers
- @.planning/phases/208-audit-log-export/208-UI-SPEC.md -- visual spec
- @.planning/phases/208-audit-log-export/208-CONTEXT.md -- phase decisions

## Tasks

### Task 1: Add export API functions and i18n keys
type="auto"

1. Add `exportAuditLogs(format, filters)` function to `frontend/src/api/admin.ts` that:
   - Builds query params from current filters (action, date_from, date_to, search)
   - Uses `fetch` directly (not apiFetch) since we need blob response, but includes auth header from useAuthStore
   - Returns a Blob for browser download
   - Endpoint: GET /admin/audit-logs/export/{format} where format is 'csv' or 'json'

2. Add i18n keys to `frontend/src/i18n/locales/en/admin.json` under audit.export:
   - csv: "Export CSV"
   - json: "Export as JSON"
   - loading: "Exporting..."
   - errorTitle: "Export failed"
   - errorBody: "Something went wrong. Try again or narrow your date range."
   - formatOptions: "Export format options"

**Done when:** Export function exists, i18n keys added for English.

### Task 2: Create ExportSplitButton component and integrate into AuditLogViewer
type="auto"

Create `frontend/src/components/admin/ExportSplitButton.tsx`:
- Split button: primary "Export CSV" + ChevronDown dropdown with "Export as JSON"
- Uses Button, DropdownMenu, DropdownMenuTrigger, DropdownMenuContent, DropdownMenuItem
- Loading state: Loader2 spinner, "Exporting..." text, both halves disabled
- Error: toast via sonner
- Downloads file as blob, creates object URL, triggers anchor click
- Filename: `audit-export-YYYY-MM-DD.{csv|json}`

Integrate into AuditLogViewer.tsx:
- Import useEdition, conditionally render ExportSplitButton
- Place between CardTitle and DataTableSearch per UI spec
- Pass current filter state as props

**Done when:** Split button renders in enterprise mode, triggers downloads, handles loading/error states.

## Verification

- Export button visible in AuditLogViewer CardHeader (enterprise mode)
- CSV and JSON download functions trigger correct endpoints with filters
- Loading spinner shows during download
- Error toast on failure
- Button hidden in community edition
- i18n keys present

## Success Criteria

1. Split button renders correctly with CSV primary action and JSON dropdown
2. Downloads trigger browser file save with correct filename format
3. Current filters applied to export request
4. Enterprise gating hides button in community edition
5. Loading and error states handled properly
