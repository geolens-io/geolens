# Plan 208-01: Backend Streaming Export Endpoints

## Objective

Add streaming CSV and JSON export endpoints to the audit log router, gated behind enterprise edition. Endpoints reuse existing filter parameters and stream results from a DB cursor to avoid memory issues with large exports.

## Context

- @backend/app/audit/router.py — existing audit log endpoint
- @backend/app/audit/service.py — query_audit_logs service
- @backend/app/audit/models.py — AuditLog model
- @backend/app/extensions/guards.py — require_enterprise dependency
- @.planning/phases/208-audit-log-export/208-CONTEXT.md — phase decisions

## Tasks

### Task 1: Add streaming export service function
type="auto"

Add a new async generator function `stream_audit_logs` to `backend/app/audit/service.py` that:
- Accepts the same filter params as `query_audit_logs` (minus skip/limit)
- Builds the same filtered query but without pagination
- Uses `stream_scalars()` to yield rows one at a time from a server-side cursor
- Orders by created_at descending

**Done when:** Function exists, accepts filters, yields AuditLog rows via async generator.

### Task 2: Add CSV and JSON export endpoints
type="auto"

Add two new endpoints to `backend/app/audit/router.py`:

1. `GET /admin/audit-logs/export/csv` — StreamingResponse with `text/csv` content type
   - Content-Disposition: `attachment; filename="audit-export-YYYY-MM-DD.csv"`
   - CSV columns: timestamp, username, action, resource_type, resource_id, ip_address, details
   - Details column serialized as JSON string
   - Uses `stream_audit_logs` generator

2. `GET /admin/audit-logs/export/json` — StreamingResponse with `application/json` content type
   - Content-Disposition: `attachment; filename="audit-export-YYYY-MM-DD.json"`
   - Standard JSON array format `[{...}, {...}]`
   - Uses `stream_audit_logs` generator

Both endpoints:
- Require `manage_settings` permission (same as list endpoint)
- Require enterprise edition via `require_enterprise` dependency
- Accept same filter query params: action, resource_type, date_from, date_to, search

**Done when:** Both endpoints return streaming responses with correct headers and content format.

## Verification

- Endpoints registered at correct paths
- Enterprise gating applied (require_enterprise dependency)
- Permission check applied (require_permission manage_settings)
- StreamingResponse used for memory-efficient delivery
- CSV has correct column ordering and headers
- JSON produces valid array format

## Success Criteria

1. Two new export endpoints exist under /admin/audit-logs/export/
2. Both are enterprise-gated and admin-only
3. Streaming delivery prevents memory issues on large exports
