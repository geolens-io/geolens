    - Enterprise gating via `Depends(require_enterprise)` on the router level (per D-08, D-09)

    **GET /{slug}/login** endpoint:
    - Look up provider by slug via `get_provider_by_slug(db, slug)`
    - Validate: provider exists, enabled, provider_type == "saml" -- else 404
    - Build ACS URL: `{public_api_url}/auth/saml/{slug}/acs`
    - Call `build_saml_client(provider, acs_url)` to get Saml2Client
    - Call `client.prepare_for_authenticate()` to get (reqid, info)
    - Extract redirect URL from `dict(info["headers"])["Location"]`
    - Return `RedirectResponse(url=redirect_url, status_code=302)`

    **POST /{slug}/acs** endpoint (per D-02):
    - Look up provider by slug, validate exists/enabled/saml type
    - Parse form data: `form_data = await request.form()`, `saml_response = form_data.get("SAMLResponse")`
    - Build ACS URL and Saml2Client
    - Call `client.parse_authn_request_response(saml_response, BINDING_HTTP_POST)` -- this validates signature, timestamps, audience (per D-07). Wrap in try/except.
    - Check replay: `replay_cache.check_and_record(authn_response.assertion.id)` -- if False, reject
    - Extract user attributes from `authn_response.ava` dict:
      - email: check keys `["email", "mail", "emailAddress", "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"]`
      - name: check keys `["displayName", "name", "cn", "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name"]`
      - sub: `authn_response.get_subject().text` (NameID)
    - Build userinfo dict: `{"sub": subject, "email": email, "name": name}`
    - If provider.group_claim is set, extract groups from attributes
    - Call `find_or_create_oauth_user(db, provider, userinfo, {})` (per D-02, same flow as OIDC)
    - Record `user.last_login_at = func.now()`
    - Issue JWT using same pattern as oauth_callback: `AuthService.create_access_token()`, `create_refresh_token()`
    - Redirect to `{frontend_url}/oauth/callback#token={access}&refresh_token={refresh}&expires_in={seconds}`
    - On any exception: redirect to `{frontend_url}/oauth/callback#error={encoded_message}`
    - Use specific error messages from UI-SPEC (invalid signature, expired, audience mismatch, generic)
