# Phase 1027 Plan: Server Cluster Tile Contract

**Status:** Complete
**Milestone:** v1006 Large Dataset Cluster Scaling
**Requirements:** SCL-01, SCL-02, SCL-03, SCL-04, SCL-05

## Goal

Add a backend server-side cluster tile contract for vector point datasets without changing saved-map, map-layer, dataset, or record schemas, and without changing normal vector tile behavior.

## Scope

- Add a new cluster tile route under the existing tile router.
- Reuse the current vector tile data-table parsing, coordinate validation, visibility, HMAC signature, and embed-token authorization model.
- Keep cluster options as query parameters and cache-key inputs, not persisted map/layer fields.
- Emit MapLibre-compatible cluster properties for future frontend routing and interaction work.
- Keep SQL bounded by tile envelope, feature budgets, validated table identifiers, and controlled empty/error responses.

## Implementation

1. Factor vector tile route helpers for table parsing, tile coordinate validation, dataset metadata lookup, authorization, and response headers.
2. Add `GET /tiles/clusters/data.{table}/{z}/{x}/{y}.pbf` with `cluster_radius` and `cluster_max_zoom` query parameters.
3. Add a bounded PostGIS MVT query for point clusters with cluster count, abbreviated count, stable cluster id, expansion zoom, and source gid properties.
4. Add cluster-specific tile cache keys that include cluster options while leaving normal vector tile cache keys unchanged.
5. Add backend tests for route success, point-only validation, private auth, embed-token access, cache-key separation, and SQL property shape.

## Acceptance

- No schema migrations or new persisted fields.
- Public, signed private, and embed-token cluster tile access uses the same authorization rules as normal vector tiles.
- Non-point vector datasets reject cluster tiles before the expensive PostGIS path.
- Empty tiles return controlled `204` responses and SQL/query errors return controlled service errors.
- Focused backend ruff and tile tests pass against Docker Postgres.
