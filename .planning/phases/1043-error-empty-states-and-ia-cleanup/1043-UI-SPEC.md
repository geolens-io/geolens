## Scene Transition State Preservation (POL-18)

Scene transitions in the builder: default scene → basemap-group → basemap-sublayer → back, and default → settings → back.

**Mechanism:** `LayerEditorPanel` currently resets `filterOpen`, `labelsOpen`, `sourceOpen`, and `confirmingDelete` on every `layer.id` or `editorScene` change (lines 165-170). This is correct for section state but currently does NOT preserve scroll position.

**Scroll preservation contract:**

The scrollable body is `<div data-testid="layer-editor-body" className="flex-1 overflow-y-auto">` at `LayerEditorPanel.tsx:286-289`.

```tsx
// Add useRef to track scroll position
const bodyRef = useRef<HTMLDivElement>(null);
const savedScrollTopRef = useRef<number>(0);

// Before navigating away (when editorScene changes), save scroll
useEffect(() => {
  return () => {
    if (bodyRef.current) {
      savedScrollTopRef.current = bodyRef.current.scrollTop;
    }
  };
}, [editorScene]);

// Restore scroll on remount / scene return
useEffect(() => {
  if (bodyRef.current && savedScrollTopRef.current > 0) {
    bodyRef.current.scrollTop = savedScrollTopRef.current;
  }
}, [layer.id, editorScene]);
```

Attach `ref={bodyRef}` to the `data-testid="layer-editor-body"` div.

**Focus preservation contract:**

When navigating from basemap-sublayer back to basemap-group (via breadcrumb click), focus should return to the basemap group header button. The `onBreadcrumbClick` handler in the caller (`UnifiedStackPanel` or `MapBuilderPage`) fires the navigation. After the scene change, focus is currently lost.

Pattern: Use a `useEffect` that focuses the panel's breadcrumb header button after scene changes that go "back" (from sublayer to group). The caller sets `editorScene` from `basemap-sublayer` to `basemap-group` — the panel detects this transition and focuses its header element.

```tsx
const headerRef = useRef<HTMLButtonElement>(null);

useEffect(() => {
  // If transitioning back from sublayer to group, restore focus to header
  if (editorScene === 'basemap-group' && prevEditorScene === 'basemap-sublayer') {
    headerRef.current?.focus();
  }
  prevEditorScene = editorScene;
}, [editorScene]);
```

Implementation note: `prevEditorScene` is tracked via a ref (`useRef`), not state, to avoid re-renders. The pattern is ref-stored previous value: `const prevSceneRef = useRef(editorScene); useEffect(() => { prevSceneRef.current = editorScene; }, [editorScene]);`

## What to Avoid

- **New design tokens.** Hard constraint from CONTEXT.md. Motion tokens already landed in Phase 1042.
- **Touching Phase 1044 scope.** Do not fill `de/es/fr` locale strings. Do not author the Playwright UAT spec. Do not run the a11y verification pass.
- **Modifying the saved-map shape.** Hard constraint.
- **Touching public viewer / shared / embed surfaces.** Hard constraint.
- **New backend endpoints.** Hard constraint.
- **Red backgrounds on recoverable errors.** Use `text-destructive` on the icon only; message body uses `text-foreground`. The error banner background is `--surface-0` (transparent / panel background).
- **Adding a "no columns" empty state to sections that already have one.** `LayerFilterEditor` and `LabelEditor` already handle their own no-columns states via `filters.noColumns` and `labels.noColumns` — do not duplicate.
- **Scroll position reset on tab/section toggle.** Only reset scroll when `layer.id` changes (a different layer was selected), not when `filterOpen`/`labelsOpen`/`sourceOpen` changes.
- **Hardcoded hex or rgb values.** All color must reference `var(--*)` tokens. Exception: the destructive hover `oklch(0.97_0.02_27)` is a computed value matching `hover:bg-[oklch(0.97_0.02_27)]` already established in the codebase for destructive ghost buttons — keep it consistent.
