<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network ↔ TanStack Query cache | DatasetSearchPanel results query is read-only catalog data; retry triggers an authenticated refetch through the existing API client. No new endpoint, no new credential surface. |
| user keyboard ↔ destructive alertdialog | AUD-09 changes only initial focus; the destructive action still requires explicit user click on Delete. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1043-01 | Tampering | invalidateQueries call | accept | Read-only refetch of an already-authenticated query; no new mutation surface, no new auth path. queryKey is locally constructed from existing state (debouncedQuery, recordType) — not user-controlled JSON. |
| T-1043-02 | Denial of Service | Retry button click loop | accept | TanStack Query's invalidateQueries is debounced by the underlying useQuery's staleTime/refetchOnMount. A user spam-clicking Try again triggers at most one in-flight fetch at a time. No backend rate-limiting concern beyond what already exists for the search endpoint. |
| T-1043-03 | Repudiation | Destructive Delete confirm | mitigate | AUD-09 autoFocus on Keep layer reduces the chance of an accidental Enter-key delete. The Delete button still requires intentional click (no Enter-to-confirm change). Audit-logging of layer deletion is unchanged (handled in use-builder-layers via existing PATCH endpoint). |
| T-1043-04 | Elevation of Privilege | search.retry i18n key injection | accept | Static string literal added to en/builder.json — no user-supplied content reaches i18n.t() argument. defaultValue is the same hard-coded English string. |
</threat_model>
