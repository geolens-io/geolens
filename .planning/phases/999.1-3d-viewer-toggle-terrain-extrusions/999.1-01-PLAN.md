<objective>
Add the `is_dem` backend infrastructure: Alembic migration, RasterAsset model field, DEM auto-detection heuristic in `extract_raster_metadata`, ingest integration, terrain tile URL modification, dataset metadata PATCH support, and SharedLayerResponse serialization.

Purpose: The viewer needs to know which raster layers are DEMs so it can create a `raster-dem` terrain source with the correct terrainrgb-encoded tile URL. This plan provides the complete backend surface for that.

Output: `is_dem` column on `raster_assets`, auto-set during ingest, PATCH-able by owner/admin, exposed on `SharedLayerResponse`, and terrain tile URL uses `algorithm=terrainrgb` for DEM datasets.
</objective>

def _is_float_dtype(dtype: str) -> bool:
    """Check if a raster dtype string represents a floating-point type."""
    return any(f in dtype.lower() for f in _FLOAT_DTYPES)
