## Accomplishments

- Registered `_height_column` in `CUSTOM_PAINT_PROPS` so it is stripped before MapLibre receives fill layer paint — prevents style spec validation errors
- Added conditional `fill-extrusion` companion layer in `addLayers` when `_height_column` is present in paint config, using `['coalesce', ['to-number', ['get', column], 0], 0]` for safe null handling
- Extended `syncPaint`, `syncVisibility`, and `getLayerIds` to handle the extrusion companion layer including the MapLibre v5 `triggerRepaint()` workaround
- Added `is_dem?: boolean` and `tile_url?: string` to `SharedLayerResponse` TS interface (required by Plan 03 ViewerMap terrain logic)
- Wrote 7 TDD test cases covering: extrusion added with height column, not added without, coalesce expression correct, minzoom 14, syncVisibility toggles extrusion, _height_column stripped from fill paint, getLayerIds returns 3 IDs

## Decisions Made

- `getLayerIds` always returns 3 IDs regardless of whether `_height_column` is configured — this gives callers a consistent cleanup API; MapLibre's `getLayer` check in syncVisibility handles non-existent layers gracefully
- `_height_column` is read from `rawPaint` at the top of `addLayers` before `stripCustomProps` is called — order is critical
- `fill-extrusion-opacity` capped at `Math.min(opacity, 0.85)` — extrusions at full opacity obscure terrain; 0.85 keeps vertical gradient readable
