## Layout Contract

This phase ships a **three-column builder layout**. All measurements are locked.

| Column | Width | Condition |
|--------|-------|-----------|
| Sidebar (layer stack) | 340px | Always mounted |
| LayerEditorPanel flyout | 380px | When a layer row is selected |
| Map canvas | `1fr` (fills remainder) | Always |

The flyout is a **layout column**, not an overlay or popover. The sidebar remains fully mounted while the flyout is open; cross-layer comparison must be possible without navigating away.

### Grid definition (CSS)

```css
/* No layer selected */
.builder-body { display: grid; grid-template-columns: 340px 1fr; }

/* Layer selected */
.builder-body.editor-open { grid-template-columns: 340px 380px 1fr; }
```

### Responsive breakpoints (locked, sketch 008A)

| Viewport | Sidebar | Editor | Map |
|----------|---------|--------|-----|
| ≥ 1100px | 340px full list | 380px (when selected) | 1fr |
| 800–1099px | 64px icon rail | 380px (when selected) | 1fr (~620px @ 1024) |
| < 800px | 64px icon rail | hidden | 1fr |

At <1100px the sidebar collapses to a 64px icon rail (separate DOM node, not CSS restyling). At <800px the editor hides entirely; re-selecting a layer restores it. Mobile <480px is out of scope for this phase.

## Spacing Scale

Declared values (multiples of 4, sourced from existing tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, caret column, inline padding |
| sm | 8px | Row padding top/bottom, section padding top/bottom, pill gaps |
| md | 16px | Default element spacing, sidebar header padding (14px is used in sketch — treated as ~md) |
| lg | 24px | Section breaks |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Rail width (exception — fixed layout value, not a spacing token) |

Exceptions:
- Sidebar header padding: `14px 14px 10px` (sketch-locked; nearest md token)
- Row grid gap: 6px (between row cells — sketch-locked)
- Stack scroll padding bottom: 80px (reserved for future "drag here to ungroup" affordance)
- Row opacity slider width: 60px (fixed, sketch-locked)
- Type icon size: 22×22px; rail icon size: 26×26px inside 40×40px touch target

- Settings icon button: Lucide `Settings` (⚙), 22×22px touch area, `title="Settings"` tooltip
- "＋ Add data" primary button: Lucide `Plus` icon + "Add data" label, `--primary` background

### 2. Layer stack rows
Grid: `16px 14px 22px 22px 1fr 60px 22px` with 6px gap, 8px vertical padding.

| Cell | Width | Element | Behavior |
|------|-------|---------|----------|
| Caret | 16px | `▸` on groups; `visibility:hidden` on non-groups | 90° rotation on expand |
| Grip | 14px | `⋮⋮` drag handle | 35% opacity rest; 70% on row hover |
| Eye | 22px | `●` / `○` visibility toggle | Click stops propagation (does NOT select row) |
| Type icon | 22px | 22×22 rounded square; icon + color per type | Read-only; reflects active render mode |
| Name | 1fr | Layer name, ellipsis-truncated | Click selects row + opens editor |
| Opacity | 60px | Range slider `min=0 max=100` | Primary-colored thumb; no numeric input |
| Kebab | 22px | `⋯` menu trigger | 0% opacity rest; 100% on hover or selected |

| Type | Glyph | Background token | Foreground token |

| State | Visual |
|-------|--------|

## Interaction Contract

### Row selection
Clicking a row simultaneously:
1. Applies selected visual state (primary-50 bg + left rail shadow)
2. Opens/swaps the LayerEditorPanel flyout to that layer's content
3. Deselects the previously selected row (one selection at a time)

Clicking the currently selected row again: deselects it and collapses the editor (editor column disappears from grid).

### Eye toggle
Clicking the eye icon toggles layer visibility on the map. It does NOT select the row. Event propagation is stopped.

### Drag-to-reorder
Library: `@dnd-kit/core` (idiomatic for React 19; matches project pattern).

- Drag ghost: 40% opacity + `--surface-2` bg + `scale(0.98)`
- Drop between rows: 2px primary insertion line with 25% primary bloom
- Drop onto group: primary-50 tint + left rail (same as selection visual)
- During any drag: all kebab buttons hide (`opacity: 0 !important`) via `.dragging-active` class on builder root
- Cross-stack constraint: basemap sublayers cannot be dragged outside the basemap group (show `not-allowed` cursor, no drop highlight)

### Editor flyout navigation
- Close `×`: collapses editor, removes `editor-open` from builder body grid class, deselects row
- Basemap sublayer breadcrumb click: swaps editor content to basemap group scene (scene B) without re-selecting in the sidebar
- Render-mode pill change: immediately swaps Appearance section content; live-updates the map

### Group expand/collapse
- Clicking the caret: toggles group children visibility (does NOT select the row for editing)
- Clicking anywhere else on a group row: selects row + opens editor (group editor scene)
- Caret animation: `transform: rotate(90deg)` over `0.15s ease`

## Accessibility Contract

- All icon-only buttons must have `aria-label` (see Copywriting Contract above)
- Row selection changes must announce via `aria-selected` on the `role="option"` or `role="listitem"` element
- Drag handle must be reachable by keyboard; kebab menu must offer "Add to group…" / "Move up" / "Move down" as keyboard-accessible alternatives to drag
- Editor flyout open: focus moves to editor close button
- Editor flyout close: focus returns to the row that triggered open
- Collapsed sections in editor: `aria-expanded` toggle
- Opacity slider: `aria-valuetext="{value}%"`
- Min/max zoom fields: `aria-label="Minimum zoom"` / `aria-label="Maximum zoom"`
