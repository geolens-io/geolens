## Layout Contract

Fully inherited from Phase 1034. No new layout columns or breakpoints.

The three-column grid (`340px sidebar · 380px flyout · 1fr map`) is the shell. The Settings
panel renders **inside the 380px flyout column**, not as an overlay, modal, or popover. The
sidebar remains fully mounted while Settings is open — identical to LayerEditorPanel behavior.

**Settings-specific layout differences from the layer editor:**

| Property | Layer Editor | Settings Panel |
|----------|-------------|----------------|
| Trigger | Row click in layer stack | `⚙` cog button in sidebar header or SidebarRail |
| Entry point | Row becomes selected (highlighted) | No row is selected; cog button gets `aria-pressed="true"` |
| Title | Layer name (semibold, truncated) | "Settings" (fixed string) |
| Type pill / subtitle | Present (layer-type chip) | Absent — Settings is sidebar-scoped, not layer-scoped |
| Back arrow | Hidden in side-by-side; shown at <800px drill-down | Hidden in side-by-side; shown at <800px drill-down |
| Footer Delete | Present for user layers | Absent — Settings has no destructive footer |
| Close × focus return | Returns to the row that triggered open | Returns to the cog button that triggered open |
| Breadcrumb | Present for sublayer scenes | Absent |

**Viewport behavior at breakpoints:**

| Viewport | Settings panel behavior |
|----------|------------------------|
| ≥ 1100px | 380px column, side-by-side with 340px sidebar |
| 800–1099px | 380px column (sidebar is 64px rail) |
| < 800px | Drill-down overlay (same Sheet/overlay as LayerEditorPanel <800px); uses back arrow, not × |

### 1. Settings editorScene (`'settings'`) — the dispatch extension

Phase 1035 established the `editorScene` dispatch pattern for the `LayerEditorPanel` flyout:

```
editorScene: 'layer' | 'basemap-group' | 'basemap-sublayer' | 'dem-raster'
```

Phase 1036 adds:

```
editorScene: 'layer' | 'basemap-group' | 'basemap-sublayer' | 'dem-raster' | 'settings'
```

When `editorScene === 'settings'`:
- The flyout column renders `SettingsEditorScene` in place of `LayerEditorPanel`'s normal section body.
- The sidebar layer stack remains fully mounted. No row is in "selected" state.
- The cog button that triggered the scene is in `aria-pressed="true"` state and receives `--primary-50` background.

**Dispatch rule:**
- Clicking the cog button when Settings is closed: sets `editorScene = 'settings'`, opens the flyout column.
- Clicking the cog button when Settings is already open: sets `editorScene = null`, closes the flyout column.
- Clicking any layer row while Settings is open: closes Settings, opens the layer editor for that row (standard row-select behavior).
- Clicking the `×` close in the Settings panel: closes the flyout, resets to no open scene.

### 2. Settings panel chrome

The Settings panel reuses the exact Phase 1034 flyout chrome. The only differences are enumerated above in the Layout Contract table. Chrome elements:

**Header (identical to LayerEditorPanel header except for content):**
- Position: top of flyout, fixed/sticky
- Left: panel title "Settings" (14px semibold)
- Right: close `×` button (Lucide `X`, 22px, `title="Close settings"`, `aria-label="Close settings"`)
- NO type pill, NO breadcrumb, NO back arrow (at ≥ 800px)
- Padding: `16px` horizontal, `12px` vertical
- Border-bottom: `1px solid --border`

**Body (scrollable section container):**
- Three collapsible sections: Terrain, Widgets, Projection
- Section order is fixed (Terrain first, Widgets second, Projection third — this order is not user-configurable)
- Body padding: `0` on horizontal edges of the scroll container; sections own their horizontal padding of `16px`
- Section separators: `1px solid --border` between sections

**Footer:**
- None. Settings has no destructive footer action.

### 3. Collapsible section pattern (Settings-specific)

Settings sections use the same collapsible pattern as the `Filter`, `Labels`, and `Source` sections in the Phase 1034 LayerEditorPanel:

- Collapsed state: `▸` caret + section name label + right-aligned hint text
- Expanded state: `▸` rotates 90° (`transform: rotate(90deg)` over `0.15s ease`), section body revealed
- Hover (collapsed): `--surface-2` background
- All three sections default to **expanded** when the Settings panel first opens for a given map session
- Section expand/collapse state is NOT persisted (resets to all-expanded on each Settings open)

**Section label style (same as Phase 1034 editor section labels):**
- 10px ALL CAPS, `letter-spacing: 0.08em`, `--muted-foreground`, weight 600 (semibold)
- `margin-bottom: 8px` inside the section body

### 4. Terrain section

**Trigger condition:** This section is always visible. Its content adapts based on whether a DEM layer is active.

**When terrain is NOT active (no DEM layer in stack with `render as: terrain`):**

```
[TERRAIN — expanded by default]
  Exaggeration    [slider disabled, value: 1.0×]
  ──────────────────────────────────────────
  "No terrain layer is active. Switch a DEM layer to
   Terrain mode to enable global terrain exaggeration."
  [12px muted hint, italic]
```

- Exaggeration slider: rendered but disabled (`disabled` attribute, cursor `not-allowed`)
- Hint: 12px `--muted-foreground`, italic, below the slider row

**When terrain IS active (a DEM layer has `render as: terrain`):**

```
[TERRAIN — expanded by default]
  Exaggeration    [====●========]   1.5×
```

- Exaggeration slider: enabled, `min=0.1 max=3.0 step=0.1`
- Value label: 12px mono (`--font-mono`), right-aligned, `--muted-foreground`, format `{value}×`
- Slider row grid: same `110px 1fr auto` layout as Phase 1034/1035 slider rows
- The source DEM layer name is shown as a secondary hint below the slider row: `"Bound to: {layer name}"` (12px, `--muted-foreground`)

**State reconciliation with DEM `render as: terrain` (BSR-09 + BSR-15 coordination):**

The DEM editor terrain mode (Phase 1035, Scene A) and the Settings terrain section both write to the same `map.terrain_config` field. The reconciliation contract:

1. When the user switches a DEM to `render as: terrain` via Phase 1035 Scene A, `handleDEMTerrainBind` sets `terrain_config.enabled = true`, `terrain_config.source_dataset_id = layer.dataset_id`, and `terrain_config.exaggeration` is preserved from any prior value.
2. The Settings terrain section reads `localTerrainConfig` and displays the current exaggeration. Changing the slider writes back to `localTerrainConfig.exaggeration` only (does not change `enabled` or `source_dataset_id`).
3. If the user switches the DEM back to a non-terrain mode (`image` or `hillshade`), `terrain_config.enabled` becomes `false`. The Settings section shows the disabled hint.
4. There is exactly ONE in-memory `localTerrainConfig` object. No duplication, no parallel state.

**ARIA for terrain slider:**
- `aria-label="Terrain exaggeration"`
- `aria-valuetext="{value}×"`
- When disabled: `aria-disabled="true"`

### 5. Widgets section

The Widgets section controls which map widgets are enabled for this map. Widget enablement is stored in `map.widgets` (a `string[]` of enabled widget IDs).

**Current registered widgets (from `register-widgets.ts`):**

| Widget ID | Label | Icon | Default visible |
|-----------|-------|------|----------------|
| `measurement` | "Measurement" | Lucide `Ruler` | false |
| `legend` | "Legend" | Lucide `Layers` | true |

**Widget toggle row layout:**

Each widget is a single row:
```
[icon 16px]  [widget name 1fr 12px]  [toggle switch 36px]
```

- Row height: 36px, `align-items: center`, padding `0 16px`
- Icon: 16px (Lucide icon matching the widget's registered icon), `--muted-foreground` color
- Widget name: 12px, 400, `--foreground`, `1fr`
- Toggle switch: shadcn `Switch` component (existing in project); `checked` = widget ID is in `map.widgets`
- Toggle `aria-label`: "Enable {widget name} widget" / "Disable {widget name} widget" (dynamic based on state)
- Hover: row gets `--surface-2` background (same as collapsed section hover)

**Toggling a widget:**
- Adds or removes the widget ID from `map.widgets` in-memory (in `localWidgets` state; mirrors the pattern of `localTerrainConfig`)
- Does NOT trigger a full save — marks map "unsaved"
- Live effect: WidgetHost responds to the updated widget list; widget appears/disappears on the map canvas immediately

**Empty state (no widgets registered — future-proofing):**
- Not possible with the current 2-widget registry, but if the list is empty: show "No widgets available." (12px, `--muted-foreground`)

### 6. Projection section

**Scope:** Runtime only for v1. The `map.projection` field is not yet persisted to the backend (the backend schema notes it as a forward-compatibility placeholder). The projection pill selection applies `map.setProjection()` on the MapLibre instance for the current session. Saving the map does NOT persist the projection in v1.

**Section layout:**

```
[PROJECTION — expanded by default]
  [Mercator]  [Globe]          ← pill strip, equal-width, full row width
  
  "Globe projection is experimental. Some layers may not render correctly."
  [12px muted hint — shown only when Globe is active]
```

- Pill strip: two pills `Mercator` and `Globe`, same visual contract as Phase 1034 render-mode pills
  - Rest: `--surface-2` background, `--foreground` text, `1px solid transparent` border
  - Active: `--primary` background, `--primary-foreground` text
  - Hover: `--surface-3` background
  - `--radius-full` border-radius, `5px 10px` padding, 12px font-size
  - Pills are equal width and span the full content area (`display: grid; grid-template-columns: 1fr 1fr; gap: 8px`)
- Globe disclaimer hint: shown below the pill strip when Globe is active; 12px, `--muted-foreground`, italic

**Default:** Mercator is the active pill on first open (matches MapLibre default and the existing map state).

**ARIA for projection pills:**
- `role="radiogroup"` on the pill container, `aria-label="Map projection"`
- Each pill: `role="radio"`, `aria-checked={boolean}`

## Interaction Contract

### Opening and closing Settings

- **Open:** User clicks the `⚙` cog button in the sidebar header OR the `⚙` cog button in the SidebarRail.
  - `editorScene` switches to `'settings'`.
  - Flyout column opens (if not already open from a layer editor) or swaps its content.
  - Any previously selected layer row is deselected.
  - Cog button receives `aria-pressed="true"` + `--primary-50` background.
  - Focus moves to the flyout's close `×` button (same as LayerEditorPanel on open).

- **Close via `×`:** User clicks the `×` close button inside the Settings panel.
  - `editorScene` switches to `null`.
  - Flyout column closes (disappears from grid).
  - Cog button receives `aria-pressed="false"`, icon color returns to `--muted-foreground`.
  - Focus returns to the cog button that triggered open.

- **Close via cog toggle:** User clicks the active `⚙` cog button a second time (while Settings is open).
  - Same effect as closing via `×`.

- **Settings closed by row selection:** User clicks any layer row in the stack while Settings is open.
  - `editorScene` switches to `'layer'` (or appropriate scene for that row type).
  - Flyout content swaps to that layer's editor.
  - Cog button returns to closed state.
  - Focus follows the standard row-select → editor-open focus contract from Phase 1034 (focus goes to editor close `×`).

### Terrain exaggeration slider

- Live update: sliding the exaggeration control calls `map.setTerrain({ source: ..., exaggeration: value })` immediately on the MapLibre instance.
- Writes `localTerrainConfig.exaggeration` to in-memory state; marks map "unsaved".
- Minimum visual value: `0.1×`; maximum `3.0×`; step `0.1`.
- Does not trigger a full save. User must explicitly save the map to persist.

### Widget toggles

- Toggling a widget adds or removes its ID from `localWidgets` state in-memory.
- Live effect: `WidgetHost` reads the updated list and mounts/unmounts the widget immediately.
- Marks map "unsaved" — does NOT auto-save.

### Projection pills

- Selecting a pill calls `map.setProjection('mercator')` or `map.setProjection('globe')`.
- The active pill updates immediately.
- Globe disclaimer hint appears/disappears based on active selection.
- NOT persisted (v1). Saved maps always load with Mercator on next open.

### Keyboard navigation

- Tab order inside Settings panel: close `×` → Terrain section toggle → exaggeration slider → Widgets section toggle → widget 1 toggle → widget 2 toggle → Projection section toggle → Mercator pill → Globe pill
- Collapsible sections: Space or Enter on the section header toggles expand/collapse
- Projection pills: Arrow keys navigate between Mercator and Globe (standard `radiogroup` keyboard pattern)
- Widget toggles: Space or Enter toggles the switch
