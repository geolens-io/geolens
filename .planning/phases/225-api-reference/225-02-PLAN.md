<objective>
Generate the actual OpenAPI snapshot by running Plan 01's script against a live geolens API instance, then commit the resulting `getgeolens.com/docs/src/content/openapi/geolens.json` file. This plan is **NOT autonomous** — it requires a running geolens backend (operator action). Once the snapshot is committed, Plan 03's starlight-openapi wiring has a real schema file to point at.

Purpose: The snapshot is the build input for the rest of the phase. Without it, starlight-openapi has nothing to render. We isolate this in its own plan because:
1. It cannot run autonomously — needs a real backend.
2. The output is a large committed artifact whose `info.version` and path-count is deterministic ONLY for a given backend state.
3. Treating it as its own atomic commit preserves a clean diff history when the snapshot is later refreshed.

Output: `getgeolens.com/docs/src/content/openapi/geolens.json` (committed)
</objective>
