starlight-openapi 0.25.0 exports:
```typescript
// default export
function starlightOpenAPI(configs: SchemaConfig[]): StarlightPlugin

// named export
const openAPISidebarGroups: SidebarItem[]  // spread into Starlight `sidebar:` array

// SchemaConfig (key fields used in this phase)
interface SchemaConfig {
  base: string                  // URL prefix without leading slash, e.g. 'guides/api'
  schema: string                // path or URL to OpenAPI doc, e.g. './src/content/openapi/geolens.json'
  sidebar?: {
    collapsed?: boolean
    label?: string              // e.g. 'Endpoints by Tag'
    operations?: { badges?: boolean; sort?: 'alphabetical' | 'method' | 'original' }
    tags?: { sort?: 'alphabetical' | 'original' }
  }
}
```

URL space generated (researcher-verified by reading `packages/starlight-openapi/libs/route.ts`):
- `{base}/` → schema overview (e.g. `/guides/api/`) — but our hand-authored `index.mdx` (Plan 06) wins because content collections override `injectRoute`
- `{base}/operations/tags/{kebab-case-tag-name}/` → tag overview (e.g. `/guides/api/operations/tags/auth/`)
- `{base}/operations/{operationId}/` → per-operation page

Plugin registration shape (RESEARCH.md §Pattern 1, lines 280-327):

```javascript
  ],
})
```
