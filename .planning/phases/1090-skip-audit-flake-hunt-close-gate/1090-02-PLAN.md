<!-- CRITICAL: how the executor coordinates with the orchestrator for Playwright MCP. -->

The executor running this plan is an agent subprocess. Agent subprocesses CANNOT directly
invoke `mcp__playwright__browser_*` tools (those are only available in the orchestrator's
tool surface — the plan-phase invocation was via `/gsd-autonomous ... --use-playwright-mcp`).

The protocol:

1. **Executor finishes all non-MCP gates (Task 1).** Captures sequential pytest + parallel
   pytest -n 4 + frontend typecheck + vitest + e2e:smoke:builder results into the close-gate
   doc.

2. **Executor returns to orchestrator with the MCP request.** The return-block shape:

   ```
   ## MCP-NEEDED

   All non-MCP close-gate gates green. Need Playwright MCP 5/5 surface check before
   TD-13 atomic commit.

   ### Surfaces to check (5 total)

   1. `http://localhost:8080/` — root (landing/home)
   2. `http://localhost:8080/maps` — map list
   3. `http://localhost:8080/datasets/<uuid>` — pick any dataset UUID from /datasets/ listing first
   4. `http://localhost:8080/maps/new` — must redirect to `/maps` per v1019 TD-11 (no 422)
   5. `http://localhost:8080/maps/<uuid>` — open any saved map

   ### Per-surface checks

   For each surface: navigate, capture console messages, capture network requests,
   take a screenshot. Report:
   - Console errors (any `level: error` messages — NOT counting expected dev-mode warnings)
   - Network 4xx/5xx (excluding expected 403/404 on resources behind auth or not-yet-loaded)
   - Page-rendered successfully (screenshot confirms expected content)

   ### Pre-flight

   Confirm stack health first:
   - `docker compose ps` shows api/worker/db healthy
   - `curl -sf http://localhost:8080/` returns 200
   - If frontend dev server is not running, start it: `cd frontend && npm run dev &`

   ### Resume signal

   After running MCP 5 surfaces, return the per-surface results (PASS/FAIL + console error
   count + network 4xx/5xx count). Executor resumes Task 2 to consume results into
   `1090-01-CLOSE-GATE.md` and commit.
   ```

3. **Orchestrator runs MCP 5/5 surfaces.** Uses `mcp__playwright__browser_navigate` +
   `mcp__playwright__browser_console_messages` + `mcp__playwright__browser_network_requests`
   per surface (see v1019 / v1010.1 close-gate patterns for exact tool invocations).

4. **Orchestrator returns results.** Per-surface PASS/FAIL with evidence cited.

5. **Executor resumes Task 2.** Writes results into `1090-01-CLOSE-GATE.md` `## Close-gate
   matrix` section. If 5/5 GREEN, proceeds to Task 3 (TD-13 atomic commit + tags). If ANY
   surface FAILED, halts and surfaces the failure to the orchestrator before tag cuts.

The plan does NOT try to script the MCP calls itself — it returns to the orchestrator at
the natural handoff point and consumes the result on resume.
