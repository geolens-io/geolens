# Phase 1028 Plan: Cluster Source Routing and Authoring Parity

**Status:** Complete
**Milestone:** v1006 Large Dataset Cluster Scaling
**Requirements:** REND-01, REND-02, REND-03, REND-04, REND-05

## Goal

Route Cluster layers to bounded client-side GeoJSON or server-side cluster vector tiles based on dataset size while keeping the existing renderAs and authoring contract unchanged.

## Scope

- Preserve `style_config.render_mode = "cluster"` and existing `style_config.builder` controls.
- Keep bounded client-side GeoJSON clustering for small point datasets.
- Route large point datasets to `/tiles/clusters/data.{table}/{z}/{x}/{y}.pbf`.
- Reuse normal signed tile URL, API key token, and embed-token transport behavior.
- Keep unsupported or unknown-count datasets on the existing Point fallback path.

## Implementation

1. Add cluster source strategy helpers for bounded GeoJSON, server tile, and fallback states.
2. Expose Cluster in renderAs options for both bounded and large point datasets with count metadata.
3. Add a cluster tile URL helper that preserves signed tile params and cluster radius/max-zoom options.
4. Update `syncLayersToMap` to choose bounded GeoJSON for small Cluster layers and server cluster vector tiles for large Cluster layers.
5. Update cluster companion layers to include `source-layer` when backed by vector tiles.
6. Keep builder/viewer bounded GeoJSON fetches limited to bounded Cluster layers.
7. Update token refresh paths for server cluster vector sources.
8. Add focused frontend tests for strategy, renderAs, map sync, adapter source-layer, viewer no-fetch behavior, and URL generation.

## Acceptance

- Large point datasets expose `Cluster` and patch only existing writable fields.
- Small point datasets keep the existing bounded GeoJSON cluster source.
- Large point Cluster layers create vector sources pointing at `/tiles/clusters/...`.
- Builder and viewer paths share the same source strategy.
- Existing Point/Symbol/Heatmap/vector/raster source behavior remains unchanged by focused tests.
