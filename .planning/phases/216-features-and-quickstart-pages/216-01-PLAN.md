Capture target shape (TypeScript):
```typescript
interface CaptureTarget {
  name: string;                                        // Human-readable (e.g., "Search")
  filename: string;                                    // Output filename, e.g., "search.png"
  viewport: { width: number; height: number };        // Per research Q6: 1600x900 default, 1600x800 for dataset detail
  requiresAuth: boolean;                               // RBAC + Map Builder + AI chat need login
  setup: (page: Page, ctx: CaptureContext) => Promise<void>;  // Navigate + wait-for-content
  skipIf?: (ctx: CaptureContext) => Promise<boolean>;  // For AI chat when no LLM key
}

interface CaptureContext {
  baseUrl: string;         // GEOLENS_URL, defaults to http://localhost:8080
  adminUser: string;       // GEOLENS_ADMIN_USERNAME, defaults to 'admin'
  adminPass: string;       // GEOLENS_ADMIN_PASSWORD, defaults to 'admin'
  aiAvailable: boolean;    // Probed via admin settings endpoint at script start
}
```

Fixed capture list (7 targets per D-13 + research Q6):
- search.png            — 1600x900 — no auth — nav `/` (SearchPage is the index route)
- map-builder.png       — 1600x900 — auth   — navigate `/maps/:id` (use first seeded map from GET /api/maps)
- data-ingestion.png    — 1600x800 — auth   — navigate `/datasets/:id` (first vector dataset)
- raster-vrt.png        — 1600x800 — auth   — navigate `/datasets/:id` (first raster dataset)
- ai-chat.png           — 1600x900 — auth   — navigate `/maps/:id`, open ChatPanel, type query, wait for response — D-14/D-15 conditional empty-panel fallback
- rbac.png              — 1600x800 — auth   — navigate `/admin/users`
- quickstart-outcome.png — 1600x900 — no auth — navigate `/` (same as search; planner's call per CONTEXT.md D-06 allows aliasing search.png)

Environment variables (all optional, documented defaults):
- GEOLENS_URL                       — default "http://localhost:8080"
- GEOLENS_ADMIN_USERNAME            — default "admin"
- GEOLENS_ADMIN_PASSWORD            — default "admin"
- CAPTURE_SKIP_AI                   — when set, skip AI chat capture without probe
- CAPTURE_DRY_RUN                   — when set, exits before launching browser (used by Plan 01's verify step)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → capture script | Script reads GEOLENS_ADMIN_PASSWORD from env — operator-controlled, local-only |
| capture script → live GeoLens | Script hits http://localhost:8080 — trusted local network only, no internet egress |
| capture script → src/assets/screenshots/ | Script writes PNG files to the repo — next plan commits them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-216-01-01 | Information Disclosure | capture-screenshots.ts | mitigate | Never log admin password; read from env and pass to loginAsAdmin() without stringifying. Script is dev-only — not shipped in production build. |
| T-216-01-02 | Tampering | OUTPUT_DIR misconfiguration | mitigate | Hard-code OUTPUT_DIR to `path.resolve(__dirname, '../src/assets/screenshots')`; add header comment explicitly forbidding `public/` (per Pitfall 1). Automated check in Task 2 verify grep confirms no `public/screenshots` string exists. |
| T-216-01-03 | Spoofing | GEOLENS_URL override | accept | Operator-controlled env var; if wrong URL is set, script fails-fast with connection refused. No secrets leak via URL. Local-only threat model. |
| T-216-01-04 | Elevation of Privilege | Playwright browser binary download | accept | `npx playwright install chromium` is operator-triggered (documented in README, not auto-run). Binary comes from Microsoft's official CDN per Playwright's install process. |
| T-216-01-05 | Denial of Service | Infinite loop on wait-for-content | mitigate | All `page.waitForLoadState` / `page.waitForSelector` calls use default timeouts (30s). Each capture is wrapped in try/catch so a stuck capture does not block the rest of the run. |
| T-216-01-06 | Information Disclosure | LLM API key in captured ai-chat.png | accept | The ChatPanel UI does not render API keys — it renders conversation content. Per D-14 the query is a benign geospatial question. Screenshot is committed to a public-facing marketing repo; operator is responsible for ensuring no secrets appear on screen. |

All dispositions are appropriate: this phase's primary risk (Pitfall 1 — wrong OUTPUT_DIR) is mitigated by hard-coding + header comment + automated grep. The Information Disclosure threat for AI chat is operator-controlled and unavoidable (you can't automate "don't show secrets on screen" — that's a human review gate in Plan 02).
</threat_model>
