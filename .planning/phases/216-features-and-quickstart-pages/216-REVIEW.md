## Summary

Phase 216 added the Features and Quickstart pages plus a Playwright-driven screenshot capture script. The Astro components are clean: preview components correctly use `astro:assets` `<Picture>` with `src/assets/screenshots/` imports (not `public/`), alt text is descriptive, `sizes`/`widths` are appropriate, and the FeatureStripe layout is straightforward. The Quickstart page content is accurate and the troubleshooting section is useful.

Three issues need attention before the site ships:

1. **Missing OG images** — `public/og/features.png`, `public/og/quickstart.png`, and `public/og/home.png` are referenced in meta tags but the `public/og/` directory does not exist. All pages currently emit broken OG image URLs.
2. **Stale screenshot bug in capture script** — when a capture's `setup()` returns early (no maps, no datasets), a screenshot is still written to disk capturing whatever page was loaded previously in the same browser session.
3. **Dead `preview-test.astro` scaffold page** — it renders at `/preview-test` with no auth guard and explicitly says to delete it before integration.

## Warnings

### WR-01: Missing OG image assets — all pages emit broken social share images

**File:** `getgeolens.com/src/pages/features/index.astro:12` and `getgeolens.com/src/pages/quickstart/index.astro:5`

**Issue:** Both pages construct OG image URLs pointing at `/og/features.png` and `/og/quickstart.png` respectively. `SiteLayout.astro` falls back to `/og/home.png` for pages that don't pass `ogImage`. The `public/og/` directory does not exist — verified by `ls public/`: only `favicon.ico`, `favicon.svg`, and `robots.txt` are present. Social crawlers (Twitter, Slack, LinkedIn) will receive 404s for all OG image requests, causing blank previews.

**Fix:** Create `public/og/` and add the three required files. The quickest approach is static fallback PNGs (1200×630). If OG images are generated by a separate process, add that step to the deployment checklist and document it in `scripts/README.md`. Minimum required files:
```
public/og/home.png
public/og/features.png
public/og/quickstart.png
```

### WR-02: Screenshot written even when setup() returns early — stale-page capture

**File:** `getgeolens.com/scripts/capture-screenshots.ts:327-330` (screenshot call) vs lines `151-153`, `168-170`, `183-186`, `202-205` (early returns)

**Issue:** The `setup()` functions for Map Builder, Data Ingestion, Raster/VRT, and AI Chat all do an early `return` when no seeded data is found (e.g., `if (!mapId) { console.warn(...); return; }`). Control then falls through to `page.screenshot()` at line 327, which captures whatever page is currently loaded in the shared browser session — typically the previous capture's final state. The output file is written with no indication it contains stale content. The operator will see a misleadingly correct-looking file with incorrect content.

**Fix:** Have `setup()` signal skip intent by throwing a specific error, or return a boolean, so the capture loop can skip the screenshot. Simplest approach — add a sentinel throw:

```typescript
// In each early-return branch:
if (!mapId) {
  console.warn('  No seeded maps found — skipping Map Builder capture');
  throw new Error('SKIP'); // caught by the outer try/catch
}

// In main loop, distinguish skip from real failure:
} catch (err: unknown) {
  const msg = err instanceof Error ? err.message : String(err);
  if (msg === 'SKIP') {
    console.log(`  SKIP ${target.name} (no data)`);
    continue; // don't increment failed
  }
  console.warn(`  WARN ${target.name}: ${msg}`);
  failed++;
}
```

Alternatively, return a `{ skipped: boolean }` object from `setup()` and check it before calling `page.screenshot()`.

### WR-03: Dead scaffold page is publicly routable

**File:** `getgeolens.com/src/pages/preview-test.astro:15`

**Issue:** The inline comment reads "Delete after integration into /features page." The `/features` page is complete (verified in `features/index.astro`), meaning this page's purpose is fulfilled. It currently renders at `/preview-test` on the live marketing site with no access control. While a marketing SSG has no sensitive data, shipping debug/QA scaffold pages is a quality concern and shows up in sitemaps (the `@astrojs/sitemap` integration will include it unless filtered).

**Fix:** Delete `src/pages/preview-test.astro`. If a visual QA page is needed during development, use a route that is excluded from the sitemap filter or keep it only in a branch.

### IN-01: Dead skipIf implementation on AI Chat capture target

**File:** `getgeolens.com/scripts/capture-screenshots.ts:197-199`

**Issue:** The `skipIf` callback for the AI Chat target always returns `false` with comment "setup() handles both branches." This makes `skipIf` permanently dead code — the capture loop evaluates it at line 315-318 but it never skips. The `SKIP_AI` env var is only checked in `probeAiAvailable()`, so setting `CAPTURE_SKIP_AI=1` does not cause the AI Chat capture to be skipped (it just probes as unavailable and takes the empty-panel branch). This creates a misleading operator interface.

**Fix:** Either remove the `skipIf` field from the AI Chat entry entirely (since `setup()` handles both branches as intended), or implement it consistently:

```typescript
skipIf: async (ctx) => SKIP_AI,  // if operator wants to skip entirely
```

### IN-02: Nav SVG clipPath uses a hardcoded ID

**File:** `getgeolens.com/src/components/layout/Nav.astro:37`

**Issue:** The SVG `<clipPath id="geolens-nav-logo-clip">` uses a static ID. SVG IDs are scoped to the document, not the element. If `Nav.astro` is ever rendered twice on the same page (e.g., sticky header + mobile drawer), the second `<clipPath>` would have a duplicate ID and both logo SVGs would reference whichever clip definition appears first in the DOM. This is fine for the current single-Nav layout but is a latent trap.

**Fix:** Low priority for now since Nav renders once per page. If a mobile drawer variant is added later, use a unique ID per instance (e.g., a random suffix or Astro's crypto.randomUUID()):

```astro
---
const clipId = `geolens-nav-logo-clip-${Math.random().toString(36).slice(2, 8)}`;
---
<clipPath id={clipId}>...</clipPath>
<g clip-path={`url(#${clipId})`}>...</g>
```
