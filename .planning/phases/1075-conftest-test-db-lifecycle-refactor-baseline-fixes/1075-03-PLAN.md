<root_cause_hypothesis>
The 3 named failures (`test_upload_success`, `test_csv_upload_success`, `test_service_job_commits_with_service_body`) are listed in the v1016 Phase 1073 audit as reproducing "on pre-1073 baseline" — meaning they are NOT v1016 regressions. Combined with the 1363 conftest `InvalidCatalogNameError` errors that pollute Phase 1074's run, the most likely root causes are:

1. **Conftest cascade (PRIMARY HYPOTHESIS).** `test_ingest.py` uses the `client` fixture extensively, which depends on `_test_db_lifecycle`. If the session DB fails to initialize cleanly, every test in this file ERRORs at collection time (not "FAILED"). The v1016 audit categorically calls these 3 tests "FAILED" — distinct signal from the 1363 errors. So the 3 failures are likely real test-logic failures EVEN AFTER Plan 01 fixes the lifecycle. **Run them post-Plan-01 first.**

2. **IngestJob response-shape drift.** v1015 / v1016 added fields to `JobStatusResponse` (the `[v1013 post-smoke inline fixes]` MEMORY.md note confirms `JobStatusResponse 'fanned_out' Literal` was added). If the test asserts a specific field set on the response (e.g., `assert resp.json()["status"] == "pending"`), but the production response now has an extra required field (or a renamed field), the test fails with a `KeyError` or `AssertionError`.

3. **File-upload contract drift.** The `mock_file_save` fixture patches `save_upload_file`. If a recent phase moved upload-handling into a different module path (e.g., `app.processing.ingest.upload.save_upload_file` instead of `app.processing.ingest.router.save_upload_file`), the patch silently no-ops. The actual file save still works in tests (because tests write to tmp_path naively), but the assertion on `mock_save.called` fails.

4. **CSV-specific column mapping.** `test_csv_upload_success` likely calls POST /ingest/upload with a CSV file + lat/lng column hints. If the v1015 ingest hardening (REMED-02 — "JobStatusResponse fanned_out Literal") changed the CSV body shape (e.g., added `lat_column`, `lng_column` as required), the test fails on the API contract.

5. **service_job_commits — registered-source dispatch.** `test_service_job_commits_with_service_body` is the only one of the three that uses POST /ingest/register (or similar service-URL endpoint). If v1015 changed how service registration commits to the DB (e.g., service detection now requires an extra adapter probe), the test's assertion on `IngestJob` row presence fails.

**Triage protocol:**
1. Run the 3 named tests in isolation first: `cd backend && uv run pytest tests/test_ingest.py::TestUpload::test_upload_success tests/test_ingest.py -k "csv_upload_success or service_job_commits_with_service_body" -x -v --tb=long`
2. Inspect each failure's traceback. The hypothesis (2, 3, 4, or 5) is determined by the exception type and the assertion line.
3. Apply the smallest fix per the hypothesis.
4. Skip (with issue link) only if root cause is fundamentally a production-code-change-required scenario that exceeds the milestone's hygiene scope.
</root_cause_hypothesis>
