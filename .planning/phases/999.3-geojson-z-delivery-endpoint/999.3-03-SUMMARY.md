## Summary

Wired GeoJSON-Z data fetching into ViewerMap and added GeoJSON source branching to the map sync system. Small 3D datasets (is_3d && feature_count <= 5000) automatically use GeoJSON sources instead of MVT tiles, preserving Z coordinates for 3D rendering.

## Key Changes

- `frontend/src/api/geojson-z.ts`: New `fetchGeoJsonZ` utility with JWT, API key, and embed token auth paths
- `frontend/src/components/viewer/ViewerMap.tsx`: `geojsonDataRef`, `geojsonZLayers` memo, fetch `useEffect`, passes `geojsonDataRef.current` to `syncLayersToMap`
- `frontend/src/components/builder/map-sync.ts`: `geojsonDataMap` parameter on `syncLayersToMap`, GeoJSON-Z branch that creates GeoJSON sources and skips MVT
- `frontend/src/components/builder/layer-adapters/types.ts`: `sourceType` on `AdapterLayerInput`
- All 4 adapters: conditional `source-layer` omission for GeoJSON sources
- `frontend/src/types/api.ts`: `is_3d`, `is_dem`, `tile_url`, `feature_count` on layer response types

## Deviations

- BuilderMap wiring deferred — ViewerMap is the primary consumer; BuilderMap can be wired in a follow-up
- Restored fill-extrusion code from phase 999.1 that was lost during worktree file copy regression
- Restored `is_dem`/`is_3d`/`tile_url`/`feature_count` on TypeScript types lost during same regression
