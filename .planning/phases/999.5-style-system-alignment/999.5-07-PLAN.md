The new doc covers FOUR sections in this exact order:

### 1. Shared Tokens (Adopted from App Guide)
Document what the marketing site and docs adopt verbatim from `geolens/docs/DESIGN-GUIDE.md`:
- **Primary Color Palette** — OKLCH hue 250, 50–900 scale, link to upstream app guide for canonical values; note the drift script enforces the mirror.
- **Typography** — IBM Plex Sans Variable (sans) + IBM Plex Mono (mono); list font strings, package names (`@fontsource-variable/ibm-plex-sans@^5.2.8` + `@fontsource/ibm-plex-mono@^5.2.7`), import paths used per surface; note the drift script asserts the font-family string in all three surfaces.
- **Logo** — canonical `logo.svg` location: `getgeolens.com/public/logo.svg` and a byte-identical copy at `getgeolens.com/docs/src/assets/logo.svg`; explain the currentColor approach; explain why two copies (Astro project-root constraint per RESEARCH.md Pitfall 3); note that the app uses a React component (`GeoLensLogo.tsx`) with different geometry — DO NOT try to converge them.
- **Aside / Callout Accents (Docs only)** — `note` and `tip` Starlight Aside variants overridden to brand primary (`--primary-700` border, `--primary-800` text, `--primary-50/--primary-900` fill); `caution` and `danger` intentionally kept at Starlight orange/red as semantic warnings (D-04). Reference `docs/src/styles/custom.css`.

### 2. Marketing-Specific Tokens (Intentional Divergence)
Document what the marketing site intentionally does NOT adopt from the app guide, with rationale:
- **Background and Neutral Colors** — cool-white (`oklch(1 0 0)`) instead of warm atlas-paper (`oklch(0.985 0.003 85)`); rationale: marketing hero imagery is tuned for white.
- **Light-Only Mode** — quote/paraphrase the new D-06 rationale comment from `global.css` ("Intentional brand decision... Rationale: marketing surfaces prioritize first-impression clarity..."). Note this is tracked as a separate backlog item; trigger conditions: design owner onboarded, or product ships dark mode to >50% of signed-in users.
- **No Surface Hierarchy** — no `--surface-0..3`; rationale: marketing pages are content-display, not data-dense.
- **No Elevation Tokens** — flat cards by design; appropriate for marketing aesthetic.

### 3. Docs-Specific Tokens (Starlight Overrides)
- **Starlight Slots Bridged (custom.css)** — table of `--sl-color-*` slots overridden and the value each maps to:
  - `--sl-color-accent-low/base/high` → `--primary-100/--primary-700/--primary-800` (light); inverted in dark
  - `--sl-color-bg-sidebar` → `oklch(0.99 0.003 85)` (light only — RESEARCH.md Pitfall 5)
  - `--sl-color-blue-low/base/high` (note) → primary-50/700/800 light; primary-900/400/100 dark
  - `--sl-color-purple-low/base/high` (tip) → primary-50/700/800 light; primary-900/400/100 dark
- **Starlight Slots at Default (NOT overridden)** — explicit list:
  - `--sl-color-orange-*` (caution) and `--sl-color-red-*` (danger) — semantic warning colors preserved
  - `--sl-color-bg`, `--sl-color-bg-nav`, `--sl-color-bg-inline-code`, `--sl-color-bg-accent`, etc.
  - `--sl-color-bg-sidebar` in dark mode (Starlight gray-6 default)
- **Expressive Code Theme** — intentionally untouched per CONTEXT Out-Of-Scope; no `--code-*` token bridge. Expressive Code's syntax theme is purpose-built for code readability.
- **Dark Mode** — Starlight dark mode is active in docs (unlike marketing); accent slots and Aside slots ARE overridden in dark; surface and elevation tokens stay at Starlight defaults.

### 4. Drift CI Contract
- **What `check-token-sync.sh` enforces:**
  - Primary palette: 10 stops (50–900) verbatim mirrored between `getgeolens.com/src/styles/global.css` and `getgeolens.com/docs/src/styles/custom.css` (Phase 224 BRAND-04)
  - Font-family: `'IBM Plex Sans Variable'` literal string present in all three surfaces (Phase 999.5 D-03), with fail-OPEN for the app leg when the geolens checkout is unavailable (CI single-repo)
  - Exit code 0 = synced; non-zero = drift → CI fails
- **What is NOT enforced:**
  - Aside color values (derive from `--primary-*` via `var()` references)
  - Sidebar tint (single-surface docs-only value)
  - Marketing surface/elevation tokens (intentionally absent)
  - Dark mode tokens (untracked across surfaces)
- **Maintenance convention:**
  - Any change to `--primary-*` in `geolens/frontend/src/index.css` must be mirrored to `getgeolens.com/src/styles/global.css` then `getgeolens.com/docs/src/styles/custom.css`; the drift script then validates.
  - Font changes: update both `@import` and font-family strings in marketing + docs; the script catches missing declarations.
