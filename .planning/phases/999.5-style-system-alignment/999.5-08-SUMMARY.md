## Findings & In-Phase Fixes

The verification gate surfaced three distinct issues that were addressed
inline (per project feedback rule: "fix review findings inline rather
than deferring to gap closure"):

### 1. M9 — Sidebar tint leaked into dark mode

Plan 04 SUMMARY claimed RESEARCH Pitfall 5 was satisfied, but the actual
`custom.css` declared `--sl-color-bg-sidebar: oklch(0.99 0.003 85)` on
plain `:root` with no `[data-theme='dark']` override. `getComputedStyle`
in dark mode confirmed the warm tint persisted.

**Fix:** Restructured `custom.css` so every surface variable is paired:
the `:root` block declares the light-mode value, and the
`:root[data-theme='dark']` block declares the dark-mode complement. The
leak is now impossible by construction.

### 2. Canonical logo geometry mismatched the app

Plan 02 created the canonical logo from the marketing `favicon.svg`
geometry (r=22, cx/cy 27, single horizontal axis + one ellipse, no
crosshair ticks). The actual app logo (`GeoLensLogo.tsx`'s `ReticleIcon`)
uses r=18, cx/cy 26, dual graticule axes (horizontal AND vertical),
crossed ellipses (rx/ry pair both directions), four NSEW crosshair
ticks, and a center dot. The two designs were visually distinct enough
that the user reported "the docs logo doesn't match the app logo".

**Fix:** Replaced `public/logo.svg` and `docs/src/assets/logo.svg` with
a static currentColor SVG extracted directly from the React component.
Both files remain byte-identical to satisfy the dual-file maintenance
convention.

### 3. Option B partial alignment felt mediocre

Phase opened with D-09 choosing Option B (typography + primary palette
+ asides only; surfaces stay at Starlight defaults). The verification
gate showed this left docs visually disconnected from the app — the
Starlight default cool-white body felt generic next to the app's warm
atlas-paper surface.

**Fix (D-09b):** Adopted the app's full surface ladder for docs in both
modes — warm atlas paper in light, cool hue-250 in dark. Marketing
remains light-only cool-white per D-06 (the mixed policy is intentional;
docs is reference material that benefits from app continuity, marketing
is hero/CTA content where flatter surfaces read better). DESIGN-GUIDE.md
section 3.1 + Section 4.2 updated accordingly.
