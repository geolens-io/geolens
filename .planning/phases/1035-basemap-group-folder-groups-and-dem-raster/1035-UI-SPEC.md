### 1. Basemap group row (extends Phase 1034 row anatomy)

The basemap group row reuses the Phase 1034 `StackRow` component. The ONLY differences are:

| Property | Value |
|----------|-------|
| Caret column | `▸` visible (group row); rotates 90° on expand |
| Type icon glyph | `⊞` |
| Type icon class | `.type-basemap` → `--primary-50` bg / `--primary-700` fg |
| Row name | e.g. "Basemap · Positron" — provider label may be appended at muted weight |
| Kebab menu items | "Swap basemap", "Reset appearance" only — no Rename, no Delete |

**Expansion behavior:**
- Caret click: toggles `.open` on the `.group-children` container; does NOT select the row for editing
- Row body click (anywhere except caret + eye + kebab): selects row AND opens editor to Scene B (basemap-group editor)
- Expansion state is persisted to the saved-map `groupMeta.{groupId}.expanded` boolean; see interaction contract below

**Sublayer rows inside the expanded basemap group:**

| Property | Value |
|----------|-------|
| Row anatomy | Same grid as Phase 1034 — caret is `visibility:hidden`, grip visible, eye, type icon, name, opacity, kebab |
| Available sublayers | Roads, Labels, Buildings, Boundaries, Land-Water (5 items) |
| Type icon | Same raster/vector type colors as the sublayer's underlying type |
| Name | Sublayer display name (e.g. "Roads", "Labels") |
| Kebab items | None — basemap sublayers have no kebab (they cannot be renamed, duplicated, or deleted). Kebab column is hidden for sublayer rows. |
| Click | Selects sublayer row AND opens editor to Scene C (basemap-sublayer editor) |
| Drag-to-reorder | Basemap sublayers CANNOT be dragged out of the basemap group. Show `cursor: not-allowed` if user attempts to drag a sublayer row. No drop highlight is shown on external targets. |

### 2. User folder group row (new component — FolderGroupRow)

Reuses the Phase 1034 `StackRow` component. New wrapping behavior:

| Property | Value |
|----------|-------|
| Caret column | `▸` visible (group row); rotates 90° on expand |
| Type icon glyph | `▸` |
| Type icon class | `.type-folder` → `oklch(0.93 0.03 80)` bg / `oklch(0.45 0.10 80)` fg |
| Row name | User-defined label (editable via kebab "Rename group") |
| Kebab menu items | "Rename group", "Add layer", "Ungroup", "Delete group" |

**Expansion state persistence:**
- Expansion state (open/closed) is stored in `groupMeta.{groupId}.expanded` on the saved-map JSON object (same field as basemap group).
- State is read on map load and applied before first paint.
- Toggling expansion writes the new boolean immediately (no debounce; no full save triggered — map is only marked "unsaved" when expansion state differs from the last saved state).

**Rename group flow:**
- Kebab → "Rename group" → the group name cell transitions to an inline `<input>` in-place (no dialog). Existing name is pre-filled, fully selected.
- `Enter` or blur commits; `Escape` cancels.
- Empty string → revert to previous name (do not permit blank group names).
- Name is trimmed of leading/trailing whitespace on commit.

**Ungroup flow:**
- Kebab → "Ungroup" → immediate action (no confirmation required). Group children are promoted to the parent container at the group's previous position, maintaining their relative order.

**Delete group flow:**
- Kebab → "Delete group" → inline confirm in the kebab callout area (no modal): "Delete this group and all its layers?" with "Delete all" (danger) + "Keep group" (secondary). Confirmation is shown in-place below the kebab trigger.
- "Delete all" removes both the group container and all child layers from the stack and the map.

**Child rows inside an expanded user folder group:**
- Same row anatomy as loose layers (caret `visibility:hidden`, grip, eye, type icon, name, opacity, kebab).
- Kebab on child rows: "Rename layer", "Duplicate", "Delete layer", "Move out of group" (replaces "Add to group…" when already inside a group).
- Child rows can be drag-reordered within the group.
- Child rows can be dragged OUT of the group by dropping onto the loose-layer stack area (produces the reorder insertion line at the target position).

### 3. `Add to group…` kebab item — now functional

Phase 1034 rendered this item disabled (grayed, `pointer-events: none`). Phase 1035 enables it.

**Trigger:** Kebab → "Add to group…" (available on any non-group layer row that is NOT already a child of a group; hidden if the layer is already inside any group).

**Sub-flow — group picker:**
- A flat dropdown list appears immediately below the "Add to group…" menu item (inline within the Radix DropdownMenu; not a modal). The list contains all currently defined user folder groups in the stack.
- Each group entry shows: `▸` glyph + group name. No type icon.
- At the bottom of the list: `＋ New group…` action. Selecting this creates a new folder group (default name "Group {N}" where N is next available index), moves the layer into it, and immediately opens the group name for inline rename.
- Selecting an existing group: moves the layer into that group as the last child; the group expands if it was collapsed.
- If no user folder groups exist, the list shows only `＋ New group…`.

**Visual:**
- The sub-list extends the DropdownMenu downward; it is not a submenu flyout (`DropdownMenuSub` pattern is acceptable if the Radix API is cleaner, but the opening direction must be downward, not rightward, to avoid clipping against the sidebar right edge).
- Group list items: 12px regular text, `--surface-1` background, hover `--surface-2`.
- `＋ New group…` item: 12px, `--primary` color (accent), no icon needed.

### 4. DEM raster row (non-group; extends Phase 1034 type icon contract)

DEM layers appear as regular raster rows with NO caret. The type icon glyph reflects the active render mode.

| Render mode | Type icon glyph | Background | Foreground |
|-------------|----------------|------------|------------|
| Image | `▦` | `--type-raster-bg` | `--type-raster` |
| Hillshade | `⛰` | `--type-raster-bg` | `--type-raster` |
| Terrain | `◬` | `--type-raster-bg` | `--type-raster` |

The row itself is identical to any other raster row (no special badge, no special inline control). Render-mode change happens only inside the editor flyout.

### 5. LayerEditorPanel — Scene A: DEM raster editor

Used when a DEM raster row is selected. Reuses exact Phase 1034 editor chrome (header, body scroll, footer). Only the content of sections changes.

**Header:**
- Layer name: e.g. "Grand Canyon DEM" (semibold 14px, truncated)
- Type pill: raster colors (`--type-raster-bg` / `--type-raster`)
- Subtitle: "1 band · EPSG:4326" (or actual band count and CRS)

**Render as section (expanded by default):**
- Pill strip: `▦ Image` | `⛰ Hillshade` | `◬ Terrain`
- Switching a pill immediately:
  1. Updates the layer type icon in the sidebar row to match the new mode glyph
  2. Swaps the Appearance section content (see below)
  3. Writes `render_mode` to the layer's `style_config` (no full map save triggered)

**Appearance section — Image mode:**
- Opacity field: opacity slider (inherits from Visibility section; do NOT duplicate opacity here — show a note: "Use Visibility section to adjust opacity")
- Color correction hint: "No additional appearance controls for image mode"
- This section may be visually minimal; that is correct per spec.

**Appearance section — Hillshade mode:**
Contains three sub-sections laid out vertically within the Appearance section:

1. **Sun position** (always shown in Hillshade mode):
   ```
   [COMPASS WIDGET 90×90px centered]
   Azimuth     [slider 0–360°] [value label]
   Altitude    [slider 0–90°]  [value label]
   Exaggeration [slider 0–5×]  [value label]
   ```
   - Compass widget: 90px diameter, circular, `border: 1px solid --border`, radial gradient background (`--surface-1` center → `--surface-2` edge), N-S and E-W crosshair lines (1px `--border`), primary-colored needle (2px wide, 38px from center, rotates to `azimuth` degrees).
   - Needle CSS: `transform-origin: center bottom; transform: translate(-50%, -100%) rotate({azimuth}deg)`.
   - Sliders use the same range input as Phase 1034's opacity slider (primary-colored thumb).
   - Value labels: 12px mono (`--font-mono`), right-aligned, `--muted-foreground`.

2. **Shading colors** (always shown in Hillshade mode):
   ```
   Highlight    [color swatch + hex]
   Shadow       [color swatch + hex]
   Accent       [color swatch + hex]
   ```
   - Color swatches reuse Phase 1034's `.color-swatch` pattern (14×14px swatch square + hex code in mono font, `border: 1px solid --border`, click opens color picker).
   - Default values: Highlight `#FFFFFF`, Shadow `#000000`, Accent `#D4A97A` (warm sand — matches MapLibre hillshade defaults).

**Appearance section — Terrain mode:**
- Shows a single hint: "Terrain uses elevation data to extrude the map surface. Adjust terrain exaggeration in Settings → Terrain."
- No paint controls here — terrain is a map-level config (moved to the Settings panel in Phase 1036). This section content intentionally stays minimal.
- Do NOT show the compass or shading controls in Terrain mode.

**Visibility section (all DEM modes):**
- Identical to Phase 1034: opacity slider + min zoom + max zoom.

**Collapsed sections (all DEM modes):**
- Filter: collapsed, hint "No filter"
- Source: collapsed, shows filename, resolution, EPSG — no Labels section for raster layers (omit Labels entirely for DEM)

**Footer:**
- `Delete layer` (danger ghost) — same as Phase 1034 user layer footer.
- No `Back` button (DEM is not a sublayer).

### 6. LayerEditorPanel — Scene B: Basemap group editor

Used when the basemap GROUP ROW is selected (not a sublayer). Reuses exact Phase 1034 editor chrome.

**Header:**
- Breadcrumb: none (basemap group is top-level; no parent context)
- Layer name: "Basemap · {preset-name}" (e.g. "Basemap · Positron"), semibold 14px
- Type pill: `--primary-50` bg / `--primary-700` fg (matches basemap type icon)
- Subtitle: "{provider} · vector tiles" (e.g. "OpenFreeMap · vector tiles")

**Render as section:** OMIT — basemap is not switchable via render-mode. This section does not appear in Scene B.

**Preset section (expanded by default, takes the place of Render as):**
- Section label: "PRESET" (10px ALL CAPS, semibold, `--muted-foreground`, `letter-spacing: 0.08em`)
- 2×2 card grid (`gap: 8px`, full-width):
  - Positron, Bright, Dark Matter, Satellite (or whatever the configured presets are)
  - Each card: `border: 1px solid --border`, `border-radius: --radius-md`, `padding: 8px`
  - Active card: `border-color: --primary`, `box-shadow: 0 0 0 1px --primary`
  - Card thumbnail: 56px height, `border-radius: --radius-sm`, actual tile preview image (real implementation uses the basemap's tile endpoint for a static preview; fallback is a hand-tuned CSS gradient matching the basemap palette)
  - Card name: 11px, 400 weight, `--foreground`
  - Card meta: 10px, `--muted-foreground` (provider name)
- Below the preset grid: `＋ Add custom basemap…` link (12px, `--primary` color, no icon needed, left-aligned)

**Sublayers section (expanded by default):**
- Section label: "SUBLAYERS" (same label style)
- Hint below label: "Click any sublayer in the sidebar to style it individually." (12px, `--muted-foreground`)
- Compressed sublayer list (NOT the full-height row grid):
  - Each item: `height: 32px`, `display: flex; align-items: center; gap: 8px; padding: 0 4px`
  - Eye toggle (22px): same visibility toggle as stack rows
  - Type icon (22px): same type icon square as stack rows (per sublayer's actual type)
  - Name (1fr): 12px, 400, `--foreground`, truncated
  - Opacity (60px): range slider, primary thumb (same as stack row opacity)
  - Sublayer rows are NOT clickable in this list (clicking them in the sidebar is the intended path)

**Visibility section:**
- Single master opacity slider for the whole basemap group. Label: "Master opacity".
- Changes cascade to all sublayers (this is a CSS/layer-level opacity, not per-sublayer).

**Footer:**
- `Reset appearance` (secondary ghost, full-width or half-width)
- `Remove basemap` (secondary ghost, full-width or half-width)
- Both are secondary ghost style — NOT danger style. Basemap can be re-added.
- Layout: side-by-side if both fit; stack vertically if not.
- No confirmation for "Remove basemap" — it is recoverable via re-add.
- No `Delete layer` button.

### 7. LayerEditorPanel — Scene C: Basemap sublayer editor

Used when a basemap sublayer row (inside the expanded basemap group) is selected. Reuses exact Phase 1034 editor chrome with a modified header.

**Header (modified — breadcrumb-style):**

```
[11px muted] Basemap · {preset-name} ›    ← clickable; navigates editor to Scene B
[14px semibold] {Sublayer name}           ← e.g. "Roads"
[type pill] basemap sublayer
```

- The breadcrumb line (`Basemap · Positron ›`) is rendered as a separate element ABOVE the main title, not in the subtitle slot.
- Font size: 11px (not 12px — see Typography section). Color: `--muted-foreground`. Letter-spacing: `0.04em`.
- Clicking the breadcrumb line: swaps the editor content to Scene B WITHOUT changing the sidebar's selected row (the sidebar still highlights the sublayer; the editor navigates contextually).
- The `×` close button remains in top-right and collapses the editor entirely.

**Render as section:** OMIT — basemap sublayers do not have a render-mode control.

**Detail level section (expanded by default, replaces Render as):**
- Section label: "DETAIL LEVEL"
- Pill strip: `Off` | `Minimal` | `Default` | `Full`
  - Off: sublayer hidden entirely (equivalent to toggling the eye)
  - Minimal: e.g. major roads only (motorway + trunk)
  - Default: standard visibility rules for this sublayer
  - Full: maximum detail, including residential/local features
- Pill strip uses identical visual contract to Phase 1034 pill strip (12px, `--radius-full`, 5px 10px padding, `--surface-2` rest, `--primary` active)
- If the user has deviated from "Default", a hint below the pills reads: "{Sublayer name} is currently customized" (12px, `--muted-foreground`, italic).

**Stroke section (expanded by default):**
- Section label: "STROKE"
- Fields (same `.field` grid pattern as Phase 1034 — `110px 1fr auto`):
  - Color: `[color swatch + hex]`
  - Width: `[slider 0–8px] [value]`
  - Casing color: `[color swatch + hex]`
  - Casing width: `[slider 0–4px] [value]`
- Casing fields are relevant to road sublayers specifically; they are shown for all sublayers (not conditional).

**Visibility section:**
- Opacity + min zoom + max zoom (same as Phase 1034).

**Reset section (collapsed by default):**
- Collapsed section label: "RESET"
- Hint in collapsed state: "Reset to preset default"
- When expanded: a single "Reset to default" button (secondary ghost, full width). Confirms inline (no modal): "This will remove all custom appearance for {sublayer name}." with "Reset" (danger) + "Keep customization" (secondary).

**Footer:**
- `Back to basemap` (secondary ghost, full width). Clicking navigates the editor back to Scene B (does NOT close the editor; does NOT deselect the sublayer in the sidebar).
- No `Delete layer` — sublayers cannot be deleted.

| Rename group (blank) | Commit empty string in inline input | Silent revert to previous name; no toast, no error message. |

## Interaction Contract

### Group expand/collapse (all group types)

- Caret click: toggles expand/collapse. Does NOT select the row for editing. Does NOT trigger editor open/swap.
- Row body click (anywhere except caret, eye, kebab): selects row AND opens editor.
- Expansion state is saved to `groupMeta.{groupId}.expanded` immediately on toggle. This field lives on the saved-map JSON and is round-tripped through the Phase 1033 normalizer.
- On map load: apply saved expansion state before first paint. Default for groups without saved state: collapsed.

### Basemap sublayer navigation (breadcrumb)

- Clicking a basemap sublayer row in the sidebar: selects the sublayer (sidebar row gets selected state) AND opens editor to Scene C.
- Clicking the breadcrumb `"Basemap · {preset-name} ›"` inside Scene C: swaps editor content to Scene B. Does NOT change the sidebar's selected row highlight (sublayer remains highlighted in the sidebar — this is intentional so the user sees which sublayer they just navigated away from).
- Clicking the basemap group row itself in the sidebar while Scene C is open: selects the group row (sidebar row selection moves to group) AND swaps editor to Scene B.
- The `×` close button in the editor always collapses the editor entirely and deselects whatever was selected.

### DEM render-mode switch

- Selecting a different pill in the DEM "Render as" strip:
  1. Immediately swaps the Appearance section content (compass widget for Hillshade, terrain hint for Terrain, image hint for Image).
  2. Updates the layer's type icon glyph in the sidebar row (`▦` / `⛰` / `◬`).
  3. Writes `style_config.render_mode` to the in-memory layer state. Does NOT trigger a full map save; marks the map as "unsaved" if it was previously saved.
  4. Re-syncs the map layer: in Hillshade mode, the MapLibre layer switches to `raster-hillshade` type; in Terrain mode, wires `map.setTerrain({...})` with the layer's source; in Image mode, uses standard `raster` type.

- **Source binding preservation:** switching render modes does NOT change the DEM source. The same raster source URL is used across all three modes. Paint config for the previous mode is preserved in `style_config` under mode-keyed namespaces (e.g. `style_config.hillshade.*`, `style_config.terrain.*`); switching back to a mode restores its last saved values.

### Folder group create flow

1. User: Kebab → "Add to group…" → "＋ New group…"
2. System: Creates a new folder group with name "Group {N}" at the position immediately above the current layer in the stack. Moves the layer into the new group as its sole child. Immediately expands the group. Transition: the group name cell opens for inline rename (input is focused, text is fully selected).
3. User types new name (or presses Escape to keep "Group {N}").
4. Flow completes. Map is marked unsaved.

### Folder group rename flow (standalone — not from create)

1. User: Kebab → "Rename group"
2. System: Group name cell transitions to inline `<input>`. Existing name is pre-filled and fully selected. Focus moves to the input.
3. `Enter` or blur: commits new name (trimmed). Empty string reverts silently.
4. `Escape`: cancels, reverts to original name. Focus returns to the group row.

### `Add to group…` — add to existing group flow

1. User: Kebab → "Add to group…" → picks existing group from sub-list
2. System: Moves the layer into the selected group as the last child. Expands the group if it was collapsed. Sub-list closes. Map marked unsaved.

### `Move out of group` (child layer kebab)

1. Available only when the layer is a child of a user folder group.
2. System: Promotes the layer to the parent stack at the position immediately after the group row. Map marked unsaved. No confirmation.

### Drag-to-reorder with groups (inherits Phase 1034; cross-constraints)

- Basemap sublayers: cannot be dragged outside the basemap group. Drag attempt shows `cursor: not-allowed`. No drop indicator appears outside the basemap group boundary.
- User folder group children: CAN be dragged out (produces "move out of group" equivalent).
- Dragging a loose layer onto an expanded group's children area: treated as "add to group" at the dragged position within the children container.
- Dragging a loose layer onto a collapsed group row: adds as last child; group expands on drop.
- Group rows themselves CAN be drag-reordered in the stack (entire group moves as one unit, children follow).
