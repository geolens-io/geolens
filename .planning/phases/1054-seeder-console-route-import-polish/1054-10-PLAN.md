<objective>
The audit's EW-05 enhancement: the STAC Catalog probe is fast and clean, but the next-step user experience (committing assets) is potentially the most expensive operation in the product — a single Sentinel-2 tile is hundreds of MB, NAIP is multi-GB. Without a "stage → review → commit" confirmation, a misclick could initiate a 10GB download.

Per CONTEXT.md's locked decision (path b): "Best-effort estimate from STAC manifest — parse `assets.{key}.file:size` or `assets.{key}.alternate.size` from the STAC item JSON. Display the sum where available, with a '(estimate based on STAC manifest)' qualifier. Falls back to 'Total: N items (size unavailable)' when manifest doesn't include size."

Three coordinated changes:

1. **Backend `stac.py`**: Extract `data_asset.get("file:size")` from the STAC item JSON's primary data asset and propagate to the response as `data_asset_size_bytes`. STAC 1.0+ defines this as the size in bytes per the [STAC File Info Extension](https://github.com/stac-extensions/file).

2. **Frontend types + `StacItemSummary`**: Add `data_asset_size_bytes: number | null` to the type.

3. **Frontend `StacImportForm.tsx`**: Add a new `'confirm'` step between `'items'` selection and the existing `'importing'` step. The confirmation panel shows total bytes (formatted human-readable), per-item count, and the source qualifier. "Import N items" button stays on the items list but routes through the confirmation panel; from the confirmation panel a primary "Confirm and Import" button calls the same `handleImport` as today.

Purpose: Close EW-05 — users get a chance to abort a multi-GB download misclick.
Output: Backend STAC adapter update + type update + new wizard step + 4-locale i18n + regression test.
</objective>

    export function formatBytes(bytes: number | null | undefined): string {
      if (bytes == null || bytes < 0) return '—';
      if (bytes < 1024) return `${bytes} B`;
      const units = ['KB', 'MB', 'GB', 'TB'];
      let n = bytes / 1024;
      let i = 0;
      while (n >= 1024 && i < units.length - 1) { n /= 1024; i++; }
      return `${n.toFixed(n < 10 ? 1 : 0)} ${units[i]}`;
    }
