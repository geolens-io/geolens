<objective>
The audit's ROUTE-03 (LOW) finding: navigating to `/register` while already authenticated silently redirects to `/` with no toast/banner/feedback. A user investigating account-creation flow as an admin gets zero signal that they're being bounced.

Replace the silent `<Navigate to="/" replace />` with a visible toast (`sonner` is already used throughout the app for this pattern — `toast.info` or `toast.success` shape) followed by the redirect. Use a short `useEffect` that fires the toast once on mount + then redirects via `useNavigate()`. The render path returns null (or a `<LoadingState />`) for the one frame between mount and redirect.

Per CONTEXT.md's locked decision: "Either show a brief toast on redirect, or let the page render with an inline 'You're already signed in as admin — sign out to register a new account' notice." We use the toast variant — minimal blast radius, mirrors the `sonner` pattern already used elsewhere, no new banner component.

Purpose: Close ROUTE-03 — operators understand why they were redirected.
Output: Updated RegisterPage + 4-locale i18n key + new regression test.
</objective>
