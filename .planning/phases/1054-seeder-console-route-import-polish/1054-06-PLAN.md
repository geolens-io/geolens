<objective>
The audit's ROUTE-04 (LOW) finding: `/m/{invalid-token}` correctly renders the "Map not found" friendly view, but the browser DevTools console shows `[ERROR] Failed to load resource: 404` for the underlying `GET /api/maps/shared/{token}/` fetch. The error noise pollutes browser devtools and any automated console-error monitoring.

The recommended fix per the audit: "Catch the 404 in the fetch wrapper and suppress the console error for known-handled HTTP statuses on this specific endpoint."

Important caveat — the browser's network-tab `[ERROR]` log entry for a 404 response is **not something the application code can suppress** (the browser logs it before any JS sees the response). What the audit's playwright-mcp evidence likely captured is one of:
(a) An unhandled promise rejection from TanStack Query's `queryFn` throwing `ApiError(404)` — logged by RTQ's default error path.
(b) A `console.error` from `apiFetch`'s error-translation step (currently `apiFetch` just throws — no console.error — so this is unlikely).
(c) The network-tab log itself, which we cannot suppress.

This plan addresses (a): wire a `expected404: true` opt-in on `getSharedMap` that converts a 404 response into a typed "not-found" return value (`null`) instead of throwing `ApiError(404)`. The `useSharedMap` hook then treats `data === null` as the not-found case, and the existing `PublicViewerPage` 404 branch picks it up. No throw means TanStack Query has nothing to log, and no global error handler fires.

For case (c), if Phase 1056 live verification confirms the network-tab entry persists after this change, we accept it — the browser's native network-tab logging is outside the application's control, and the audit's recommendation is satisfied by removing the application-level error surface.

Purpose: Close ROUTE-04 — eliminate the application-layer 404 error surface for the share-token endpoint while preserving real-error visibility for other failure modes.
Output: Updated `apiFetch` (or sibling helper) + updated `getSharedMap` + regression test.
</objective>
