# Phase 999.2 Plan 02: Ingest Pipeline 3D Detection Wiring Summary

**detect_3d_metadata and promote_z_to_elev wired into _finalize_ingest and reupload_file; create_dataset stores is_3d/n_dims/z_min/z_max on every vector ingest**

## Performance

- **Duration:** ~15 min
- **Started:** 2026-04-12T00:00:00Z
- **Completed:** 2026-04-12
- **Tasks:** 2
- **Files modified:** 3

## Accomplishments

- `promote_z_to_elev()` added to `metadata.py`: for 3D Point/MultiPoint geometries, adds `elev double precision` column populated from `ST_Z(geom)`. Idempotent via column existence check.
- `create_dataset()` signature extended with `is_3d`, `n_dims`, `z_min`, `z_max` keyword args; all four passed to `Dataset()` constructor.
- `_finalize_ingest` now calls `detect_3d_metadata` after `extract_metadata`, then conditionally calls `promote_z_to_elev`, then passes 3D values into `create_kwargs`.
- `reupload_file` now calls `detect_3d_metadata` on the staging table before `_apply_reupload_swap`, and assigns `dataset.is_3d/n_dims/z_min/z_max` after the swap.

## Files Created/Modified

- `backend/app/datasets/service.py` - Extended `create_dataset()` with 4 new kwargs; passed to `Dataset()` constructor
- `backend/app/ingest/tasks.py` - Updated imports in `_finalize_ingest` and `reupload_file`; wired 3D detection calls in both paths

## Decisions Made

- `promote_z_to_elev` placed immediately after `detect_3d_metadata` in `metadata.py` for spatial co-location.
- 3D kwargs in `create_dataset` are keyword-only, placed after `source_url` and before `visibility`.
- In `reupload_file`, 3D fields are assigned directly on the `dataset` object after `_apply_reupload_swap`, consistent with how `geometry_type` and `feature_count` are already assigned there.
- `reupload_service` was not updated — plan scope explicitly covers `reupload_file` only. Deferred to future work.
