<objective>
Guard the DEM hillshade dual-consumer case (per D-POLISH-02). When a DEM layer is simultaneously bound to the map's terrain source (`terrainConfig.enabled && terrainConfig.source_dataset_id === layer.dataset_id`), `syncRasterLayer` in map-sync.ts must NOT spin up a second `raster-dem` hillshade consumer for that same DEM — the two `raster-dem` consumers with mismatched tile sizes cause MapLibre `backfillBorder "dem dimension mismatch"` console spam. `DEMEditorScene` surfaces a muted advisory note so the user understands why hillshade is suppressed.

Purpose: Stops console error spam on Map A without touching Map B's healthy hillshade path (predicate is false when terrain is off). Safety property: predicate TRUE only when `terrainConfig.enabled=true AND source_dataset_id === layer.dataset_id`.

Output: `isHillshadeTerrainBound` helper exported from map-sync.ts; `SyncOptions.terrainConfig` optional field; `syncRasterLayer` skips/suppresses the hillshade consumer when the predicate is true; `DEMEditorScene` renders a note conditional on the new optional `isTerrainBound` prop; 4-locale i18n key `demEditor.hillshadeTerrainNote`; unit tests for the predicate, the skip, and the note.
</objective>

```typescript
/**
 * POLISH-02: Returns true when the given DEM layer is already consumed by the
 * active terrain source. In this state, starting a second raster-dem consumer
 * (for hillshade) causes MapLibre backfillBorder "dem dimension mismatch" errors.
 * Guard: terrain must be enabled AND the same dataset powers both consumers.
 */
export function isHillshadeTerrainBound(
  layer: Pick<SyncLayerInput, 'dataset_id' | 'is_dem'>,
  terrainConfig: MapTerrainConfig | null | undefined,
): boolean {
  return (
    layer.is_dem === true &&
    terrainConfig?.enabled === true &&
    terrainConfig.source_dataset_id === layer.dataset_id
  );
}
```
