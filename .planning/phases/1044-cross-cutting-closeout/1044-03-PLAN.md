Drag simulation in Playwright (the tricky bit):
- dnd-kit pointer sensor: prefer `await page.mouse` ops with `dragHandle.boundingBox()` → `mouse.move(x,y)` → `mouse.down()` → small intermediate moves → `mouse.up()`.
- dnd-kit keyboard sensor: focus the drag handle → `keyboard.press('Space')` (lift) → `keyboard.press('ArrowDown'/'ArrowUp')` (navigate) → `keyboard.press('Space')` (drop) OR `keyboard.press('Escape')` (cancel).
- The keyboard sensor approach is MORE reliable in headless Chromium because it does not depend on pointer coordinate precision against MapLibre's canvas. Use keyboard sensor for the drag-from-catalog tests when feasible (per builder-unified-stack.spec.ts Test 1 which uses Space/Arrow/Space for in-stack reorder).
- For drag-from-catalog specifically, the source is in the modal and the target is the stack. The keyboard path is: focus handle in modal → Space (lift) → Tab/Arrow into the stack → Enter (drop). If keyboard cross-context drop is unreliable, fall back to pointer simulation with explicit `mouse.move` to the stack listbox bounding box.

Selection in Playwright:
- Cmd-click (macOS) or Ctrl-click (other platforms): `page.locator('[id^="stack-row-"]').nth(1).click({ modifiers: ['Meta'] })` — Playwright accepts 'Meta' (macOS Cmd), 'Control' (others). Use 'Meta' on darwin runner; the project's Playwright base headless Chromium runs in linux container in CI but key labels are Chromium-native. Project pattern: use `process.platform === 'darwin' ? 'Meta' : 'Control'`.
- Shift+click: `.click({ modifiers: ['Shift'] })` — same across platforms.
