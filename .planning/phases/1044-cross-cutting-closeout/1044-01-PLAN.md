```
describe('i18n resources', () => {
  it('keeps locale key parity with the English fallback bundles', async () => {
    const resources = await loadAllResources();
    for (const ns of namespaces) {
      const englishKeys = flattenKeys(resources.en[ns]).sort();
      for (const lng of supportedLngs) {
        expect(flattenKeys(resources[lng][ns]).sort()).toEqual(englishKeys);
      }
    }
  });
});
```
- `supportedLngs` is `['en','de','es','fr']` per `frontend/src/i18n/config.ts`.
- The parity gate compares FLATTENED key sets — value content is irrelevant to the gate, but the verification step also asserts native content for a known sample to avoid silent passthrough.
- All four locale files MUST be valid JSON. Trailing commas, comments, and duplicate top-level keys break the loader.
