<objective>
Close SEC-S09 (MEDIUM, CVSS 5.0). The dataset-export endpoint (`backend/app/processing/export/router.py:40-99`) accepts a `where` query parameter that flows to `ogr2ogr -where`. The existing `validate_where_clause()` (`backend/app/processing/export/service.py:49-77`) only validates that *identifiers* extracted by `re.findall(r"[A-Za-z_][A-Za-z0-9_]*", where)` are columns of the exported dataset. SQL operators, subqueries on the same table, and UNION-via-bound-columns are **not parsed** — the validator can't reject UNION grammar because it never builds an AST.

The audit recommends reusing the existing `app.platform.sandbox.validator.validate_sql` (which parses with sqlglot and rejects non-SELECT/multi-statement/blocked-function). That validator parses **full SQL statements** (SELECT/UNION/INTERSECT/EXCEPT). The export `where` is a **WHERE-clause fragment**, not a statement — so we cannot reuse `validate_sql` directly. Instead, we adapt the same sqlglot approach to fragment-shaped input: wrap the user's `where` in `SELECT 1 FROM t WHERE <where>`, parse it, and walk the AST asserting the WHERE node only contains whitelisted expression types.

Purpose:
- Reject UNION-via-bound-columns and other SQL grammar that the identifier-only regex cannot detect.
- Block subqueries (which can leak data via `EXISTS (SELECT ... FROM data.other_table WHERE ...)` if the table name happens to use the same columns).
- Establish a reusable AST-based fragment validator for future `where`-shaped inputs (filter expressions on other endpoints).

Output:
- New `backend/app/processing/export/where_validator.py` module with `validate_where_ast(where: str) -> None` — parses `SELECT 1 FROM _t WHERE <where>` via sqlglot postgres dialect, then walks the WHERE node and raises `ValueError` if any disallowed expression type is present.
- `validate_where_clause` in `service.py` now calls `validate_where_ast(where)` BEFORE the existing identifier check. The identifier check stays — defense-in-depth.
- Backend pytest in `test_export_where_validator.py` with 30+ test cases covering the whitelist (pass) and blocklist (reject).
- e2e `sec-audit.spec.ts` S09 test (already drafted at lines 206-212) passes.

Allowed expression types in WHERE:
- `Column` (column references)
- `Literal` (numeric, string, boolean)
- `EQ`, `NEQ`, `LT`, `LTE`, `GT`, `GTE` (comparison)
- `And`, `Or`, `Not` (logical)
- `In`, `Is`, `Like` (containment / null / pattern)
- `Between`
- `Paren` (parenthesized subexpression)
- `Neg`, `Pos` (unary +/-)

Disallowed (any of these raises ValueError):
- `Select`, `Subquery`, `Union`, `Intersect`, `Except` (subqueries / set ops)
- `Func`, `Anonymous` (any function call — incl. `pg_sleep`, `pg_read_file`, even `lower()`)
- `Insert`, `Update`, `Delete`, `Create`, `Drop`, `Alter` (DDL/DML — sqlglot will catch these in the multi-statement check)
- `Star` (SELECT * tries to slip through)
- Anything not in the allowlist (deny-by-default — `getattr` fallback)

sqlglot fragment parsing approach:
```python
import sqlglot
from sqlglot import exp

def _parse_where_fragment(where: str) -> exp.Expression:
    wrapped = f"SELECT 1 FROM _t WHERE {where}"
    try:
        statements = sqlglot.parse(wrapped, dialect="postgres")
    except sqlglot.errors.ParseError as e:
        raise ValueError(f"Invalid WHERE syntax: {e}")
    if len(statements) != 1 or not isinstance(statements[0], exp.Select):
        raise ValueError("Only a single WHERE expression is allowed")
    where_node = statements[0].args.get("where")
    if where_node is None:
        raise ValueError("Empty WHERE expression")
    return where_node.this  # unwrap exp.Where -> the actual expression
```

Allowlist walk:
```python
ALLOWED = (
    exp.Column, exp.Literal, exp.Boolean, exp.Null,
    exp.EQ, exp.NEQ, exp.LT, exp.LTE, exp.GT, exp.GTE,
    exp.And, exp.Or, exp.Not,
    exp.In, exp.Is, exp.Like, exp.ILike, exp.Between,
    exp.Paren, exp.Neg,
)

def _walk_allowlist(node: exp.Expression) -> None:
    if not isinstance(node, ALLOWED):
        raise ValueError(f"Disallowed expression in WHERE: {type(node).__name__}")
    for child in node.walk():
        if child is node:
            continue
        if not isinstance(child, ALLOWED):
            raise ValueError(f"Disallowed expression in WHERE: {type(child).__name__}")
```

UNION attack walkthrough:
- Input: `where=gid > 0 UNION SELECT 1, 2, 3`
- Wrapped: `SELECT 1 FROM _t WHERE gid > 0 UNION SELECT 1, 2, 3`
- sqlglot parses this as a Union of two Selects. `statements[0]` is `exp.Union`, NOT `exp.Select`. The early `isinstance(statements[0], exp.Select)` check rejects with "Only a single WHERE expression is allowed". 

Subquery attack walkthrough:
- Input: `where=gid IN (SELECT 1 FROM users)`
- Wrapped: `SELECT 1 FROM _t WHERE gid IN (SELECT 1 FROM users)`
- Parses as a single Select. `where_node.this` is an `exp.In`. Walking, we find an `exp.Subquery` (the sub-select) → rejected by allowlist. 

Function-call attack walkthrough:
- Input: `where=pg_sleep(10)`
- Wrapped: `SELECT 1 FROM _t WHERE pg_sleep(10)`
- Parses. `where_node.this` is `exp.Anonymous` (or `exp.Func`). Not in allowlist → rejected.

    ```python
    """AST-based WHERE-clause validator (SEC-S09).

    Wraps the user-supplied `where` fragment in `SELECT 1 FROM _t WHERE <fragment>`
    and parses with sqlglot postgres dialect. Walks the resulting WHERE node and
    raises ValueError if any expression type outside the strict allowlist appears.

    This is a peer-companion to `app.platform.sandbox.validator.validate_sql` —
    same parser, same dialect, but allowlist-based (statement-level validator is
    blocklist-based because the input shape is broader). The two share no code
    paths; cross-cutting refactor not justified for SEC-S09 scope.
    """
    from __future__ import annotations

    import sqlglot
    from sqlglot import exp

    ALLOWED_EXPRESSIONS: tuple[type, ...] = (
        exp.Column, exp.Identifier,
        exp.Literal, exp.Boolean, exp.Null,
        exp.EQ, exp.NEQ, exp.LT, exp.LTE, exp.GT, exp.GTE,
        exp.And, exp.Or, exp.Not,
        exp.In, exp.Is, exp.Like, exp.ILike, exp.Between,
        exp.Paren, exp.Neg,
        exp.Where,  # the top-level WHERE node itself
    )

    def validate_where_ast(where: str) -> None:
        """Validate that `where` is a safe WHERE-clause fragment.

        Raises ValueError on disallowed grammar.
        """
        if not where or not where.strip():
            raise ValueError("Empty WHERE expression")

        wrapped = f"SELECT 1 FROM _t WHERE {where}"
        try:
            statements = sqlglot.parse(wrapped, dialect="postgres")
        except sqlglot.errors.ParseError as e:
            raise ValueError(f"Invalid WHERE syntax: {e}")

        statements = [s for s in statements if s is not None]
        if len(statements) != 1:
            raise ValueError("Only a single WHERE expression is allowed")

        stmt = statements[0]
        if not isinstance(stmt, exp.Select):
            raise ValueError("Only a single WHERE expression is allowed")

        where_node = stmt.args.get("where")
        if where_node is None:
            raise ValueError("Empty WHERE expression")

        # Walk every node in the WHERE subtree; reject anything outside allowlist.
        for node in where_node.walk():
            if not isinstance(node, ALLOWED_EXPRESSIONS):
                raise ValueError(
                    f"Disallowed expression in WHERE clause: {type(node).__name__}"
                )
    ```
