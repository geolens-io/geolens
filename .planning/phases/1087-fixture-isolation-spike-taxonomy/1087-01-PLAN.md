**JUnit rootdir/classname contract (important — drives the parser shape below):**
- pytest is invoked from `cd backend && uv run pytest tests/`, so pytest's rootdir is `backend/` and `testpaths = ["tests"]` in `backend/pyproject.toml`.
- JUnit XML emits `classname` as a dotted path RELATIVE TO ROOTDIR — i.e. `tests.test_x.TestY` (class form) or `tests.test_x` (function form), NOT `backend.tests.test_x.TestY`.
- The parser MUST prepend `backend/` to the path component when synthesizing the pytest node-ID, because every other consumer in the spike (validator regex, Section 3 row regex, audit cross-reference) expects `backend/tests/...`.

(1) JUnit XML parser — write a small Python helper inline (no committed file) that extracts every failing testcase from `/tmp/v1020-junit.xml` and emits one JSON object per failure to `/tmp/v1020-failure-inventory.json`. The `backend/` prefix is prepended to the path string in BOTH branches of the class/function detection so downstream consumers see `backend/tests/test_x.py::TestY::test_z` (class) or `backend/tests/test_x.py::test_z` (function):
```
python3 <<'PY' > /tmp/v1020-failure-inventory.json
import xml.etree.ElementTree as ET, json, sys
root = ET.parse('/tmp/v1020-junit.xml').getroot()
out = []
for tc in root.iter('testcase'):
    fail = tc.find('failure')
    err = tc.find('error')
    if fail is None and err is None:
        continue
    node = fail if fail is not None else err
    # Extract error class — first line of message attr or first line of text
    msg = (node.get('message') or '').strip()
    text = (node.text or '').strip()
    first_line = msg.split('\n')[0] if msg else text.split('\n')[0]
    # path::TestClass::test_name node-ID
    classname = tc.get('classname', '')
    name = tc.get('name', '')
    # JUnit's classname is dotted form relative to pytest rootdir (=backend/).
    # rootdir-relative class form:    tests.test_x.TestY        -> backend/tests/test_x.py::TestY::test_name
    # rootdir-relative function form: tests.test_x              -> backend/tests/test_x.py::test_name
    # We MUST prepend 'backend/' so downstream regex + Section 3 row matchers see the full path.
    if '.' in classname:
        parts = classname.split('.')
        # Heuristic: last part is class if PascalCase, otherwise it's part of the path
        if parts[-1] and parts[-1][0].isupper():
            cls = parts[-1]
            path = 'backend/' + '/'.join(parts[:-1]) + '.py'
            node_id = f"{path}::{cls}::{name}"
        else:
            path = 'backend/' + '/'.join(parts) + '.py'
            node_id = f"{path}::{name}"
    else:
        # Fallback for classname without dots (rare — typically conftest-level or module-level fixture errors)
        # Prepend backend/ if the classname already looks like a path component; otherwise emit raw.
        if classname.startswith('tests'):
            node_id = f"backend/{classname}::{name}" if name else f"backend/{classname}"
        else:
            node_id = f"{classname}::{name}" if classname else name
    # Error class — extract the exception type from the message head (e.g. "asyncpg.exceptions.CannotConnectNowError: ...")
    err_class = first_line.split(':')[0].strip() if ':' in first_line else first_line[:80]
    snippet = first_line[:200]
    out.append({
        'node_id': node_id,
        'error_class': err_class,
        'snippet': snippet,
        'is_error': err is not None,  # vs assertion failure
    })
print(json.dumps(out, indent=2))
PY
```
