## Component Inventory

New controls for this phase, all inside existing editor scenes. No new top-level components.

### EDITOR-DEM-04: Contour Lines section (inside DEMEditorScene, hillshade and terrain modes)

**Section header:** `CONTOUR LINES` â€” same `text-[10px] font-semibold uppercase tracking-[0.08em] text-muted-foreground mb-2` as all other section headers.

**Controls (in order):**

1. **Enable toggle** â€” `Switch` (shadcn/Radix) with label "Contour lines", right-aligned. Layout: `flex items-center justify-between`. When off, the remaining controls collapse (conditional render, not hidden with CSS â€” consistent with `fillEnabled` pattern in `FillEditor.tsx`).

2. **Interval slider** â€” `SliderRow` (existing shared component from `HeatmapStyleControls.tsx` or the local `SliderRow` in `DEMEditorScene.tsx`). Label "Interval", min=10, max=500, step=10, suffix=" m". Reads/writes `_contour-interval` on `paint`. Default: 100.

3. **Color picker** â€” `StyleColorPicker` (existing). Label "Color". Reads/writes `_contour-color` on `paint`. Default: `#555555`.

4. **Weight slider** â€” `SliderRow`. Label "Weight", min=0.5, max=4, step=0.5, suffix=" px". Reads/writes `_contour-weight` on `paint`. Default: 1.

**Placement:** New section between APPEARANCE and VISIBILITY sections. Visible only when `mode === 'hillshade' || mode === 'terrain'`. Hidden entirely in `image` mode (no section rendered).

**Assumption:** Contour overlay is implemented as a separate MapLibre `line` layer derived from DEM data (frontend-side generation or backend contour tile endpoint â€” scope TBD at plan-phase). Paint props are stored as builder-private underscore keys on the DEM layer's `paint` dict, consistent with `_contour-*` naming convention used by other builder-internal props (e.g. `_heatmap-ramp`, `_outline-color`).

---

### EDITOR-DEM-05: Hypsometric Tint section (inside DEMEditorScene, hillshade and terrain modes)

**Section header:** `HYPSOMETRIC TINT` â€” same header style as all other sections.

**Controls:**

1. **Enable toggle** â€” `Switch` with label "Elevation tint". When off, remaining controls collapse.

2. **Ramp picker** â€” Reuses the existing `ColorRampPicker` component. `mode="graduated"`. Shows the curated sequential + diverging ramp set (`SEQUENTIAL_RAMPS` + `DIVERGING_RAMPS` from `frontend/src/lib/color-ramps.ts`). Reads/writes `_hypso-ramp` on `paint`. Default ramp: `'Viridis'`.

   The picker is presented inline (not in a Popover) â€” same pattern as `HeatmapStyleControls.tsx` `ColorRampPicker` usage. The `max-h-40 overflow-y-auto` container is already built into `ColorRampPicker`.

**Placement:** New section after the CONTOUR LINES section and before VISIBILITY. Same mode gating as EDITOR-DEM-04 (hillshade/terrain only).

**Assumption:** Hypsometric tint is applied as a MapLibre color expression on the hillshade layer using the selected ramp's stop colors mapped across the elevation range. Paint key `_hypso-ramp` stores the ramp name. The tint only applies in hillshade mode (terrain mode renders the DEM surface via MapLibre terrain, not a paint layer). Section may be conditionally hidden in terrain mode â€” document as a plan-phase decision; default is to show for hillshade mode, hide for terrain mode.

### EDITOR-RASTER-COLORMAP: Colormap section (inside RasterEditor, for single-band rasters)

**Section header:** `COLORMAP` â€” same header style.

**Visibility condition:** This section renders only when the raster layer has `band_count === 1` (single-band). The executor detects this from `layer.dataset?.band_count` or equivalent API field. Multi-band rasters show no colormap section.

**Controls (in order):**

1. **Colormap select** â€” `Select` (shadcn, `h-8 text-xs`). Label: "Colormap" (left label span `w-28 shrink-0 text-xs text-muted-foreground`). Options list a curated subset of Titiler built-in colormap names. Default: `'gray'`. Reads/writes `_colormap` on `paint`.

   Curated option list (display name / Titiler name):
   - Grayscale / `gray`
   - Viridis / `viridis`
   - Inferno / `inferno`
   - Plasma / `plasma`
   - Magma / `magma`
   - Yellow-Red / `ylorrd`
   - Blue-Green / `bugn`
   - Terrain / `terrain`

   Assumption: This list is fixed at design time; plan-phase confirms available Titiler colormap names. Rendered as a standard `SelectTrigger` + `SelectContent` + `SelectItem` stack, same as the `Select` in `LineEditor.tsx` for cap/join.

2. **Stretch type select** â€” `Select` (same anatomy). Label: "Stretch". Options:
   - Min/Max / `minmax`
   - Percentile (2â€“98%) / `percentile`
   - Standard deviation / `stddev`

   Default: `'minmax'`. Reads/writes `_stretch` on `paint`.

**Placement:** New section appended after the existing APPEARANCE section (brightness/contrast/saturation/hue sliders) in `RasterEditor.tsx`. The existing Reset collapsible moves to the bottom, after the COLORMAP section.

**Tile re-render on change:** Changing either control triggers a new tile URL (query params `colormap_name` and/or `rescale` strategy). This is a URL-level change, not a MapLibre paint property change. The control writes to `paint` (via `onPaintProp`); the raster adapter/tile URL builder reads `_colormap` and `_stretch` to construct the Titiler URL. The tile source `tiles` array is updated, causing MapLibre to re-fetch tiles. This behavior is documented here as a design constraint; implementation details are scoped at plan-phase.

## Interaction Contracts

| Control | Interaction | Expected Response |
|---------|-------------|-------------------|
| Contour enable Switch | Toggle | Remaining contour controls mount/unmount immediately (no animation); map layer appears/disappears |
| Contour interval Slider | Drag | Debounce via `coalesceFrame` (existing pattern from `RasterEditor.tsx`) before dispatching paint change |

| Hypsometric tint enable Switch | Toggle | Remaining tint controls mount/unmount; map color expression applied/removed |
| Hypsometric ramp ColorRampPicker | Row click | Immediate selection; `ring-1 ring-primary` applied to selected row; paint key updated |
| Colormap Select | Value change | Paint prop `_colormap` updated â†’ tile URL rebuilt â†’ MapLibre re-fetches tiles |

## Assumptions (Autonomous Mode Decisions)

The following decisions were made autonomously per `<important_autonomous_mode>` instructions.
Each is flagged for plan-phase review.

| ID | Decision | Rationale |
|----|----------|-----------|
| A-01 | Contour/hypsometric sections show only in hillshade and terrain modes, not image mode | Image mode has no elevation context for contours or tinting; consistent with the existing image-mode hint ("No additional appearance controls for image mode") |
| A-02 | Hypsometric tint is hidden (not shown with a hint) in terrain mode | Terrain mode uses MapLibre terrain 3D â€” color expressions apply to the hillshade layer, not the 3D terrain surface. The hint copy is defined if plan decides to show-with-hint instead |
| A-03 | Colormap section gates on `band_count === 1` | Multi-band rasters render as RGB; colormap remapping only makes sense for single-band scientific/elevation data |
| A-04 | `_colormap` and `_stretch` stored in paint as builder-private underscore keys | Consistent with `_heatmap-ramp`, `_hypso-ramp`, `_contour-*` convention; does not leak to MapLibre paint layer directly |
| A-05 | Titiler colormap names are lowercase (`gray`, `viridis`, etc.) | Titiler's built-in colormap registry uses lowercase; executor must verify exact names at plan-phase research step |
| A-06 | Contour lines control placement order: toggle â†’ interval â†’ color â†’ weight | Toggle gating the group is always first; interval is the most important numeric control; color and weight are secondary styling |
| A-07 | Hypsometric tint uses the existing `ColorRampPicker` inline (no Popover wrapping) | Matches `HeatmapStyleControls.tsx` usage; the ramp picker already has `max-h-40 overflow-y-auto` scroll handling |
| A-08 | Reset collapsible in `RasterEditor.tsx` remains; COLORMAP section goes above it | Preserves existing Reset affordance; colormap is functional appearance, not a reset trigger |
