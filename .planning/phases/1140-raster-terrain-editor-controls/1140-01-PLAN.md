<objective>
Add the backend half of EDITOR-RASTER-COLORMAP: let the raster tile proxy accept and forward a validated `colormap_name` + `stretch` to Titiler, fix the nginx raster-tiles location so those params survive to the api and into the cache key, and add `band_count` to `MapLayerResponse` so the frontend can gate the colormap control to single-band rasters.

Purpose: The colormap control is the only requirement in this phase with a backend dependency. The frontend plan (04) cannot build the gated control or trigger tile re-render until the API accepts these params and reports `band_count`. The allowlist validation is a security gate (T-1140-01) — `colormap_name` is user-controlled and forwarded into a Titiler URL, so it must be constrained to the 8 confirmed names, never passed through arbitrarily.

Output: `raster_tile_proxy` with two validated optional params; nginx forwarding + cache-key fix; `band_count` on `MapLayerResponse` (backend schema + populated in the layer-row query); a focused pytest. NO OpenAPI/SDK regeneration here — that is a Phase 1143 close-gate task (STATE.md HARD INVARIANT 6); flag it in the SUMMARY.
