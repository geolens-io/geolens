### Step 1 — Claim the `@geolens` npm organization (D-00)

**Why:** `npm publish @geolens/sdk` fails with a scope-not-found error if the org doesn't exist. Required prerequisite for the granular token (Step 4) and for Plan 03's hot publish. Confirmed missing as of 2026-05-02 via `https://registry.npmjs.org/-/org/geolens` → `{"code":"ResourceNotFound"}`.

**Do:**
1. Open <https://www.npmjs.com/org/create> in your browser, signed in as the npm account that owns `geolens-io`.
2. **Organization name:** `geolens` (lowercase, exactly as written — this claims the `@geolens` scope).
3. **Plan:** Free.
4. Click **Create**.
5. Confirm your account is listed as an admin (it should be by default since you created it).

**Verify:**
```bash
curl -s https://registry.npmjs.org/-/org/geolens | python3 -c "import sys, json; d = json.load(sys.stdin); print('OK — org exists, member count:', len(d.get('users', d)))" 2>/dev/null \
  || echo "FAILED — org not found, repeat Step 1"
# Or visit https://www.npmjs.com/org/geolens — should render an org page, not a 404.
```

Expected: a valid JSON response (no `ResourceNotFound`) and the page renders.

### Step 2 — Configure PyPI Trusted Publishing for `geolens-sdk` (D-01)

**Why:** `uv publish --trusted-publishing automatic` (now in publish-sdks.yml after Plan 01) requires PyPI to recognize the GitHub workflow as a trusted publisher. Pending publishers work BEFORE the project exists — PyPI converts the pending publisher to a live publisher on first OIDC handshake AND creates the project simultaneously (RESEARCH §"Order of Operations").

**Do:**
1. Sign in to <https://pypi.org> with the maintainer account.
2. Navigate to <https://pypi.org/manage/account/publishing/>.
3. Scroll to **"Add a new pending publisher"**.
4. Select **GitHub** as the publisher type.
5. Fill the form EXACTLY:

| Field | Value |
|-------|-------|
| PyPI project name | `geolens-sdk` |
| Owner | `geolens-io` |
| Repository name | `geolens` |
| Workflow name | `publish-sdks.yml` |
| Environment name | *(leave blank)* |

6. Click **Add**.

**Verify:** The new pending publisher appears in the list at <https://pypi.org/manage/account/publishing/>. Take a screenshot or copy the entry text into a paste buffer for Step 6 (audit trail).

### Step 4 — Generate the npm Granular Access Token (D-02)

**Why:** npm OIDC trusted publishing is still beta as of 2026-05; npm permanently removed the "Automation" token type on 2025-12-09 (RESEARCH §State of the Art). The replacement is a granular access token with Bypass 2FA + Read/Write. Token scope MUST be the `@geolens` org (NOT `@geolens/sdk` — the package picker won't show `@geolens/sdk` until first publish, per Pitfall 3).

**Do:**
1. Open <https://www.npmjs.com/settings/~/tokens> (signed in as the maintainer account).
2. Click **Generate New Token** → **Granular Access Token**.
3. Fill the form:

| Setting | Value |
|---------|-------|
| Token name | `geolens-sdk-ci-publish` |
| Description | `Phase 228 — initial publish credential; rotates 90d` |
| Token expiration | **Maximum** (90 days from today) |
| Allowed IP ranges | *(leave default — no restriction)* |
| Permissions: Packages and scopes | **Read and write** |
| Packages and scopes scoping | Select **Selected packages and scopes** → search **@geolens** → check the `@geolens` org scope |
| Permissions: Organizations | (no organization permissions needed) |
| Bypass two-factor authentication | **CHECK THIS BOX** (required for non-interactive CI per RESEARCH §npm Token Runbook Step 2) |

4. Click **Generate Token**.
5. **COPY THE TOKEN IMMEDIATELY** — npm shows it only once. Paste it into a temporary location (you'll use it in Step 5).

**Set a calendar reminder for ~85 days from today to rotate this token.** The 90-day max is enforced by npm.

| Maintainer keyboard → PyPI web UI | Pending publisher form; binds workflow filename to project name |
| Maintainer keyboard → GitHub Settings | Secret creation surface (gh CLI or web UI); secret value never re-readable after set |
| Local clipboard → `gh secret set` stdin | Token transits clipboard for ~10s during Step 5 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-228-CKPT-01 | Information Disclosure | NPM_TOKEN visible in clipboard during paste | accept | Token is single-use during Step 5 paste; clear clipboard after. No persistent storage. The 90-day rotation cycle limits damage if leaked. Document rotation in Plan 04 docs/sdks.md update. |
