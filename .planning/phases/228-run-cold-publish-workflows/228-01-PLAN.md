```yaml
# Clean-machine smoke verification of published GeoLens packages.
# Phase 228 ships this alongside the first hot publish.
# Jobs run inside fresh Docker images (python:3.13-slim, node:22-slim) — no
# repo context, no runner pollution; mimics what a downstream user gets.
# See docs/sdks.md (§Publishing) for context.
name: Verify Published Packages

on:
  workflow_dispatch:
    inputs:
      version:
        description: 'Package version to verify (default: latest)'
        required: false
        type: string
        default: 'latest'
  push:
    tags:
      - 'release/*'

permissions:
  contents: read

jobs:
  verify-python:
    name: Verify Python packages on python:3.13-slim
    runs-on: ubuntu-latest
    steps:
      - name: pip install + smoke import (geolens-sdk + geolens CLI)
        run: |
          docker run --rm python:3.13-slim sh -c "
            pip install --no-cache-dir geolens-sdk geolens &&
            geolens --version &&
            python -c 'from geolens_sdk import GeolensClient; print(GeolensClient)'
          "

  verify-typescript:
    name: Verify TypeScript package on node:22-slim
    runs-on: ubuntu-latest
    steps:
      - name: npm install + smoke import (@geolens/sdk)
        run: |
          docker run --rm node:22-slim sh -c "
            npm install --no-save @geolens/sdk &&
            node -e 'import(\"@geolens/sdk\").then(m => console.log(typeof m.createGeolensClient))'
          "
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → PyPI | OIDC token (short-lived, auto-issued by Actions) crosses to PyPI's token-exchange endpoint |
| GitHub Actions runner → npm registry | `NODE_AUTH_TOKEN` (long-lived granular token, 90-day max) crosses to npm publish API |
| Workflow YAML → repo secrets | Workflow files reference `secrets.NPM_TOKEN` by name; secret values are GitHub-encrypted at rest, masked in logs |
| Untrusted PR fork → publish workflow | NOT a boundary — `workflow_dispatch` is maintainer-only; fork PRs cannot trigger |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-228-01 | Information Disclosure | `secrets.NPM_TOKEN` (long-lived granular token) | mitigate | GitHub auto-masks secret values in logs; token has Bypass-2FA + Read/Write scoped to `@geolens` org only; 90-day max lifetime forces rotation cadence (documented in docs/sdks.md by Plan 04). PYPI_TOKEN attack surface eliminated entirely (D-01 Trusted Publishing — no long-lived secret to leak). |
| T-228-02 | Tampering / Spoofing | Package name squatting before first publish | mitigate | Pre-flight gate (this plan, both publish workflows) checks `pip index versions` and `npm view` before each hot publish; logs `::warning::` if name is already taken. PyPI Trusted Publishing's repository-binding ensures only this repo's configured workflow can publish under `geolens-sdk` / `geolens`. RESEARCH §Open Question 2 notes the gate could be tightened to a hard `exit 1` after first publish; current shape is informational warning to allow re-publish. |
| T-228-03 | Tampering | Package poisoning via untrusted build dependencies | accept | Build deps (`hatchling`, `uv`, `npm`) are pinned in workflow toolchain (`setup-uv@v6` with `version: "0.10.2"`, `setup-node@v6` with `node-version: 22`). PyPI Trusted Publishing emits attestations automatically (visible on PyPI release page after publish in Plan 03). Code-signing of artifacts deferred to Phase 999.15 (SBOM phase) per CONTEXT.md Out of Scope. |
| T-228-04 | Tampering | Clean-machine install bypass (verify-published runs against poisoned image) | mitigate | `verify-published.yml` pulls `python:3.13-slim` and `node:22-slim` fresh from Docker Hub each run (no caching across runs by default). Verification jobs are stateless and use no repo-side artifacts — they install from PyPI/npm registries only, mimicking a real downstream user. |

No `high` severity threats remain unmitigated. Block-on-`high` passes.
