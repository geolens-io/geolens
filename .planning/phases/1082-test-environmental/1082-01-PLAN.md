<objective>
Close TD-04 by replacing the live `ogrinfo` subprocess invocation in `test_owner_gets_non_404_on_service_preview` with a mock so the test does not depend on host tooling (`gdal-bin` / `ogrinfo`). The test pins the IDOR/auth invariant (owner of a private dataset is NOT rejected by `check_dataset_access` on the service-preview path → response is 400 or 502, NEVER 404). The `ogrinfo` call inside `run_service_preview` is incidental plumbing — its failure mode (raising `IngestionError`, which the handler converts to 502) is what the test asserts on. Mocking `run_service_preview` to raise `IngestionError` reproduces the production 502 path deterministically, regardless of whether `gdal-bin` is installed on the host.

Purpose: restore green pytest signal for the owner-allowed IDOR sanity check on macOS developer hosts (the v1017 audit's named failure environment), without weakening the IDOR coverage or the production gate. Test-only fix; zero production code touched. Mirrors the Plan 1081-02 lazy-import patch target rule by patching the DEFINING module (`app.modules.catalog.sources.preview.run_service_preview`), even though the caller's import at `router_reupload.py:44` is module-top (single canonical patch target across the v1018 sweep).

Output: one atomic commit modifying `backend/tests/test_reupload_idor.py` only — adds an AsyncMock patch + extends the docstring with the disposition rationale, asserts the same 400/502/non-404 invariant as before, exits pytest 0 on a host without `gdal-bin`.
