# Phase 1024 Plan: MapLibre Point Cluster Renderer

**Phase:** 1024 — maplibre-point-cluster-renderer
**Milestone:** v1005 Builder Point Cluster Foundation
**Status:** Complete
**Requirements:** CLUS-01, CLUS-02, CLUS-03, CLUS-04, CLUS-05

## Goal

Consume the bounded GeoJSON source contract from Phase 1023 with a native MapLibre point-cluster adapter, while preserving the existing point/vector-tile fallback and existing map-layer schema.

## Implementation

1. Add a `cluster` layer adapter.
   - Use one GeoJSON source with MapLibre `cluster`, `clusterRadius`, and `clusterMaxZoom` source options.
   - Emit stable cluster-circle, cluster-count, and unclustered-point layers.
   - Keep the unclustered point layer on the parent `layerId` so popup/query identity remains stable.

2. Extend map-sync source handling.
   - Resolve `style_config.render_mode === "cluster"` to the cluster adapter only when bounded GeoJSON data exists.
   - Fall back to the normal circle/vector-tile path if bounded GeoJSON is unavailable.
   - Replace source/layer sets when switching between vector and GeoJSON source types.

3. Preserve parent lifecycle.
   - Visibility, opacity, filter, zoom range, reorder, and stale cleanup must apply to cluster companions.
   - Labels should not create an extra label layer for cluster mode; cluster count is the label companion.

4. Add focused coverage.
   - Adapter tests for layer specs, source-layer omission on GeoJSON, styling, visibility, and cleanup IDs.
   - Map-sync tests for clustered GeoJSON source options, fallback to vector circle, source replacement, zoom range, reorder, and stale cleanup.
