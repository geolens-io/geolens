# i18n Contributor Guide

GeoLens keeps its frontend i18n source of truth in `src/i18n/config.ts`, `src/i18n/resources.ts`, and `src/i18n/options.ts`.

## Current Namespaces

| Namespace | Scope |
|-----------|-------|
| `common` | Shared labels, buttons, states, enums |
| `admin` | Admin dashboard, settings, user management |
| `auth` | Login, registration, password flows |
| `builder` | Map builder UI, layer controls, styling |
| `collections` | Collection browsing, membership, cards |
| `dataset` | Dataset detail tabs, metadata, structure |
| `import` | File upload, service import, preview |
| `search` | Search bar, filters, facets, results |

## Add a Namespace

1. Add the namespace name to `namespaces` in `src/i18n/config.ts`.
2. Create `src/i18n/locales/<language>/<namespace>.json` for every entry in `supportedLngs`.
3. Import the namespace in components with `useTranslation('<namespace>')`.
4. If the strings are shared enum/state labels, prefer putting them in `common.json` and reading them through `src/i18n/labels.ts`.
5. Run `npm run test:i18n`.

## Add a Language

1. Add the language code to `supportedLngs` and `languageNames` in `src/i18n/config.ts`.
2. Create a new `src/i18n/locales/<language>/` directory.
3. Add one JSON bundle per namespace listed in `namespaces`.
4. Add or update the matching root README translation (`README.<language>.md`) unless the language is `en`.
5. Run `npm run test:i18n` to verify key parity and `npm run check:i18n:changed` if you changed locale bundles in the current branch.
6. Run `npm run check:readme-locales` from the repository root to verify public README coverage.

## Public README Translations

The root README is the canonical English source. GeoLens also keeps localized root README files for every non-English language in `supportedLngs`:

| Language | File |
|----------|------|
| Spanish | `README.es.md` |
| French | `README.fr.md` |
| German | `README.de.md` |

Each README should keep the same language navigation line so GitHub visitors can switch languages quickly. The repository-root `npm run check:readme-locales` script parses `src/i18n/config.ts` and fails when a supported locale has no matching README file or a README is missing a language-link marker.

## Notes

- App init and test init should stay in sync through `src/i18n/options.ts` instead of duplicating options inline.
- Shared labels for API/state enums should live in `common.json` and go through `src/i18n/labels.ts` so components do not each invent their own fallback logic.
