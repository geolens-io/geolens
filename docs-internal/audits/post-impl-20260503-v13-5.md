## Seam Quality

Seam Quality grades **A**.

### PermissionExtension — Ready

`PermissionExtension` is a real seam, not a paper protocol:

- Protocol: `backend/app/platform/extensions/protocols.py` defines `PermissionExtension` with `check_permission()`, `filter_visible()`, and `can_access_dataset()`.
- Default implementation: `DefaultPermissionExtension` preserves Community capability checks, admin behavior, anonymous public/published filtering, private owner access, restricted grants, and admin bypass behavior.
- Accessor: `get_permission_extension()` returns a singleton registry entry or the Community default.
- Production call sites: `require_permission()` delegates capability decisions to `check_permission()`. `catalog/authorization.py` delegates list filtering to `filter_visible()` and dataset detail checks to `can_access_dataset()`.
- Overlay tests: `backend/tests/test_permission_extension.py` proves registry replacement can allow/deny permission checks, alter visibility filters, and deny dataset access without core changes.
- Architecture guard: `backend/tests/test_layering.py::test_permission_chokepoints_use_extension` fails if `require_permission()`, `apply_visibility_filter()`, or dataset access helpers bypass the extension.

### WorkflowExtension — Ready

`WorkflowExtension` is also a real seam:

- Protocol: `backend/app/platform/extensions/protocols.py` defines `WorkflowTransitionContext` and `WorkflowExtension` with `status_order()`, `allowed_transitions()`, and `on_transition()`.
- Default implementation: `DefaultWorkflowExtension` preserves the Community `draft -> ready -> internal -> published` lifecycle and default allowed-transition map.
- Accessor: `get_workflow_extension()` returns a singleton registry entry or the Community default.
- Production call sites: `/status/`, `/target-status/`, and metadata PATCH `record_status` writes build `WorkflowTransitionContext`, call `allowed_transitions()`, and invoke `on_transition()` for real transitions.
- Overlay tests: `backend/tests/test_workflow_extension.py` proves overlays can add direct transitions, observe intermediate target-status transitions, block transitions, block metadata PATCH publication, and persist custom `review` status.
- Architecture guard: `backend/tests/test_layering.py::test_workflow_publication_chokepoints_use_extension` fails if the known publication chokepoints bypass `WorkflowExtension`.

The seam grade moves the 2026-05-03 open-core audit's policy/workflow entries from Adaptable to Ready for the surfaces in v13.5 scope. Product UI for field-level RBAC and full approval workflows remains future scope, not a seam-quality blocker.
